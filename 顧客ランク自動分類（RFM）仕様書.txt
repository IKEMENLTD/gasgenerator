# 顧客ランク自動分類（RFM）システム仕様書

## 1. システム概要

### 1.1 基本情報
- システム名: 顧客ランク自動分類（RFM）
- バージョン: 1.0.0
- 提供形態: Google Apps Script (GAS) ウェブアプリ
- スプレッドシートID: 1uQOsVim0PYvHv9e3B9ic55USEnEdvew8DZsuhDcL4VM

- デプロイメントID: AKfycbwYn8uAPW16oEs7MuY2LDQQ2fIui8hNd7bP-nDSbPfM47OERiLK2XU1fsVUUpDhl__h
- Web App URL: https://script.google.com/macros/s/AKfycbwYn8uAPW16oEs7MuY2LDQQ2fIui8hNd7bP-nDSbPfM47OERiLK2XU1fsVUUpDhl__h/exec

### 1.2 システムの目的
購入履歴データからRFM分析（Recency/Frequency/Monetary）を自動計算し、顧客をランク別に自動分類するシステム。施策管理・レポート出力・ダッシュボード可視化を統合したワンストップの顧客管理ウェブアプリ。

### 1.3 技術スタック
- フロントエンド: HTML/CSS/JavaScript SPA（Index.html単一ファイル）
- バックエンド: Google Apps Script (Code.gs)
- データベース: Google Spreadsheet（18シート構成）
- UIデザイン: Soft UI / Neumorphic デザイン
- グラフ描画: Chart.js（遅延読み込み）
- デプロイ: clasp（自動化スクリプト付き）

### 1.4 対象ユーザー
- EC事業者、小売業、サービス業の顧客管理担当者
- マーケティング担当者
- 経営分析担当者

## 2. 主要機能

### 2.1 RFMスコア自動計算
- Recency（最終購入日からの経過日数）
- Frequency（購入回数）
- Monetary（合計購入金額）
- 各指標を5段階スコア（1-5）で自動評価
- スコア閾値はカスタマイズ可能

### 2.2 顧客ランク自動分類
- RFMスコア合計に基づく自動ランク付け
- デフォルトランク: VIP、優良、一般、休眠、離脱
- ランク名・条件・表示色はカスタマイズ可能
- ランク変動履歴の自動記録

### 2.3 顧客管理
- 顧客一覧表示（ページネーション・ソート・検索・ランクフィルタ）
- 顧客詳細画面（基本情報/RFM履歴/購入履歴/施策履歴タブ切替）
- 顧客の新規登録・編集・無効化
- タグ管理（複数タグ付与・カテゴリ分類）

### 2.4 購入履歴インポート
- CSVファイルインポート（ドラッグ&ドロップ対応）
- プレビュー表示・バリデーション
- インポート履歴管理

### 2.5 施策管理
- 施策記録（対象顧客・施策内容・チャネル・効果記録）
- テンプレート管理
- 施策実行履歴

### 2.6 レポート
- 期間指定レポート生成
- ランク分布集計
- RFMスコア再計算（手動トリガー）

### 2.7 ダッシュボード
- 顧客数サマリー（総数/アクティブ/新規/休眠）
- ランク分布チャート（ドーナツグラフ）
- RFMスコア分布チャート（棒グラフ）
- 最近のアクティビティ表示

### 2.8 設定管理
- RFM設定（スコア閾値カスタマイズ）
- ランク設定（ランク名・条件・色）
- ユーザー管理（管理者のみ）
- 店舗/チャネル管理
- タグ管理
- システム設定

| triggerDailyReport | 毎日 | 日次レポート生成 |
| triggerWeeklyReport | 毎週 | 週次レポート生成 |
| triggerMonthlyReport | 毎月 | 月次レポート生成 |
| triggerNotification | 毎時 | 通知チェック・送信 |
| triggerDataCleanup | 毎週 | 古いデータのクリーンアップ |

## 8. 初期セットアップ手順

1. GASエディタで `initializeSheets()` を実行（18シート自動作成）
2. デフォルト管理者: admin@example.com / admin123
3. RFM設定・ランク設定がデフォルト値で初期化される
4. 顧客データ・購入履歴をCSVインポートまたは手動登録

---

## 9. デプロイ方法

```bash
npm run deploy
```
内部で `scripts/deploy-gas.sh` が実行され、clasp push + clasp deploy を自動実行。

---

## 10. セキュリティ

- GAS WebApp: ANYONE_ANONYMOUS（URLを知っている人がアクセス可能）
- executeAs: USER_DEPLOYING（デプロイユーザーの権限で実行）
- フロントエンド-バックエンド通信: google.script.run（GAS内部通信）
- XSS対策: HTMLエスケープ、イベント委任パターン使用
