```
【敵②】自分でやるのは無理

・3日で挫折した

```
【敵③】AIツールは結局難しい

ChatGPTにコード書かせてみた。

・「これでいけます」って言われたコードが動かない

AIは万能じゃない。
「コードを書く」と「実務で使える」は別物。
```

### 敵④：時間の罠

```
【敵④】時間がない

「自動化の勉強をする時間があったら、
その時間で手作業した方が早い」

そう思ってしまう。

でも、それは罠です。

今日2時間手作業をする。

「自動化しない」から時間がないのです。

## 6. 差別化・メカニズムセクション

### セクションタイトル

```
なぜ「選ぶだけ」で業務が変わるのか？
```

### 理由①：プロ品質

```
【理由①】プロが「実務で使える」前提で作っている

TaskMateのシステムは、
「動く」ではなく「使える」を基準に作られています。

▼ ChatGPT生成コードとの違い

【ChatGPT】
・「動くかもしれない」レベル
・エラー処理が不十分
・実務特有のケースに対応していない
・メンテナンスしにくい構造

【TaskMate】
・「実務で使われている」レベル
・エラー処理完備
・業界特有の要件を反映
・保守しやすい設計

現役エンジニアが設計し、
実際の企業で稼働しているシステムを
テンプレート化しています。

「試しに使ってみたら動かなかった」は、ありません。
```

### 理由②：即戦力

```
【理由②】設定不要で、すぐ使える

ダウンロードしたシステムは、
そのまま使える状態で提供されます。

・必要な設定：最小限
・マニュアル：付属
・設置方法：動画で解説

「ダウンロードしたけど、使い方が分からない」
そんな心配は無用です。

【導入ステップ】
1. LINEで「〇〇システムください」と送る
2. ダウンロードリンクが届く
3. Googleスプレッドシートにコピー
4. 完了。使い始める。

ITに詳しくない方でも、
5分で導入できます。
```

### 理由③：柔軟性

```
【理由③】「完全オーダーメイド」じゃないけど「柔軟」

TaskMateは、完全なフルスクラッチ開発ではありません。
でも、「テンプレートそのまま」でもありません。

▼ カスタマイズ対応の範囲

【対応可能】
・送信先メールアドレスの変更
・通知タイミングの変更（毎日→毎週など）
・連携先の変更（Slack→Chatwork など）
・項目名・ラベルの変更
・集計ロジックの調整
・権限設定の変更

【要相談】
・大幅な機能追加
・外部APIとの連携追加
・複数システムの統合

「ちょっとだけ変えたい」に、
追加料金なしで対応します。
```

### 理由④：コスト

```
【理由④】失敗しても痛くない価格設計

外注開発の相場を知っていますか？

▼ 外注開発の相場
・営業日報システム：50〜100万円
・在庫管理システム：80〜150万円
・請求書自動化：30〜80万円
・勤怠管理：100〜200万円

▼ TaskMateの場合
・1システムあたり：約1.7万〜2万円

差額は、最大で「50倍」。

外注で100万円かけて、
「思ってたのと違う...」は致命的。

TaskMateなら、
「試しに使ってみて、合わなかったらやめる」
ができる価格です。

リスクなく、業務自動化を始められます。
```

## 7. カスタマイズ対応セクション

### セクションタイトル

```
「ここだけ変えたい」も、LINEで解決
```

### 導入テキスト

```
システムをダウンロードしたら終わり、ではありません。
「ここをこう変えたい」にも対応します。
```

### カスタマイズの流れ

```
【カスタマイズの流れ】

STEP 1：LINEで相談
「売上日報の送信先を、Slackに変えたい」
「毎週月曜じゃなくて、毎日送信にしたい」
など、変えたい内容をLINEで送る。

STEP 2：エンジニアが対応
公式LINEには、全システムの情報が読み込まれています。
「ここをこう変える」という指示を、
エンジニアがシステムに反映。

STEP 3：修正版を受け取る
修正されたシステムをダウンロード。
差し替えて、完了。

【ポイント】
・追加料金：なし（プランに含まれる）
・対応時間：営業時間内、原則24時間以内
・修正回数：無制限
```

### 具体的なカスタマイズ例

```
【実際のカスタマイズ例】

▼ 例1：売上日報システム
「毎朝9時ではなく、7時に送信してほしい」
→ トリガー時間を変更して対応

▼ 例2：失客アラート
「30日未来店ではなく、14日で通知してほしい」
→ 判定条件を変更して対応

▼ 例3：経費精算
「承認者を2段階（上司→経理）にしたい」
→ 承認フローを追加して対応

▼ 例4：請求書自動生成
「自社のロゴと振込先口座を入れたい」
→ テンプレートを編集して対応

▼ 例5：勤怠管理
「残業時間が月30時間を超えたらアラートを出したい」
→ アラート条件を追加して対応

▼ 例6：在庫アラート
「Slackではなく、Chatworkに通知したい」
→ 連携先を変更して対応
```

### 対応範囲の明示

```
【カスタマイズ対応範囲】

◎ 追加料金なしで対応

### 理由①：時間損失

```

明日も、明後日も、来月も、来年も。

▼ あなたが失う時間

1日2時間 × 20営業日 = 月40時間
月40時間 × 12ヶ月 = 年間480時間
年間480時間 = 丸々20日分の労働時間

「来月から始めよう」
「落ち着いたら検討しよう」

その1ヶ月で、40時間失います。
その1年で、480時間失います。

時間は、取り戻せません。
```

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
