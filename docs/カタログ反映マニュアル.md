## 1. やることの全体像

カタログにシステムを追加した後、**2つの反映作業**があります。

| 作業 | 何をする？ | 自動化状況 |
|------|-----------|-----------|
| **A: システム数の表記更新** | LP・LINE Botに表示されている数字を最新に合わせる | **自動**（GitHub Actions + prebuild） |
| **B: AI診断データの追加** | 新システムがAI診断で推薦されるようデータを追加 | 手動（Claude Code で対応） |

> **前提:** カタログへのシステム追加（[page.tsx](../app/systems/catalog/page.tsx)）は完了済みであること。

---

## 2. 作業 A: システム数の表記更新（自動）

### 仕組み

`app/systems/catalog/page.tsx` を変更して `git push` すると、**GitHub Actions が自動で以下を実行**します。

1. カタログの `systems` 配列からシステム数をカウント
2. 以下のファイルの数字を自動更新:
   - `lib/generated/system-count.ts` — React/Next.js用の定数
   - `public/lp-page.html` — Render版LP（3箇所）
   - `netlify-tracking/frontend/index.html` — Netlify版LP（3箇所）
   - `netlify-tracking/netlify/functions/admin-agencies.js` — 代理店メール（5箇所）
3. 更新結果を自動コミット＆プッシュ
4. **pushをトリガーにNetlifyが自動ビルド＆デプロイ**（taskmateai.net に即反映）

さらに、Renderデプロイ時に `prebuild` スクリプトが再実行されるため、二重に安全です。

> **GitHub Actions の手動実行:** カタログ変更なしでも再デプロイしたい場合は、
> [Actions → Sync system count → Run workflow](https://github.com/IKEMENLTD/gasgenerator/actions/workflows/sync-system-count.yml) から手動実行できます。

### 担当者がやること

**なし。** `page.tsx` を編集して `git push` するだけで、全箇所のシステム数が自動的に更新されます。

### 手動実行（必要な場合のみ）

```bash
npm run sync-count
```

> **注意:** カタログを連続で追加してpushする場合、2回目のpushが拒否されることがあります（自動コミットとの競合）。`git pull` してから再度 `git push` してください。

## 3. 作業 B: AI診断データの追加（手動）

新システムをAI診断の推薦候補に加えるための手順です。

### 対象ファイル

| ファイル | 役割 |
|----------|------|
| [netlify-tracking/frontend/shindan.html](../netlify-tracking/frontend/shindan.html) | Web版AI診断のシステムデータ（`SYSTEMS` オブジェクト） |
| [lib/data/systems-data.ts](../lib/data/systems-data.ts) | LINE Bot版AI診断のデータソース（`getSystemsData()` 関数） |

### Claude Code への指示（コピペ用）

```text
以下の新システムをAI診断データに追加して。

対象ファイル:
1. netlify-tracking/frontend/shindan.html の SYSTEMS オブジェクト
2. lib/data/systems-data.ts の getSystemsData() 関数

追加するシステム:
- ID: ＿＿, name: '＿＿＿', tagline: '＿＿＿', tags: ['＿＿', '＿＿']

※ shindan.html では INDUSTRY_SYSTEMS と PAIN_SYSTEMS のマッピングにも追加が必要な場合あり。
  適切な業種カテゴリ・課題カテゴリに新システムのキーを追加すること。
```

<summary>データ構造の参考（クリックで展開）</summary>

**shindan.html:**
```javascript

```

**systems-data.ts:**
```typescript

```

## 4. コミット＆デプロイ

### Step 1: コミット＆プッシュ（Render自動デプロイ）

```bash
git add netlify-tracking/frontend/shindan.html lib/data/systems-data.ts
git commit -m "feat: AI診断データに新システム追加"
git push origin main
```

> **完了:** push 後、Render（Next.js版）は5〜10分で自動デプロイされます。
> システム数の表記は GitHub Actions が自動で更新・コミットします。

### Step 2: Netlify LP のデプロイ（自動）

**手動デプロイは不要です。** Netlifyの自動ビルドが有効になっているため、`git push` をトリガーにNetlifyが自動でビルド＆デプロイします（taskmateai.net）。

> **手動デプロイが必要な場合（緊急時のみ）:**
> ```bash
> cd netlify-tracking && npx netlify deploy --prod --dir=frontend
> ```
> ※ `--no-build` は絶対に付けないこと。

## 5. 反映確認チェックリスト

デプロイ後、以下の順番で確認します。**Netlify = 即時反映**、**Render = 5〜10分後**。

### Phase 1: 即時確認（Netlifyデプロイ直後）

- [ ] **Netlify版LP のシステム数**
  - [https://taskmateai.net](https://taskmateai.net) を開く
  - `Ctrl + F` で「`システム公開中`」を検索 → 数字が正しいか確認

- [ ] **AI診断ページの動作**
  - [https://taskmateai.net/shindan](https://taskmateai.net/shindan) を開く
  - 業種を選択 → 課題を選択 → 診断結果が表示されることを確認
  - 新しく追加したシステムが結果に含まれるか確認

- [ ] **LP の「AIシステム診断」ボタン**
  - [https://taskmateai.net](https://taskmateai.net) を少しスクロール
  - 右下の緑色「AIシステム診断」ボタンをクリック
  - [taskmateai.net/shindan](https://taskmateai.net/shindan) に遷移すればOK

### Phase 2: 5〜10分後に確認（Render自動デプロイ完了後）

> git push から5〜10分後に確認してください。まだ古い表示の場合は `Ctrl + Shift + R`（強制再読み込み）を試してください。

- [ ] **Render版LP のシステム数**
  - [https://gasgenerator.onrender.com](https://gasgenerator.onrender.com) を開く
  - `Ctrl + F` で「`システム公開中`」を検索 → Netlify版と同じ数字ならOK

- [ ] **LINE Bot のメニュー表示**
  - TaskMate公式LINEで「`メニュー`」と送信
  - 返信メニュー内の「**N種類の自動化システム**」の数字を確認

- [ ] **LINE Bot のAI診断**
  - TaskMate公式LINEで「`AI診断`」と送信
  - 診断を進め、新システムが推薦候補に含まれるか確認
  - ※ 業種・課題の組み合わせによっては表示されない場合もある（正常動作）

| 確認項目 | URL / 操作 | 反映タイミング |
|----------|-----------|---------------|
| Netlify版LP | [https://taskmateai.net](https://taskmateai.net) | 即時 |
| AI診断ページ | [https://taskmateai.net/shindan](https://taskmateai.net/shindan) | 即時 |
| Render版LP | [https://gasgenerator.onrender.com](https://gasgenerator.onrender.com) | 5〜10分後 |

> **全項目OK:** 6項目すべて正しい数字・データが表示されていれば、反映完了です。

## 6. トラブルシューティング

| 問題 | 解決策 |
|------|--------|
| git push が拒否された（non-fast-forward） | GitHub Actions の自動コミットと競合。`git pull` してから再度 `git push` |
| AI診断で新システムが出ない | `shindan.html` の `SYSTEMS` に追加されているか確認。`INDUSTRY_SYSTEMS` / `PAIN_SYSTEMS` のマッピングも確認 |
| LINE Bot のAI診断で新システムが出ない | [lib/data/systems-data.ts](../lib/data/systems-data.ts) に追加されているか確認。git push が必要 |
| GitHub Actions が失敗した | [Actions ログ](https://github.com/IKEMENLTD/gasgenerator/actions) で確認。手動で `npm run sync-count` を実行してcommit |
| git push が権限エラー | GitHub の Collaborator に追加されているか確認 |
| Netlify 自動ビルドが走らない | Netlify Dashboard → Deploy settings で「Activate builds」が有効か確認（info@ikemen.ltdでログイン） |
| Render版がいつまでも古い | [Render ダッシュボード](https://dashboard.render.com) でデプロイ状況を確認 |
