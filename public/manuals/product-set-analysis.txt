========================================
商品セット販売分析 使い方説明書
========================================

■ このツールでできること
────────────────────────────────────────
・注文データから「一緒に買われやすい商品ペア」を自動で抽出・集計できます
・同時購買率や売上貢献額のランキングで、セット販売の候補を客観的に見つけられます
・月別・週別のトレンド推移で、季節ごとの購買パターンの変化を把握できます
・カテゴリ別・チャネル別・顧客セグメント別のクロス分析ができます
・セット候補のシミュレーション（セット価格・割引率・粗利率）が行えます
・キャンペーン施策の前後で同時購買率の変化を検証できます
・同時購買率の急上昇・急低下をアラートで検知し、メール通知できます
・ダッシュボードで全体の傾向を一目で確認できます


■ 画面構成（全9タブ）
────────────────────────────────────────

1. ダッシュボード
2. 商品マスタ
3. 注文管理
4. ペア分析（5つのサブタブ）
5. セット候補
6. キャンペーン
7. マスタ管理（3つのサブタブ）
8. アラート
9. 設定（2つのサブタブ）


========================================
1. ダッシュボード
========================================

システム全体の概況を確認する画面です。

【表示項目】
・統計サマリー（5つのカード）
  - 総ペア数、Top1ペア同時購買率、今月注文件数、
    セット候補ステータス（検討中/採用/却下）、未通知アラート件数

・同時購買率 Top10 ペア一覧
  同時購買率が高い商品ペアのランキングです。
  どの組み合わせがよく一緒に購入されているかがわかります。

・売上貢献額 Top10 ペア一覧
  同時購買による売上額が大きい商品ペアのランキングです。
  売上インパクトの大きいセット候補の判断に使えます。

・トレンド推移（Top3ペア）
  上位3ペアの月別同時購買率の推移をグラフで表示します。
  トレンドの上昇・下降が視覚的にわかります。

・カテゴリ別クロス購買ヒートマップ
  カテゴリ同士の同時購買率をマトリクス形式で表示します。
  どのカテゴリの組み合わせが強いかが一目でわかります。

・注目ペア（前月比+10%以上）
  先月と比べて同時購買率が10%以上上昇したペアを表示します。
  新たに注目すべき組み合わせの発見に役立ちます。

・最新アラート
  直近のアラート（急上昇・急低下など）を表示します。

・チャネル別ペアランキング比較
  店舗・ECなどチャネルごとのTop5ペアを横並びで比較できます。

・セット採用商品の売上貢献合計額
  「採用」ステータスのセット候補の売上貢献額合計です。

・データ最終更新日時
  集計データが最後に更新された日時を表示します。


========================================
2. 商品マスタ
========================================

販売商品の基本情報を管理する画面です。

【操作方法】
・「+ 商品登録」ボタン → 新しい商品を登録
  必須項目：商品名、カテゴリ
  任意項目：サブカテゴリ、単価、原価、販売チャネル、季節フラグ、ステータス、備考

・検索ボックス → 商品名やカテゴリで絞り込み検索
・ステータスフィルタ → 販売中/販売停止/廃番で絞り込み
・各行の「編集」ボタン → 商品情報を編集
・各行の「削除」ボタン → 商品を削除（確認ダイアログあり）

【登録項目】
・商品ID（自動採番）
・商品名、カテゴリ、サブカテゴリ
・単価（円）、原価（円）、粗利率（自動計算）
・販売チャネル（店舗/EC/両方）
・季節フラグ（通年/春/夏/秋/冬）
・ステータス（販売中/販売停止/廃番）
・備考


========================================
3. 注文管理
========================================

注文データの登録・管理を行う画面です。

【操作方法】
・「+ 注文登録」ボタン → 注文を手動で登録
  注文ヘッダ：注文日時、チャネル、顧客ID、顧客セグメント
  注文明細：商品を選択して数量・単価を入力（複数行追加可能）

・「CSV取込」ボタン → CSVデータから一括取込
  ヘッダ行とデータ行を貼り付けて取り込めます。
  注文IDの重複チェックが自動で行われ、重複行はスキップされます。

・検索ボックス → 注文IDや商品名で絞り込み検索

【登録される項目】
・注文ID（自動採番：ORD + 日付 + 連番）
・明細ID（自動採番）
・注文合計金額・商品点数（自動計算）


========================================
4. ペア分析
========================================

同時購買ペアの分析結果を確認する画面です。
5つのサブタブで構成されています。

【共通操作】
・画面上部の「再集計」ボタン
  → 全ペア分析（ランキング・トレンド・カテゴリクロス・チャネル別・
     セグメント別）を一括で再集計します

────────────────────────
4-1. ランキング
────────────────────────
・同時購買ペアを同時購買率の高い順にランキング表示
・表示項目：順位、商品A、商品B、同時購買件数、
  同時購買率（A基準/B基準/平均）、売上貢献額
・ページネーション対応

────────────────────────
4-2. トレンド
────────────────────────
・月次/週次の切り替えが可能
・上位ペアの同時購買率推移をグラフで表示
・テーブルでも各期間の詳細数値を確認可能
  （集計年月、商品ペア、同時購買件数、同時購買率、前月比/前週比、売上貢献額）

────────────────────────
4-3. カテゴリクロス
────────────────────────
・カテゴリ同士のクロス購買をテーブルで表示
・表示項目：カテゴリA、カテゴリB、クロス購買件数、クロス購買率、売上貢献額
・どのカテゴリの組み合わせが多いか把握できます

────────────────────────
4-4. チャネル別
────────────────────────
・チャネル（店舗/ECなど）ごとの同時購買ペアランキング
・チャネルフィルタで絞り込み可能
・表示項目：順位、商品A、商品B、同時購買件数、同時購買率

────────────────────────
4-5. セグメント別
────────────────────────
・顧客セグメント（新規/リピーター/VIPなど）ごとの同時購買ペア
・セグメントフィルタで絞り込み可能
・表示項目：順位、商品A、商品B、同時購買件数、同時購買率


========================================
5. セット候補
========================================

セット販売の候補を管理する画面です。

【操作方法】
・「候補生成」ボタン → 主商品を選択すると、相性の良い商品Top5を自動抽出
  セット価格・割引率の入力も可能です。
  ステータス「検討中」で登録されます。

・ステータスフィルタ → 検討中/採用/却下で絞り込み
・各行の「採用」「却下」ボタン → ステータスを変更

【表示項目】
・候補ID、セット名称
・主商品名、組み合わせ商品名
・同時購買率
・想定セット価格、通常合計価格、割引率
・想定粗利率
・ステータス（検討中/採用/却下）
・登録日、担当者コメント


========================================
6. キャンペーン
========================================

セット販売キャンペーンの登録と効果検証を行う画面です。

【操作方法】
・「+ キャンペーン登録」ボタン → 新規キャンペーンを登録
  入力項目：キャンペーン名（必須）、対象ペアID、商品名A、商品名B、
  期間開始日（必須）、期間終了日（必須）

・「効果検証」ボタン（各行）
  → キャンペーン期間の前・中・後の同時購買率を自動集計
  → 効果判定（効果あり/効果なし）を自動付与

・「削除」ボタン（各行）→ キャンペーンを削除

【表示項目】
・キャンペーン名、商品A、商品B
・期間
・施策前/施策中/施策後の同時購買率
・効果判定


========================================
7. マスタ管理
========================================

商品以外のマスタデータを管理する画面です。
3つのサブタブで構成されています。

────────────────────────
7-1. カテゴリ
────────────────────────
・商品カテゴリの登録・編集・削除
・登録項目：カテゴリ名、親カテゴリID、表示順、備考
・検索ボックスで絞り込み可能

────────────────────────
7-2. チャネル
────────────────────────
・販売チャネルの登録・編集・削除
・登録項目：チャネル名、チャネル種別、担当者名、ステータス
・検索ボックスで絞り込み可能

────────────────────────
7-3. 顧客セグメント
────────────────────────
・顧客セグメントの登録・編集・削除
・登録項目：セグメント名、購買頻度条件、累計購買金額条件、備考
・検索ボックスで絞り込み可能


========================================
8. アラート
========================================

同時購買率の急変を検知・確認する画面です。

【操作方法】
・「手動チェック」ボタン → アラート検知を即時実行
  前回集計と比較して、設定した閾値を超える変化があったペアを検知します。

【アラート種別】
・急上昇：同時購買率が閾値以上に上昇したペア
・急低下：同時購買率が閾値以上に低下したペア
・新規ペア出現：新たに検知されたペア
・ペア消滅：前回あったが今回消えたペア

【表示項目】
・アラートID、検知日時、アラート種別
・商品A、商品B
・変化前の値、変化後の値、変化率
・通知先メールアドレス、通知済フラグ

【メール通知】
・設定画面で「通知先メールアドレス」を設定すると、
  未通知のアラートをメールで自動送信します。


========================================
9. 設定
========================================

システムの各種設定とトリガー管理を行う画面です。

────────────────────────
9-1. システム設定
────────────────────────
以下の設定項目を変更できます。変更後は「保存」ボタンを押してください。

・同時購買率アラート閾値（上昇）：初期値 20（%以上変化でアラート）
・同時購買率アラート閾値（低下）：初期値 -20（%以上変化でアラート）
・最小同時購買件数フィルタ：初期値 1（件未満のペアを分析から除外）
・集計対象期間（月）：初期値 12（直近N月分を集計対象）
・通知先メールアドレス：アラート通知の送信先
・Top表示件数：初期値 20（ランキング表示件数）

【データ操作】
・「シート初期化」ボタン → 全シートのヘッダー作成、初期設定投入、
  条件付き書式・入力規則の設定を実行
・「クリーンアップ」ボタン → 古いデータのアーカイブ、不要データの削除

────────────────────────
9-2. トリガー管理
────────────────────────
自動実行トリガーの設定・削除を行います。

・「トリガー設定」ボタン → 以下のトリガーを一括設定
  - 毎日 2:00：商品ペア同時購買集計
  - 毎日 3:00：ダッシュボード更新
  - 毎日 7:00：アラート検知・通知
  - 毎週月曜 8:00：月別・週別トレンド集計
  - 毎週月曜 9:00：週次サマリーレポートメール送信
  - 毎月1日 3:00：カテゴリ別・チャネル別クロス集計
  - 毎月1日 4:00：データクリーンアップ

・「全削除」ボタン → 全トリガーを削除
・現在設定中のトリガー一覧を確認できます


========================================
スプレッドシートのカスタムメニュー
========================================

スプレッドシートの上部メニューバーに「🔍 セット分析」メニューが
表示されます（スプレッドシートを開き直すと表示されます）。

・注文データ取込 → CSV形式の注文データを一括取込
・ペア集計実行 → 商品ペア同時購買を手動集計
・セット候補提案 → セット候補の自動生成
・キャンペーン効果検証 → キャンペーン効果の集計
・トレンド集計 → 月別・週別トレンドの手動集計
・クロス集計 → カテゴリ別・チャネル別クロス集計の手動実行
・アラートチェック → アラート検知の手動実行


========================================
基本的な使い方の流れ
========================================

1. 初期設定
   → 設定 > システム設定 で初回の「シート初期化」を実行
   → 必要に応じて設定値を変更して「保存」

2. マスタデータ登録
   → マスタ管理でカテゴリ・チャネル・顧客セグメントを登録
   → 商品マスタで商品情報を登録

3. 注文データ登録
   → 注文管理で注文を手動登録、またはCSV一括取込

4. ペア分析の実行
   → ペア分析タブの「再集計」ボタンで全分析を一括実行
   → ランキング・トレンド・カテゴリクロス・チャネル別・セグメント別を確認

5. セット候補の検討
   → セット候補タブの「候補生成」で主商品を選んで候補を自動抽出
   → 同時購買率・割引率・粗利率を確認して「採用」「却下」を判断

6. キャンペーン効果検証
   → キャンペーンを登録し、期間終了後に「効果検証」で成果を確認

7. 定期運用
   → トリガーを設定すると、集計・アラート・レポートが自動実行されます
   → ダッシュボードで定期的に全体の傾向を確認
