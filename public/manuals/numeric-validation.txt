============================================================
  数値入力バリデーション　使い方説明書
============================================================

■ このシステムについて
  見積・発注・請求の数値入力時に、自動でバリデーション
  （整合性チェック）を行い、入力ミスを未然に防ぐシステムです。
  桁違い・範囲超過・計算不一致などを自動検出し、
  警告・エラーとして表示します。

============================================================
【1】ダッシュボード
============================================================
  システムを開くと最初に表示される画面です。

  ■ 上部カード（5つ）
    ・今日の入力件数 … 本日登録されたデータの件数
    ・今日の警告率 … 本日の入力のうち警告/エラーが出た割合
    ・未確認の警告 … まだ対応していない警告の件数
    ・ミス防止件数 … 「修正済」にした件数
    ・ミス防止金額 … 修正前後の差額合計（修正による節約額）

  ■ グラフ
    ・ミスパターン別件数（円グラフ）
      → どんな種類の警告が多いかを表示
      → グラフにカーソルを合わせると全文表示
    ・個人別警告発生率（棒グラフ）
      → 入力者ごとの警告率を表示
    ・商品別異常値検出率（横棒グラフ）
      → 商品ごとに異常値が出た割合を表示

  ■ 最近の警告・エラー
    直近10件の警告/エラーを一覧表示します。

============================================================
【2】入力データ管理
============================================================
  見積・発注・請求データの入力と管理を行います。

  ■ 新規入力の手順
    1.「+ 新規入力」ボタンをクリック
    2. 入力種別（見積/発注/請求）を選択
    3. 入力者名を入力
    4. 取引先名を入力（任意・候補から選択可）
    5. 商品を選択
       → 選択すると商品情報カード（標準単価・許容範囲・
         過去平均など）が表示されます
    6. 入力単価・入力数量を入力
       → 小計は自動計算されます
    7.「チェックして保存」ボタンをクリック

  ■ バリデーション結果の3パターン
    ・OK → そのまま自動保存されます
    ・警告 → 警告内容が表示されます
      → 確定理由を入力して「理由を入力して警告を確定」で保存
    ・エラー → エラー内容が表示されます
      → 確定理由を入力して「理由を入力してエラーを確定」で保存
    ※ 警告・エラーとも、理由の入力は必須です

  ■ 一覧の操作
    ・種別フィルタ … 見積/発注/請求で絞り込み
    ・結果フィルタ … OK/警告/エラーで絞り込み
    ・検索 … キーワードで検索
    ・編集/削除 … 各行の操作ボタンから

============================================================
【3】商品マスタ
============================================================
  バリデーションの基準となる商品情報を管理します。

  ■ 登録項目
    ・商品名（必須）
    ・カテゴリ
    ・標準単価 … この金額を基準に乖離率を計算
    ・単価下限/上限 … この範囲外で警告が出ます
    ・数量単位（個/箱/式など）
    ・最小/最大発注数量 … この範囲外で警告が出ます
    ・過去平均単価/数量 … 比較用の参考値
    ・ステータス（有効/廃止）
      → 廃止にすると入力時の商品選択に表示されません

  ■ 操作
    ・「+ 新規登録」で追加
    ・一覧の「編集」で修正、「削除」で削除
    ・検索ボックスでキーワード検索

============================================================
【4】バリデーションルール
============================================================
  チェック条件を管理します。
  初期設定で6つのルールが登録されています。

  ■ 初期ルール
    ・単価下限チェック … 標準の50%以下で警告
    ・単価上限チェック … 標準の200%以上で警告
    ・異常数量チェック … 過去平均の10倍以上で警告
    ・高額確認 … 1億円以上で確認
    ・ゼロ入力チェック … 単価0円でエラー
    ・マイナス値チェック … マイナス値でエラー

  ■ 登録項目
    ・ルール名（必須）
    ・対象項目（単価/数量/金額/その他）
    ・チェック種別（範囲/桁数/乖離率/計算）
    ・条件 … チェックの閾値
    ・警告レベル（警告/エラー/確認）
    ・適用対象（全商品/特定カテゴリ）
    ・有効/無効 … 無効にするとチェックされません
    ・警告メッセージ … 検出時に表示される文言

============================================================
【5】警告ログ
============================================================
  検出された警告・エラーの履歴と対応状況を管理します。

  ■ 一覧の表示項目
    ログID / 日時 / 入力者 / 商品名 / 入力値 / 基準値 /
    乖離率 / レベル / 対応状況

  ■ 対応の手順
    1. 一覧の「対応」ボタンをクリック
    2. 警告の詳細が表示されます
    3. 対応を選択
       ・修正 → 修正後の値を入力
       ・そのまま確定 → 問題なしと判断
       ・キャンセル → 入力自体を取り消す判断
    4. 対応者コメントを入力（必須）
    5.「保存」で対応完了

    ※「修正」を選んで修正後の値を入力すると、
      ダッシュボードの「ミス防止金額」に差額が反映されます

  ■ フィルタ
    ・レベル別（警告/エラー/確認）
    ・対応別（未対応/修正/そのまま確定/キャンセル）
    ・キーワード検索

============================================================
【6】取引先管理
============================================================
  取引先ごとの基準値を管理します。

  ■ 登録項目
    ・取引先名（必須）
    ・商品 … 紐づける商品を選択
    ・過去平均単価/数量
    ・過去最高単価/最低単価
    ・取引回数
    ・最終取引日

  ■ 操作
    ・「+ 新規登録」で追加
    ・一覧の「編集」で修正、「削除」で削除

============================================================
【7】分析レポート
============================================================
  入力ミスの傾向を分析するレポートです。

  ■ 個人別傾向タブ
    ・「週次レポート生成」ボタン
      → 過去1週間の個人別集計を生成
    ・「月次レポート生成」ボタン
      → 当月の個人別集計を生成
    ・表示項目：
      氏名 / 期間 / 総入力件数 / 警告数 / 警告率 /
      修正数 / 主なミスパターン / 要注意フラグ
    ※ 警告率30%以上で「要注意」フラグが付きます

  ■ 全体傾向タブ
    ・月次の全体集計が表示されます
    ・表示項目：
      期間 / 総入力件数 / 警告数 / 警告率 /
      桁違い検出件数 / 計算ミス検出件数 /
      ミス防止金額 / 最多ミスパターン

  ■ 自動レポート生成
    ・週次：毎週月曜 9:00 に自動生成
    ・月次：毎月1日 8:00 に自動生成

============================================================
【8】設定
============================================================
  システムの詳細設定を管理します。

  ■ 桁違いパターン検出設定
    桁違い（0の個数間違い）を検出するパターンです。
    初期設定で4パターンが登録されています。
    ・0が1個多い（入力値÷基準値=10）
    ・0が1個少ない（入力値÷基準値=0.1）
    ・0が2個多い（入力値÷基準値=100）
    ・0が2個少ない（入力値÷基準値=0.01）

  ■ 計算チェック設定
    計算の整合性をチェックする設定です。
    初期設定で3パターンが登録されています。
    ・単価×数量の一致チェック（許容誤差1円）
    ・高額取引確認（1億円以上）
    ・消費税計算チェック（税抜×1.1=税込、許容誤差1円）

  ■ トリガー情報
    自動実行されるトリガーの一覧を確認できます。
    ・weeklyReportTrigger … 毎週月曜9時
    ・monthlyReportTrigger … 毎月1日8時

============================================================
【よくある質問】
============================================================

  Q. 警告やエラーが出ても保存できますか？
  A. はい。確定理由を入力すれば保存できます。
     警告・エラーとも理由の入力が必須です。

  Q. バリデーションの基準を変えたいです。
  A. 「商品マスタ」で標準単価・許容範囲を変更するか、
     「バリデーションルール」で条件を編集してください。

  Q. レポートは自動で作成されますか？
  A. はい。週次は毎週月曜9時、月次は毎月1日8時に
     自動生成されます。手動でも「分析レポート」画面から
     ボタンで即時生成できます。

  Q. 商品を使わなくなりました。
  A. 「商品マスタ」でステータスを「廃止」に変更して
     ください。入力時の選択肢に表示されなくなります。
     データは残ります。

  Q. ミス防止件数・金額が0のままです。
  A. 「警告ログ」で対応を「修正」にして修正後の値を
     入力すると反映されます。入力データの確認ステータスを
     「修正済」にするとミス防止件数に加算されます。
