======================================================
  トラブル対応履歴システム 使い方説明書
======================================================

■ はじめに

このシステムは、トラブル対応の記録・検索・分析を行うためのWebアプリです。
過去の対応履歴や解決パターンをチーム全体で共有し、
対応のスピードと品質を向上させることを目的としています。


======================================================
■ 画面の見方
======================================================

左側のサイドバーからページを切り替えます。
スマホではハンバーガーメニュー（≡）をタップしてください。

  メニュー                    用途
  ─────────────────────────────────────────────────
  ダッシュボード              全体の状況確認（最初に見る画面）
  トラブル検索                過去のトラブルを探す
  トラブル対応履歴            トラブルの登録・編集
  カテゴリマスタ              トラブル分類の管理
  解決パターン                よくある解決手順の管理
  再発防止策                  防止策の進捗管理
  エスカレーション            上位者への引き継ぎ記録
  月次分析サマリー            月ごとの集計データとグラフ
  カテゴリ別分析              カテゴリごとの傾向
  担当者別パフォーマンス      担当者ごとの実績
  再発トラブル追跡            繰り返し発生するトラブルの監視
  検索ログ                    検索キーワードの履歴


======================================================
■ よくある操作
======================================================

------------------------------------------------------
● トラブルが発生したとき
------------------------------------------------------

1. サイドバーから「トラブル対応履歴」を開く
2. 右上の「新規登録」をクリック
3. 以下を入力する
   - 発生日時        : トラブルが起きた日時
   - 報告者          : 報告してきた人の名前
   - 報告経路        : 電話・メール・チャット・自動検知から選択
   - カテゴリ        : トラブルの分類を選択
   - トラブル概要    : 何が起きたかを簡潔に
   - 症状・現象      : 具体的な症状（エラーメッセージなど）
   - 影響範囲        : 個人・部署・全社・顧客から選択
   - 緊急度          : 緊急・高・中・低から選択
   - 対応担当者ID / 担当者名 : 対応する人
4. 「登録」をクリック

※ 登録すると、自動で類似トラブルと解決パターンの候補が表示されます。
   過去に同じようなトラブルがあれば、その対応方法を参考にできます。


------------------------------------------------------
● 過去のトラブルを探したいとき
------------------------------------------------------

1. サイドバーから「トラブル検索」を開く
2. キーワードを入力（例: サーバー 停止）
   ※ スペースで区切ると、すべてのキーワードを含む結果だけが
     表示されます（AND検索）
3. 必要に応じてカテゴリ・ステータス・緊急度・期間で絞り込み
4. 「検索」をクリック

【類似検索を使う場合】
  「類似検索」タブに切り替えて、症状を入力すると、
  似たトラブルと解決パターンが自動でマッチングされます。


------------------------------------------------------
● トラブルの対応経過を記録するとき
------------------------------------------------------

1. 「トラブル対応履歴」でトラブルIDをクリック → 詳細モーダルが開く
2. 「詳細を編集」ボタンをクリック
3. 以下を記入する
   - 発生環境        : OS、ブラウザ、バージョンなど
   - 再現手順        : どうすれば再現できるか
   - エラーメッセージ: 表示されたエラー
   - 根本原因        : 調査で判明した原因
   - 解決方法        : 実施した解決手順
   - ワークアラウンド: 暫定的な回避策
   - 対応結果        : 最終的な結果
   - 顧客への回答内容: 顧客に伝えた内容
4. 「保存」をクリック


------------------------------------------------------
● トラブルのステータスを変更するとき
------------------------------------------------------

1. 「トラブル対応履歴」で対象トラブルの「編集」ボタンをクリック
2. ステータスを変更（受付 → 調査中 → 対応中 → 解決 → クローズ）
3. 解決した場合は「対応完了日時」も入力
4. 使用した解決パターンがあれば「適用パターン」ドロップダウンから選択
5. 「更新」をクリック

※ ステータスを「解決」または「クローズ」にすると、対応時間が自動計算されます。
※ また、解決パターンとして登録するか確認するダイアログが表示されます。


------------------------------------------------------
● 解決パターンを登録するとき
------------------------------------------------------

トラブル解決後のダイアログから登録する方法と、手動で登録する方法があります。

【手動で登録する場合】
1. サイドバーから「解決パターン」を開く
2. 「新規登録」をクリック
3. 以下を入力
   - パターン名        : わかりやすいタイトル
   - カテゴリ          : 該当するカテゴリ
   - 症状・現象        : 対象となる症状のキーワード（カンマ区切り）
   - 想定原因          : よくある原因
   - 解決手順          : ステップバイステップで記載
   - 所要時間目安      : 対応にかかる目安時間（分）
   - 必要な権限・ツール: 管理者権限、特定のツールなど
   - 注意事項          : 対応時に注意すべきこと
4. 「登録」をクリック

※ 登録したパターンは、トラブル検索時の類似検索で
   自動的にマッチング候補として表示されます。


------------------------------------------------------
● エスカレーションするとき
------------------------------------------------------

1. サイドバーから「エスカレーション」を開く
2. 「新規登録」をクリック
3. 以下を入力
   - トラブルID          : ドロップダウンから対象トラブルを選択
   - エスカレーション日時: 引き継ぎの日時
   - エスカレーション元  : 自分の名前
   - エスカレーション先  : 引き継ぎ先の名前
   - 理由                : なぜエスカレーションが必要か
   - 依頼内容            : 引き継ぎ先にお願いしたいこと
4. 「登録」をクリック

※ 登録すると、エスカレーション先に自動で通知メールが送信されます。

【対応結果を記録する場合】
  一覧の「編集」ボタンから、対応内容・完了日時・結果を入力できます。


------------------------------------------------------
● 再発防止策を登録するとき
------------------------------------------------------

1. サイドバーから「再発防止策」を開く
2. 「新規追加」をクリック
3. 以下を入力
   - トラブルID    : ドロップダウンから対象トラブルを選択
                     （概要が自動入力されます）
   - 根本原因      : トラブルの根本的な原因
   - 再発防止策内容: 具体的な対策
   - 対策種別      : 恒久対応・運用改善・教育・システム改修から選択
   - 担当者        : 対策を実行する担当者
   - 期限          : 完了予定日
4. 「登録」をクリック

【進捗を更新する場合】
  一覧の「編集」ボタンからステータスを変更（未着手 → 進行中 → 完了）
  完了時は「完了日」と「効果確認」を記入してください。

【フィルター】
  上部のドロップダウンでステータスを絞り込めます。


======================================================
■ ダッシュボードの見方
======================================================

ダッシュボードでは、チーム全体の対応状況を確認できます。

【上部のKPIカード（7つ）】
  今日の件数、今月の件数、未解決件数、平均対応時間などの
  重要指標が表示されます。

【アラート欄】
  赤色 : 緊急・高優先の未解決トラブル（すぐに対応が必要）
  黄色 : 期限が超過している再発防止策
  青色 : 監視・対策中の再発トラブル

【グラフ（4つ）】
  - 月別トラブル件数と平均対応時間の推移
  - カテゴリ別の件数ランキング
  - 緊急度別の未解決トラブル分布

【テーブル】
  - 今日のトラブル一覧        : 今日発生したトラブルの状況
  - 解決パターンTOP10         : よく使われているパターン
  - 担当者別対応件数          : 今月の担当者ごとの件数
  - 検索ヒットしなかったキーワード : ナレッジ不足の領域を示します


======================================================
■ 分析ページの見方
======================================================

【月次分析サマリー】
  月ごとの傾向を確認できます。
  手動で最新データを生成したい場合は「月次分析実行」ボタンを
  クリックしてください。

【カテゴリ別分析】
  カテゴリごとの発生件数・平均対応時間・再発率・主な原因TOP3・
  パターン適用率が確認できます。
  「パターン登録推奨」と表示されている場合、そのカテゴリの
  トラブルが多いのに解決パターンが未登録であることを意味します。

【担当者別パフォーマンス】
  担当者ごとの対応件数・平均対応時間・目標内完了率・
  パターン活用率が確認できます。

【再発トラブル追跡】
  再発が検出されたトラブルの一覧です。
  再発回数が3回以上の場合は赤太字で表示されます。
  「編集」ボタンから防止策の実施状況・要因分析・
  対応ステータスを更新できます。


======================================================
■ 自動で行われること
======================================================

以下はシステムが自動で処理するため、手動操作は不要です。

  自動処理                      タイミング
  ─────────────────────────────────────────────────
  タイムラインへの記録          トラブル登録時、ステータス変更時
  対応時間の計算                対応完了日時を入力したとき
  再発トラブルの検出            新しいトラブル登録時に類似トラブルをチェック
  カテゴリ件数の更新            トラブル登録時
  検索ログの記録                トラブル検索実行時
  未解決アラートメール          毎朝8時に自動送信
  週次レポートメール            毎週月曜に自動送信
  月次分析データ生成            毎月1日に自動実行
  エスカレーション通知メール    エスカレーション登録時


======================================================
■ テーブルの操作
======================================================

  - 横スクロール  : テーブルの列が多い場合、左右にスクロールして
                    全項目を確認できます

  - 残りX件を表示 : データが多い場合、下部のボタンをクリックすると
                    全件表示されます

  - 並び順        : 各テーブルは新しいデータが上に表示されます


======================================================
■ 困ったときは
======================================================

  状況                          対処法
  ─────────────────────────────────────────────────
  データが表示されない          ページを再読み込み（F5キー）してください

  分析ページが空                「月次分析サマリー」画面の
                                「月次分析実行」ボタンをクリックしてください

  登録ボタンが反応しない        必須項目が未入力の可能性があります
                                入力内容を確認してください

  スマホでメニューが出ない      左上のハンバーガーメニュー（≡）を
                                タップしてください
