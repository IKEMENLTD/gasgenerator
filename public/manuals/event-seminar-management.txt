====================================================
イベント・セミナー集客管理 使い方説明書
====================================================

----------------------------------------------------
■ このシステムでできること
----------------------------------------------------

イベントやセミナーの申込受付から事後フォローまでを一元管理するシステムです。

・セミナー・説明会・体験レッスン・ウェビナーなどの申込管理
・申込確認→リマインド→当日案内→事後フォローのメール全自動送信
・リアルタイム集客ダッシュボードで複数イベントの状況を一目で把握
・SNS・メルマガ・紹介など申込経路ごとの集客効果を自動計測
・満席時のキャンセル待ち受付と自動繰り上げ
・アンケート回収と参加者の温度感に応じたフォロー自動出し分け
・参加→商談→成約の転換率・ROI追跡
・月次レポートの自動作成・メール送信

こんな方におすすめ:
・コンサルタント・講師（セミナー集客→受注）
・スクール・教育機関（説明会→受講転換）
・営業・マーケティング担当（イベント→商談獲得）
・イベント運営スタッフ（受付・出欠・アンケート効率化）
・マネージャー・経営者（集客ROIの一元把握）


----------------------------------------------------
■ 画面構成（10タブ）
----------------------------------------------------

1. ダッシュボード
2. イベント管理
3. 申込管理
4. 出欠管理
5. メール管理（テンプレート / 配信ログ）
6. アンケート
7. 商談・転換
8. レポート（イベント別 / 経路別 / 月次 / フォロー状況 / キャンセル分析）
9. マスタ管理（申込経路 / 担当者）
10. 設定


====================================================
1. ダッシュボード
====================================================

システムを開くと最初に表示される画面です。
集客状況の全体像をリアルタイムで確認できます。

【KPI表示】
・今月転換金額: 今月の商談・転換で発生した金額合計
・今月総申込数: 今月の有効申込件数
・今月参加数: 今月の出欠記録「参加」の件数
・今月転換数: 今月の商談・転換記録の件数
・フォロー未対応: フォローアップ対象なのに未対応の件数
・本日送信予定メール: 本日送信予定のメール件数と種別内訳
・次回メール送信予定: 本日残りのメール送信スケジュール

【グラフ（5種類）】
・申込経路別集客数: 今月の経路ごとの申込数（横棒グラフ）
・月別申込・転換推移: 直近12ヶ月の申込数と転換数（折れ線グラフ）
・平均満足度推移: 直近6イベントの平均満足度（折れ線グラフ）
・キャンセル率推移: 直近6イベントのキャンセル率（棒グラフ）
・アンケート満足度分布: 最新イベントの1〜5点分布（ドーナツグラフ）

【テーブル】
・開催予定イベント一覧: イベント名・開催日・申込数・残席・充足率・ステータス
・転換率ランキング: イベント別の転換数・転換率TOP5
・フォロー未対応: 担当者別の未対応件数


====================================================
2. イベント管理
====================================================

イベント・セミナーの基本情報を登録・管理します。

【登録手順】
1.「+ 新規登録」ボタンをクリック
2. 以下の項目を入力:
   ・イベント名（必須）
   ・イベント種別: セミナー/説明会/体験レッスン/ウェビナー/ワークショップ/懇親会/その他
   ・開催日・開催時間（開始・終了）
   ・開催形式: オフライン/オンライン/ハイブリッド
   ・会場名 / オンラインURL
   ・会場住所（オフラインの場合）
   ・定員（名）
   ・参加費（円）
   ・担当者（マスタ管理で登録した担当者から選択）
   ・申込受付開始日・申込締切日
   ・ステータス: 準備中/受付中/満席/受付終了/開催済/中止
   ・集客目標数（名）
   ・転換目標率（%）
   ・概要・詳細 / 備考
3.「保存」ボタンで登録

【ステータスの意味】
・準備中: イベント準備段階、申込受付前
・受付中: 申込受付中
・満席: 定員に達した（申込時に自動変更される場合あり）
・受付終了: 申込締切後
・開催済: イベント終了後
・中止: 開催中止

【編集・削除】
・一覧の「編集」ボタンで内容を修正
・「削除」ボタンで確認後に削除


====================================================
3. 申込管理
====================================================

イベントへの参加申込を登録・管理します。

【登録手順】
1.「+ 新規登録」ボタンをクリック
2. 以下の項目を入力:
   ・イベント（登録済みイベントから選択）
   ・参加者名（必須）
   ・メールアドレス
   ・電話番号
   ・会社名 / 所属
   ・役職
   ・申込経路（マスタ管理で登録した経路から選択）
   ・経路詳細（例: Instagram投稿URL、紹介者名など）
   ・参加人数
   ・ステータス
   ・取込元: 手動入力 / Googleフォーム自動
   ・備考
3.「保存」ボタンで登録

【ステータスの意味】
・申込済: 申込受付完了
・確認済: 申込内容を担当者が確認済み
・参加確定: 参加が確定
・キャンセル待ち: 定員超過のため待機中（自動判定）
・キャンセル: キャンセル済み
・欠席: イベント当日に不参加
・参加済: イベントに参加完了

【自動処理】
・申込時に定員を超えた場合、自動的に「キャンセル待ち」になります
・満席になるとイベントステータスが「満席」に自動更新されます
・申込確認メールが自動送信されます（テンプレート登録時）

【キャンセル待ち繰り上げ】
・ステータスを「キャンセル」に変更すると、同イベントのキャンセル待ち1番目の方に自動で繰り上げ案内メールが送信されます
・「キャンセル待ち繰り上げ実行」ボタンで、全イベントの繰り上げを手動実行することもできます

【フィルター機能】
・イベント名やステータスで一覧を絞り込めます
・キーワード検索で参加者名・メールアドレスなどから検索できます


====================================================
4. 出欠管理
====================================================

イベント当日の出欠を記録します。

【登録手順】
1.「+ 新規登録」ボタンをクリック
2. イベントを選択 → 参加者名が自動でリスト表示されます
3. 参加者を選択するとメールアドレスが自動入力されます
4. 以下の項目を入力:
   ・出欠区分: 参加/欠席/遅刻/早退
   ・チェックイン時刻
   ・チェックアウト時刻
   ・滞在時間（分）
   ・受付担当者
   ・備考
5.「保存」ボタンで登録


====================================================
5. メール管理
====================================================

自動送信メールのテンプレートと配信履歴を管理します。

---------- 5-1. テンプレート ----------

メール自動送信の内容を設定します。

【登録手順】
1.「+ 新規登録」ボタンをクリック
2. 以下の項目を入力:
   ・テンプレート名（必須）
   ・送信タイミング: 申込直後/3日前/前日/当日朝/翌日/3日後/1週間後
   ・送信対象: 全参加者/参加者のみ/欠席者/キャンセル者/アンケート未回答者
   ・有効フラグ: TRUE（有効）/ FALSE（無効）
   ・件名テンプレート
   ・本文テンプレート
   ・備考
3.「保存」ボタンで登録

【プレースホルダー（自動置換）】
件名・本文に以下を記入すると、送信時に自動で置換されます:
   {{イベント名}}    → イベント名
   {{参加者名}}      → 参加者の氏名
   {{開催日時}}      → 開催日 + 時間
   {{開催日}}        → 開催日のみ
   {{会場}}          → 会場名 / オンラインURL
   {{会場住所}}      → 会場住所
   {{Zoom URL}}      → オンライン/ハイブリッド開催時のURL
   {{申込ID}}        → 申込ID
   {{メールアドレス}} → 参加者のメールアドレス

---------- 5-2. 配信ログ ----------

送信されたメールの履歴を確認できます（閲覧専用）。

表示項目:
・配信ログID、メール種別、送信日時、宛先、送信結果、失敗理由

メール種別の一覧:
・申込確認: 申込時の確認メール
・3日前リマインド: 開催3日前のリマインド
・前日リマインド: 開催前日のリマインド
・当日案内: 開催当日朝の案内
・事後フォロー: 参加者への事後フォロー＋アンケート依頼
・キャンセル待ち繰り上げ: キャンセル発生時の繰り上げ案内


====================================================
6. アンケート
====================================================

イベント参加者のアンケート回答を登録・管理します。

【登録手順】
1.「+ 新規登録」ボタンをクリック
2. イベントを選択 → 参加者名が自動でリスト表示されます
3. 参加者を選択するとメールアドレスが自動入力されます
4. 以下の項目を入力:
   ・満足度（1〜5）
   ・内容評価（1〜5）
   ・今後の関心度: 高/中/低
   ・検討状況: すぐ検討したい/3ヶ月以内/半年以内/未定/不要
   ・個別相談希望: あり/なし
   ・次回参加希望: あり/なし/未定
   ・良かった点（自由記述）
   ・改善点（自由記述）
5.「保存」ボタンで登録

【自動処理】
・関心度「高」、検討状況「すぐ検討したい」「3ヶ月以内」、個別相談「あり」のいずれかに該当する回答者には、フォローアップフラグが自動で付与されます
・フォローアップフラグがTRUEの参加者には、3日後に温度感に応じたフォローメールが自動送信されます:
  - 関心度「高」＋個別相談「あり」→ 個別商談案内メール
  - 関心度「高」→ 詳細資料案内メール
  - その他 → 次回セミナー案内メール


====================================================
7. 商談・転換
====================================================

イベント参加者の商談・成約への転換を記録します。

【登録手順】
1.「+ 新規登録」ボタンをクリック
2. イベントを選択 → 参加者名が自動でリスト表示されます
3. 参加者を選択
4. 以下の項目を入力:
   ・転換日
   ・転換種別: 個別相談申込/体験レッスン申込/資料請求/正式契約/その他
   ・転換金額（円）
   ・担当者
   ・備考
5.「保存」ボタンで登録

この記録がダッシュボードの転換率・転換金額・ROIに反映されます。


====================================================
8. レポート
====================================================

各種集計データを5つのサブタブで確認できます（閲覧専用）。
データは毎日深夜2:00に自動更新されます。
設定画面の「集計更新」ボタンで手動更新も可能です。

---------- 8-1. イベント別サマリー ----------
イベントごとの成果を一覧表示:
・定員・申込数・充足率
・キャンセル数・キャンセル率・キャンセル待ち数
・実参加数・出席率
・アンケート回収数・回収率・平均満足度
・フォローアップ対象数
・転換数・転換率・転換金額・集客費用・ROI

---------- 8-2. 経路別効果 ----------
申込経路（Instagram/メルマガ/紹介など）ごとの集客効果:
・申込数・参加数・参加率
・転換数・転換率・転換金額
・費用・CPA（顧客獲得単価）・ROI

---------- 8-3. 月次サマリー ----------
月ごとの実績推移:
・開催イベント数・総申込数・総参加数・平均出席率
・総転換数・平均転換率・転換金額・集客費用・ROI
・平均満足度・アンケート回収率

---------- 8-4. フォロー状況 ----------
参加者ごとのフォローアップ進捗:
・出欠区分・関心度・検討状況・個別相談希望
・フォローアップフラグ・事後フォローメール送信済・転換済

---------- 8-5. キャンセル分析 ----------
イベントごとのキャンセル傾向:
・キャンセル数・キャンセル率
・タイミング別内訳（1週間以上前/3〜6日前/前日〜当日）
・繰り上げ成功数・繰り上げ率


====================================================
9. マスタ管理
====================================================

---------- 9-1. 申込経路 ----------
申込経路の種類を登録します。

登録項目:
・経路名（例: Instagram / メルマガ / 紹介 / 自社HP / Connpass / Peatix）
・経路種別: SNS/メールマガジン/紹介/広告/SEO/イベントサイト/その他
・費用（円）: その経路にかけた広告費用（無料の場合は0）
・備考

---------- 9-2. 担当者 ----------
イベント担当者を登録します。

登録項目:
・氏名（必須）
・メールアドレス
・部署・役割
・ステータス: 在籍中/休職/退職


====================================================
10. 設定
====================================================

メール送信のタイミングや通知先を設定できます。

【設定項目一覧】
・申込確認メール送信: 即時（申込から何分以内に送信するか）
・リマインドメール_第1回: 3（開催N日前に送信）
・リマインドメール_第2回: 1（開催N日前に送信）
・当日案内メール送信時刻: 8:00（当日朝の送信時刻）
・事後フォローメール送信: 1（開催翌N日に送信）
・アンケート依頼メール送信: 1（開催翌N日に送信）
・フォロー対象アンケート関心度: 高（この関心度以上を優先フォロー対象とする）
・通知先メールアドレス（管理者）: 管理者通知先
・満席通知送信先: 満席時の通知先
・フォーム連携URL: Googleフォーム連携URL

設定値を変更後「設定を保存」ボタンで反映します。

【運用ボタン】
・集計更新: レポートデータを即時再計算
・ダッシュボード更新: ダッシュボードの集計値を即時更新
・月次レポート送信: 月次レポートメールを管理者・担当者全員に送信
・リマインドメール送信: リマインドメールを即時送信
・全集計一括実行: 全レポートを一括再集計

【GAS処理履歴】
自動処理の実行ログが表示されます:
・処理名・実行日時・処理件数・メール送信件数・処理結果・エラー内容


====================================================
■ 自動処理スケジュール
====================================================

以下の処理がトリガーで自動実行されます:

毎日 2:00  集計更新（イベント別/経路別/キャンセル分析/フォロー状況）
毎日 3:00  ダッシュボード更新
毎日 7:00  リマインドメール送信（3日前・前日）
毎日 8:00  当日案内メール送信
毎日 10:00 事後フォローメール送信（参加者＋欠席者）
毎日 10:30 関心度別フォローアップメール送信
毎日 11:00 アンケート取込・フォローフラグ更新＋対象者通知
毎月1日 3:00  月次サマリー集計
毎月1日 9:00  月次レポートメール送信

※スプレッドシート上でステータスを直接「キャンセル」に変更した場合も、
  キャンセル待ち繰り上げが自動で実行されます。


====================================================
■ メール自動送信の流れ
====================================================

1. 申込時 → 申込確認メール（即時）
2. 定員超過時 → キャンセル待ち（メール送信なし）
3. キャンセル発生時 → キャンセル待ち1番目に繰り上げ案内メール
4. 開催3日前 → 3日前リマインドメール
5. 開催前日 → 前日リマインドメール
6. 開催当日朝 → 当日案内メール（会場/Zoom URL含む）
7. 開催翌日 → 事後フォローメール＋アンケートURL（参加者向け）
              資料案内メール（欠席者向け）
8. アンケート回答3日後 → 温度感別フォローメール
9. 毎月1日 → 月次レポートメール（管理者・担当者全員）

※テンプレートが登録されている場合はテンプレートの内容で送信されます。
  未登録の場合はシステムのデフォルト文面で送信されます。


====================================================
■ よくある使い方の流れ
====================================================

【イベント開催前】
1. マスタ管理で申込経路・担当者を登録
2. イベント管理でイベントを登録（ステータス: 受付中）
3. メール管理でメールテンプレートを登録
4. 申込が入ったら申込管理に登録（→確認メール自動送信）

【イベント当日】
5. 出欠管理で参加者のチェックイン・チェックアウトを記録

【イベント後】
6. アンケートで参加者の回答を登録
7. 商談・転換で成果を記録
8. レポートで結果を確認
9. ダッシュボードで全体の数値を把握

【定期確認】
・ダッシュボードで集客状況を毎日確認
・フォロー未対応件数をチェックし、対応を進める
・月次レポートで前月の成果を振り返る
