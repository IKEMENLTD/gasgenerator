=====================================
LINE流入トラッキング＆代理店報酬管理システム
使い方説明書
=====================================

■ このシステムでできること
────────────────────────────
・代理店ごとにユニークな紹介リンクを発行し、LINE友だち追加を自動追跡
・どの代理店経由で何人がLINE登録したかをリアルタイムで集計
・スプレッドシートの案件データとmatch_code（紹介コード）で突合し、報酬を自動計算
・案件・単価・報酬率の設定、認証情報の管理をすべて管理画面で完結
・スプシが横持ち（1行1人）でも縦持ち（1行1案件）でも対応

■ 活用シーン
────────────────────────────
・代理店紹介制度のLINE流入管理（Web制作会社、不動産、保険など）
・アフィリエイト報酬の自動計算
・紹介キャンペーンの効果測定
・複数代理店の成果を一元管理


■ 初回ログイン（管理者として利用開始する方法）
────────────────────────────
1. 管理画面URL（https://line-referral-tracker.netlify.app）にアクセス
2. 「新規登録」をクリック
3. 会社名・担当者名・メールアドレス・パスワードを入力
4. 「管理者コード」欄に以下のコードを入力：

   管理者コード: LRT-ADMIN-2026

5. 「登録する」をクリック
6. 承認待ちなしで、すぐにダッシュボード（管理者メニュー付き）が表示されます

※ 管理者コードは何回でも使用できます。複数人を管理者として登録できます。
※ 管理者コードを入力せずに登録した場合は、通常の代理店アカウント（承認待ち）になります。
※ 管理者権限は後から「代理店管理」画面でも付与・解除できます。


=====================================
代理店向け画面
=====================================

■ ダッシュボード
────────────────────────────
自分の代理店の成果をひと目で確認できるトップ画面です。

【表示内容】
・発行リンク数 … 自分が発行した紹介リンクの総数
・ユニーククリック … 重複を除いたクリック数（1時間以内の同一ユーザーは1回とカウント）
・LINE登録者 … 自分のリンク経由でLINE登録した人数
・報酬合計 … 現在の報酬合計額
・最新LINE登録者 … 直近5名の登録者一覧（アイコン・突合コード・友だち/ブロック状態）

ページを開くたびに最新データが表示されます。


■ リンク管理
────────────────────────────
紹介リンクの発行・管理を行います。

【リンク発行】
1. 「+ 新規発行」ボタンをクリック
2. ラベル（リンクの識別名）と説明を入力
3. 「発行」ボタンで紹介リンクが自動生成されます

【リンク一覧】
・ラベル … リンクの識別名
・リンクコード … 自動生成された一意のコード
・クリック数 … ユニーククリック数
・状態 … 有効/無効
・操作 … コピー（リンクURLをクリップボードにコピー）/ 有効・無効切替

【リンクの使い方】
生成されたURL（例: https://line-referral-tracker.netlify.app/r/xxxxxxxx）を
代理店が顧客に共有します。顧客がこのURLをタップすると：
1. クリックが自動記録される
2. LINEアプリが開き、友だち追加画面が表示される
3. 紹介コード入力画面が表示される
4. 登録完了

無効化したリンクをクリックすると「このリンクは無効化されています」と表示されます。


■ リンク詳細
────────────────────────────
リンク管理でリンク名をクリックすると詳細画面が開きます。

【表示内容】
・リンクURL（コピーボタン付き）
・ユニーククリック数・LINE登録者数・作成日
・LINE登録者一覧 … アイコン・名前・突合コード・友だち/ブロック状態・登録日
・クリックログ（最新50件）… 日時・ユニーク判定・リファラー


■ 報酬確認
────────────────────────────
自分の報酬を確認する画面です。

【表示内容】
・サマリー … 報酬合計金額（¥マーク付き）
・詳細テーブル … 紹介者名・案件ごとの売上と報酬・合計報酬
・月別フィルタ … 年月を選択すると該当月のデータのみ表示

【報酬計算の仕組み】
・パーセンテージ方式 … 売上 × 報酬率（例: 500,000円 × 10% = 50,000円）
・固定金額方式 … 売上があれば固定額（例: 成約1件につき30,000円）


■ プロフィール
────────────────────────────
アカウント情報の編集画面です。

【基本情報】
・会社名・担当者名・電話番号を編集できます
・代理店コード（自動生成、変更不可）が表示されます

【メールアドレス変更】
・現在のメールアドレスが表示されます
・新しいメールアドレスを入力して「変更リクエスト」を押すと確認メールが届きます
・メール内のリンクをクリックすると変更が確定します

【パスワード変更】
・新しいパスワード（6文字以上）を入力して変更できます


=====================================
管理者向け画面
=====================================

■ 代理店管理
────────────────────────────
代理店の承認・管理を行います。

【承認待ち】
・新規登録された代理店が黄色のハイライトで表示されます
・「承認」ボタンで代理店を有効化（ダッシュボードにアクセス可能に）
・「却下」ボタンで登録を拒否

【全代理店一覧】
・各代理店をクリックすると詳細が展開されます
・表示内容: 会社名・担当者名・メール・電話・登録日・承認日・管理者権限
・「無効化」… 代理店のアクセスを停止
・「再有効化」… 無効化した代理店を再度有効化
・「管理者権限を付与」… 代理店に管理者メニューへのアクセスを許可
・「管理者権限を解除」… 管理者権限を削除（管理者が1人の場合は解除不可）

【管理者の引き渡し手順】
1. 新しい管理者がシステムに新規登録
2. 現在の管理者が代理店管理で承認
3. 「管理者権限を付与」ボタンをクリック
4. 新しい管理者がログインすると管理者メニューが表示される


■ 案件管理
────────────────────────────
報酬計算の対象となる案件を登録・編集します。

【案件追加】
・案件名（必須）… 例: ホームページ制作、LINE構築
・単価 … 案件の基本単価（参考値）
・対象列名（横持ちモード時）… スプシのどの列を売上として参照するか（例: HP売上）
・対象値（縦持ちモード時）… スプシの案件名列の値（例: ホームページ制作）
・説明 … 案件の補足情報

【注意事項】
・横持ちモードでは「対象列名」がスプシの列名と完全一致する必要があります
・同じ対象列名を複数案件で使うと、報酬確認画面で列が重複表示されます


■ 報酬ルール
────────────────────────────
代理店×案件ごとの報酬率を設定します。

【設定方法】
・行: 代理店、列: 案件のマトリクス形式で表示
・各セルに報酬率と種別を入力
  - パーセンテージ（%）… 売上に対する割合（例: 10%）
  - 固定金額 … 成約1件あたりの固定額（例: 30,000円）
・値を変更すると自動保存されます


■ スプシ設定
────────────────────────────
Google スプレッドシートとの連携を設定します。

【スプシ構造モード】
・横持ち … 1行=1人。列が案件種別ごとに存在（例: HP売上、LINE売上）
・縦持ち … 1行=1案件。同じ人が複数行に出現

【共通設定】
・Sheet ID … スプレッドシートのID（URLの /d/ と /edit の間の文字列）
・タブ名 … 読み取るシートのタブ名
・ヘッダー行 … ヘッダーがある行番号（通常は1）
・ID列 … 各行のユニークID列名
・突合コード列 … match_codeが入っている列名
・名前列 … 人名が入っている列名
・日付列（任意）… 月別フィルタに使う日付列名

【縦持ちモード追加設定】
・案件名列 … 案件名が入っている列名（例: 案件名）
・金額列 … 金額が入っている列名（例: 金額）

【テスト接続】
・「テスト接続」ボタンでスプシにアクセスできるか確認
・成功するとヘッダー列名一覧とデータ行数が表示されます

【同期】
・「今すぐ同期」ボタンで手動同期
・自動同期: 毎時0分に自動でスプシデータを取り込みます
・同期時にmatch_codeでLINE登録者と自動突合されます

【match_code入力を必須にする】
・チェックを入れると、LIFF画面でmatch_code未入力では登録できなくなります


■ 手動突合
────────────────────────────
自動突合できなかったデータを手動で紐付けます。

【画面構成】
・上段: 未突合LINE登録者（match_codeが未入力またはスプシと一致しない）
・下段: 未突合スプシデータ（LINE登録者と紐付いていない行）

【紐付け方法】
1. 上段からLINE登録者を1人選択（クリック）
2. 下段からスプシデータを1件選択（クリック）
3. 「紐付け」ボタンをクリック


■ データビュー
────────────────────────────
全代理店横断でスプシの取り込みデータを閲覧・エクスポートします。

【表示内容】
・ID・match_code・代理店名・データ（JSON形式）・同期日時
・代理店フィルタ … 全代理店/特定代理店で絞り込み
・データフィルタ … 全データ/突合済み/未突合

【CSVエクスポート】
・「CSV出力」ボタンでデータをCSVファイルとしてダウンロード
・UTF-8 BOM付き（Excelで文字化けしません）


■ 認証情報
────────────────────────────
LINE連携とGoogle連携に必要な認証情報を設定します。

【LINE設定（4項目）】
・LINE Channel ID … LINE Loginチャネルの10桁数字
  ※Messaging APIチャネルではなく、LIFFを作成したLINE LoginチャネルのID
・LINE Channel Secret … Messaging APIチャネルの32文字英数字（Webhook署名検証用）
・LIFF ID … LINE LoginチャネルのLIFFタブで作成したID（数字-英数字の形式）
・LINE Access Token … Messaging APIチャネルで発行した長期アクセストークン

【Google設定（1項目）】
・Google SA Key (JSON) … GCPのサービスアカウントキーのJSON全文を貼り付け
  保存後にclient_emailが表示されるので、スプシに閲覧者として共有追加してください

【Webhook URL（案内パネル）】
・LINE DevelopersのMessaging API設定に貼り付けるURLが表示されます
・コピーボタンでクリップボードにコピーできます

【各項目の操作】
・「設定する」/「再設定」… 値を入力して保存
・「接続テスト」… 設定済みの値が正しいか検証
・取得方法（❓アイコン）… 値の取得手順がインラインヘルプで表示されます


■ 監査ログ
────────────────────────────
管理者の操作履歴を記録・閲覧します。

【表示内容】
・日時・操作者（メールアドレス）・アクション・対象・詳細
・アクションフィルタ … 全アクション/特定アクションで絞り込み

【記録される操作】
・認証情報の設定・テスト・削除
・その他の管理操作

監査ログは削除できません（改ざん防止）。


=====================================
全体の運用フロー
=====================================

■ 初期設定（1回のみ）
────────────────────────────
1. 認証情報画面でLINE・Googleの認証情報を設定
2. スプシ設定で連携先シートを設定・テスト接続
3. 案件管理で案件を登録（対象列名/対象値を設定）
4. 報酬ルールで代理店×案件の報酬率を設定

■ 日常運用
────────────────────────────
1. 代理店が新規登録 → 管理者が承認
2. 代理店がリンクを発行 → 顧客に共有
3. 顧客がリンクをタップ → LINE友だち追加 → match_code入力
4. スプシに案件データを入力（事業者の通常業務）
5. 毎時自動同期でmatch_codeにより自動突合
6. 報酬確認画面で報酬を確認
7. 未突合データは手動突合で対応


=====================================
導入・セットアップガイド
=====================================

すべての設定は管理画面から行えます。コードの編集は不要です。

■ 初期セットアップ手順（全9ステップ）
────────────────────────────

【ステップ1: LINE連携の設定】
1. LINE Developersでチャネルを作成（下記手順参照）
2. 管理画面の「認証情報」で以下4つを設定・接続テスト
   ・LINE Channel ID
   ・LINE Channel Secret
   ・LIFF ID
   ・LINE Access Token

【ステップ2: Google連携の設定】
3. GCPでサービスアカウントを作成し、Sheets APIを有効化
4. 管理画面の「認証情報」でGoogle SA Key (JSON)を設定
5. 表示されたclient_emailをスプレッドシートに「閲覧者」として共有追加

【ステップ3: 業務設定】
6. 「スプシ設定」で連携先シート・モード・列名を設定 → テスト接続で確認
7. 「案件管理」で案件を登録（対象列名/対象値を設定）
8. 「報酬ルール」で代理店×案件の報酬率を設定
9. 「今すぐ同期」でデータ取り込み → 報酬確認画面で動作確認


■ LINE Developers設定手順
────────────────────────────
お客様のLINE公式アカウントに対して、以下の設定が必要です。

【必要なもの】
・LINE公式アカウント（LINE Official Account Manager で作成）
・LINE Developersアカウント（developers.line.biz）

【設定手順】
1. LINE Developersでプロバイダーを作成
2. Messaging APIチャネルを作成（LINE公式アカウントと連携）
   → Channel Secret / チャネルアクセストークン（長期）を取得
3. LINE Loginチャネルを作成
   → Channel ID を取得
4. LINE LoginチャネルでLIFFを追加
   - Endpoint URL: 管理画面の認証情報ページに表示されるURLを使用
   - Size: Full
   - Scopes: profile, openid
   - Bot link feature: Aggressive
   → LIFF IDを取得
5. LINE LoginチャネルにMessaging APIボットをリンク（Linked OA）
6. LINE Loginチャネルを「Published」に変更
7. Messaging API設定 → Webhook URLに管理画面に表示されるURLを設定 → ONにする

※ 各値の取得方法は管理画面の認証情報ページ（❓ボタン）にも記載しています。


=====================================
注意事項
=====================================

・同じ対象列名を複数の案件で使用すると、報酬確認画面で列が重複表示されます。
  案件ごとに異なるスプシ列を使用してください。

・同じLINEユーザーが別の代理店リンクから再登録すると、紐付く代理店が
  最新のリンク元に上書きされます（最後にクリックした代理店に帰属）。

・本番運用ではSupabaseのカスタムSMTP設定（SendGrid等）を推奨します。
  デフォルトのメール送信には1時間あたり4通の制限があります。

・スプシの列名は完全一致で照合されます。
  スプシ側で列名を変更した場合は、案件管理の対象列名も合わせて更新してください。

・スプレッドシートのデータ量について：
  500行単位のバッチ処理を実装済みです。数千行程度は問題なく同期可能ですが、
  1万行を超える場合はEdge Functionのタイムアウト（60秒）に達する可能性があります。
  大量データが見込まれる場合は、シートの分割やフィルタリングをご検討ください。

・セキュリティについて：
  認証情報（LINE Channel Secret・Google SA Keyなど）はデータベース上で
  AES暗号化して保存されています。管理画面からは設定状態のみ表示され、
  値そのものは表示されません。
