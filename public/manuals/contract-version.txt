============================================================
  契約書バージョン管理 - 使い方説明書
============================================================

このシステムは、契約書テンプレートのバージョン管理を行い、
古いテンプレートでの契約締結を防ぎます。
左のメニューから各画面に移動して操作してください。
スマホではハンバーガーボタン（≡）でメニューを開きます。


────────────────────────────────────────
  1. ダッシュボード
────────────────────────────────────────

システムを開くと最初に表示される画面です。
全体の状況をひと目で把握できます。

【統計カード（上部4枚）】
  ・テンプレート総数
  ・有効テンプレート数
  ・見直し中の数
  ・未対応アラート数

【グラフ】
  ・契約種別の分布（ドーナツグラフ）
  ・法改正対応状況（棒グラフ）

【一覧テーブル】
  ・旧版使用アラート … 古いバージョンで締結された契約
  ・見直し期限到来 … 見直し期限が近いテンプレート
  ・契約書テンプレート一覧 … 全テンプレートの概要
  ・最近の改訂履歴 … 直近の改訂内容
  ・テンプレート利用状況 … ダウンロード数の集計


────────────────────────────────────────
  2. テンプレートマスタ
────────────────────────────────────────

契約書テンプレートの基本情報を管理します。

【テンプレートを登録する】
  1. 「新規登録」ボタンを押す
  2. 契約書名（必須）、契約種別、カテゴリなどを入力
  3. 「保存」を押す

【テンプレートを編集する】
  1. 対象行の「編集」ボタンを押す
  2. 内容を修正して「保存」を押す

【テンプレートを削除する】
  1. 対象行の「削除」ボタンを押す
  2. 確認ダイアログで「OK」を押す

※ステータスは「有効」「廃止」「見直し中」から選択します。
※次回見直し予定日を設定すると、期限前にリマインドが届きます。


────────────────────────────────────────
  3. バージョン履歴
────────────────────────────────────────

テンプレートの改訂履歴を管理します。
改訂を登録すると承認フローが自動で開始されます。

【改訂を登録する】
  1. 「改訂登録」ボタンを押す
  2. 契約書名をドロップダウンから選択（必須）
  3. バージョン番号（必須）、改訂日、改訂理由などを入力
  4. 「保存」を押す
  → 承認フロー設定に基づき、承認者へメール通知されます
  → ステータスは「承認待ち」で登録されます

【改訂を承認する】
  1. ステータスが「承認待ち」のレコードに「承認」ボタンが表示されます
  2. 「承認」ボタンを押す → 承認者名と承認日を入力
  3. 承認されると以下が自動で実行されます：
     ・同テンプレートの旧バージョンが「旧版」ステータスに変更
     ・テンプレートマスタの現行バージョンが更新
     ・関連部署へ改訂通知メールが送信


────────────────────────────────────────
  4. 変更詳細ログ
────────────────────────────────────────

改訂でどの条項がどう変わったかを記録します。

【変更を記録する】
  1. 「変更追加」ボタンを押す
  2. 契約書名をドロップダウンから選択（必須）
  3. 履歴IDを入力（必須）… バージョン履歴のIDを指定
  4. 変更箇所（条項番号）、変更前文言、変更後文言を入力
  5. 変更種別（追加/削除/修正/移動）とリスク影響度を選択
  6. 「保存」を押す

【変更を編集する】
  1. 対象行の「編集」ボタンを押す
  2. 内容を修正して「保存」を押す

※リスク影響度は「高」「中」「低」の3段階です。
  高リスクの変更は特に注意して確認してください。


────────────────────────────────────────
  5. 契約締結記録
────────────────────────────────────────

実際の契約締結をテンプレートと紐づけて記録します。
登録時にバージョンチェックが自動実行されます。

【契約を登録する】
  1. 「新規登録」ボタンを押す
  2. 契約書名をドロップダウンから選択（必須）
     → 使用バージョンが自動入力されます
  3. 契約相手先、締結日、開始日、終了日などを入力
  4. 「保存」を押す
  → 使用バージョンが最新でない場合：
     ・バージョンチェック結果が「旧版使用」になります
     ・旧版使用アラートが自動で発行されます
     ・法務部門へ通知メールが送信されます

【契約を編集する】
  1. 対象行の「編集」ボタンを押す
  2. 内容を修正して「保存」を押す


────────────────────────────────────────
  6. 旧版使用アラート
────────────────────────────────────────

旧バージョンのテンプレートで締結された契約のアラートです。
毎朝のバージョンチェックでも自動検出されます。

【アラートに対応する】
  1. 対象行の「対応」ボタンを押す
  2. 対応ステータスを選択（確認中/承認済/差替済）
  3. 対応者と対応内容を入力
  4. 「保存」を押す

※ステータスの意味：
  ・未対応 … まだ確認していない
  ・確認中 … 確認作業中
  ・承認済 … 旧版使用を承認した（問題なし）
  ・差替済 … 最新版で再締結した


────────────────────────────────────────
  7. 改訂通知履歴
────────────────────────────────────────

テンプレートが改訂された際に送信された通知の履歴です。
この画面は閲覧専用です。

通知は以下のタイミングで自動送信されます：
  ・テンプレートの改訂が承認されたとき（改訂通知）
  ・見直し期限が近づいたとき（周知リマインド）
  ・テンプレートが廃止されたとき（廃止通知）


────────────────────────────────────────
  8. 閲覧・DLログ
────────────────────────────────────────

テンプレートの閲覧・ダウンロードの記録です。
最新版でないバージョンを使おうとした場合は警告が表示されます。

【ログを記録する】
  1. 「ログ追加」ボタンを押す
  2. 契約書名をドロップダウンから選択（必須）
     → バージョンが自動入力されます
  3. アクション（閲覧/ダウンロード/コピー作成）を選択
  4. 実行者（必須）と部署を入力
  5. 「保存」を押す
  → 最新版でない場合、旧版警告ダイアログが表示されます


────────────────────────────────────────
  9. 見直しスケジュール
────────────────────────────────────────

テンプレートの定期見直しを管理します。
期限30日前に担当者へリマインドメールが届きます。

【スケジュールを登録する】
  1. 「新規登録」ボタンを押す
  2. 契約書名をドロップダウンから選択（必須）
  3. 見直し周期（1年/2年/法改正時など）を入力
  4. 担当者、前回見直し日、次回見直し予定日を入力
  5. ステータスを選択（予定/進行中/完了/延期）
  6. 「保存」を押す

※リマインドは1度だけ送信されます（送信済フラグで管理）。
  次回の見直し時は新しいスケジュールを登録してください。


────────────────────────────────────────
  10. 関連法令マスタ
────────────────────────────────────────

契約書に関係する法令・規制を管理します。
新しい法令を登録すると、関連テンプレートが自動的に
「見直し中」ステータスに変わります。

【法令を登録する】
  1. 「新規登録」ボタンを押す
  2. 法令名（必須）を入力
  3. 関連テンプレートID（カンマ区切りで複数指定可）を入力
  4. 最終改正日、施行日、影響条項を入力
  5. 対応ステータスを選択（対応不要/要確認/対応中/対応済）
  6. 「保存」を押す
  → 関連テンプレートIDに指定されたテンプレートが
    自動的に「見直し中」になります


────────────────────────────────────────
  11. 承認フロー設定
────────────────────────────────────────

テンプレートの改訂時に誰が承認するかを設定します。

【承認フローを登録する】
  1. 「新規登録」ボタンを押す
  2. 対象（新規作成/改訂/廃止）を入力
  3. ステップ1〜3の承認者を入力
  4. 条件（重要度高の場合のみ役員承認など）を入力
  5. 「保存」を押す

※バージョン履歴で改訂を登録すると、
  対象が一致するフローの承認者がセットされます。


────────────────────────────────────────
  自動処理について
────────────────────────────────────────

以下の処理はシステムが自動で実行します。
手動操作は不要です。

【毎日朝（自動）】
  ・全契約のバージョンチェック
    → 旧版使用があればアラート発行＋法務へ通知
  ・見直し期限のリマインド
    → 期限30日前の担当者へメール送信

【改訂承認時（自動）】
  ・旧バージョンのステータスを「旧版」に変更
  ・テンプレートマスタの現行バージョンを更新
  ・関連部署へ改訂通知メールを送信
  ・改訂通知履歴に記録

【契約締結登録時（自動）】
  ・使用バージョンと最新版を照合
  ・旧版使用の場合、アラート発行＋法務へ通知

【法令登録時（自動）】
  ・関連テンプレートを「見直し中」に変更

【毎月1日（自動）】
  ・月次バージョン管理レポートを生成


────────────────────────────────────────
  画面の共通操作
────────────────────────────────────────

【テーブルの操作】
  ・データが多い場合、下部にページ切替ボタンが表示されます
  ・スマホではテーブルを横にスワイプできます
  ・タイトルとボタンはスクロールしても固定されます

【フォームの操作】
  ・「*」マークは必須項目です
  ・×ボタンまたは背景をタップで閉じます
  ・スマホでは入力後、保存ボタンを押す前に
    一度他の場所をタップしてください

【削除について】
  ・削除は取り消せません。確認ダイアログで内容を確認してから
    OKを押してください
