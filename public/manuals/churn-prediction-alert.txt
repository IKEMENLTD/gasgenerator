========================================
サブスク解約予兆アラート 使い方説明書
========================================

■ 概要
SaaS/サブスクリプションサービスの顧客がどのくらい解約しそうかを
予兆スコアで数値化し、リスクの高い顧客をCSM（カスタマーサクセスマネージャー）に
自動でアラート通知するシステムです。
フォロー対応の記録・効果検証まで一気通貫で管理できます。

■ こんなことに使えます
- 顧客のログイン頻度や利用状況から解約リスクを自動スコアリング
- スコアが高リスクになった顧客をCSM担当者にメールで即時通知
- 契約更新の60日前/30日前にリマインダーを自動送信
- CSMごとのフォロー実施率・解約防止成功率をレポートで比較
- どのフォローアクション（架電/メール/商談など）が効果的かを数値で検証
- 月別チャーンレート推移やスコア分布をダッシュボードで一目確認
- 未対応アラートのリマインダーを毎朝自動送信


========================================
画面構成（全11タブ）
========================================

管理画面は左サイドバーのメニューから各画面に切り替えます。

【1. ダッシュボード】
システム全体の状況を一覧で把握できるメイン画面です。

表示される指標:
- 契約中顧客数 / 合計MRR（月次経常収益）
- リスク別顧客数（高/中/低）
- 未対応アラート件数
- 解約防止成功率
- 今月解約件数 / 解約MRR / チャーンレート

グラフ:
- リスクランク分布（円グラフ）
- スコア分布（棒グラフ: 0-20/21-40/41-60/61-80/81-100の5段階）
- 月別チャーンレート推移（折れ線グラフ）
- 平均予兆スコア推移 30日（折れ線グラフ）
- 月別アラート発生・対応完了件数（棒グラフ）

テーブル:
- 高リスク顧客 Top10（スコア順）
- 未対応アラート数（CSM別）
- アクション効果ランキング（成功率順）
- CSM別パフォーマンス比較

操作ボタン:
- 全集計実行: 全ての集計処理を一括実行
- アラート検知: アラート検知処理を手動実行
- ダッシュボード更新: ダッシュボードデータを再集計

--------------------------------------

【2. 顧客管理】
契約中の顧客情報を登録・編集・削除します。

登録項目:
- 会社名（必須）
- プラン（プルダウン選択）
- 契約開始日 / 次回更新日
- 契約形態（月次/年次）
- MRR（月次経常収益）
- 担当CSM / 担当営業
- 業種 / 従業員規模 / 契約ユーザー数
- ステータス（契約中/解約予告/解約済/休止）
- 解約日 / 解約理由（解約時のみ）
- 備考

操作:
- +新規登録ボタンで新しい顧客を追加
- 各行の「編集」ボタンで情報を修正
- 各行の「削除」ボタンで顧客を削除（確認ダイアログあり）
- 検索ボックスで会社名を絞り込み

--------------------------------------

【3. プラン管理】
サブスクリプションプランの情報を管理します。

登録項目:
- プラン名（必須）
- 月額料金 / 年額料金
- 最大ユーザー数
- 主要機能フラグA / B（ON/OFF）
- サポートランク
- 備考

--------------------------------------

【4. CSMスタッフ】
CSM担当者の情報を管理します。

登録項目:
- 氏名（必須）
- メールアドレス（アラート通知先）
- ステータス（稼働中/休職中/退職済）

自動計算:
- 担当顧客数: 顧客マスタから自動集計
- 担当MRR合計: 担当顧客のMRR合計を自動集計

--------------------------------------

【5. スコアリング設定】
予兆スコアの計算に使う5つの指標を調整できます。

指標一覧:
- SCR001: ログイン間隔（日）- 重み30%、閾値14日、高いほど危険
- SCR002: 月次ログイン回数 - 重み25%、閾値3回、低いほど危険
- SCR003: 利用機能数（種類）- 重み20%、閾値2種類、低いほど危険
- SCR004: サポート問い合わせ件数 - 重み10%、閾値5件、高いほど危険
- SCR005: 契約更新まで残日数 - 重み15%、閾値60日、低いほど危険

各指標の「重み」「危険閾値」を変更すると、スコア算出ロジックに反映されます。
変更後は「設定を保存」ボタンを押してください。

--------------------------------------

【6. 利用ログ】
顧客の利用状況ログを確認できます。

表示項目:
- ログID / 顧客ID / ユーザーID
- ログイン日時 / セッション時間
- 利用機能数 / 利用機能リスト
- エクスポート実施 / エラー発生
- 取込日時

操作:
- ログ取込ボタン: 利用ログを一括取込（重複検知あり）

--------------------------------------

【7. サポート問い合わせ】
顧客からのサポート問い合わせを登録・管理します。

登録項目:
- 顧客（プルダウン選択）
- カテゴリ / 重要度
- ステータス（未対応/対応中/解決済）
- 解決日時
- 備考

--------------------------------------

【8. アラート管理】
自動検知されたアラートの一覧と対応状況を管理します。

アラート種別（5種類）:
- スコア閾値超え: 予兆スコアが高リスク閾値（初期値70）以上
- 急激な低下: 前日比でスコアが急落閾値（初期値15）以上悪化
- 連続未ログイン: 最終ログインから警告日数（初期値14日）以上経過
- 更新60日前: 契約更新日の60日前
- 更新30日前: 契約更新日の30日前

対応ステータス:
- 未対応: アラート発生直後の状態
- 対応中: CSMがフォロー対応を開始した状態
- 完了: フォロー対応が終了した状態
- 継続確定: 顧客の継続が確定した状態

操作:
- 各行の「変更」ボタンでステータスを変更（カスタムダイアログ表示）
- ダイアログにはアラートの詳細（ID/顧客/種別/現在のステータス）が表示されます

--------------------------------------

【9. フォロー対応記録】
アラートに対して実施したフォロー対応を記録します。

登録項目:
- アラートID（プルダウン選択、自動で顧客情報を取得）
- 顧客（プルダウン選択）
- 担当CSM（プルダウン選択）
- アクション種別（架電/メール送信/オンライン商談/訪問/特典提供/オンボーディング再実施/機能トレーニング/その他）
- 対応内容詳細
- 顧客反応（好意的/中立/否定的/応答なし）
- 次回アクション予定日 / 次回アクション内容
- 結果（継続確定/様子見/解約確定/未確定）
- 解約防止成功フラグ

フォロー対応を登録すると、関連するアラートのステータスが自動的に「対応中」に更新されます。

--------------------------------------

【10. レポート】
各種集計データを確認・分析できます。6つのレポートがタブで切り替えられます。

(1) スコアトレンド
- 顧客を選択すると、日別の予兆スコア推移を折れ線グラフで表示
- スコア構成（ログイン間隔/ログイン回数/機能利用数/サポート/更新残日数）も確認可能

(2) 日次指標
- 顧客ごとの日次集計データ一覧
- ログイン回数/間隔/月次ログイン/機能数/サポート件数/スコア/変化/リスクランク

(3) 月次サマリー
- 顧客ごとの月次集計データ一覧
- 月間ログイン回数/機能数/サポート件数/平均間隔/月末スコア/前月比

(4) CSM別パフォーマンス
- CSMごとの月次パフォーマンス
- 担当数/アラート数/フォロー件数/実施率/防止成功数/成功率/MRR/チャーン率

(5) アクション効果検証
- アクション種別ごとの効果分析
- 実施件数/成功件数/成功率/回復件数/回復率/平均スコア変化

(6) 解約分析
- 月別の解約状況分析
- 解約件数/解約MRR/チャーンレート/検知あり・なし/理由別内訳

--------------------------------------

【11. 設定】
システム全体の設定値を変更できます。

設定項目:
- スコア高リスク閾値（初期値: 70）: このスコア以上でアラート送信
- スコア急落閾値（初期値: 15）: 前日比でこの点数以上低下したらアラート
- 連続未ログイン警告日数（初期値: 14）: この日数以上ログインなしでアラート
- 更新前アラート_第1通知（初期値: 60）: 更新N日前にアラート送信
- 更新前アラート_第2通知（初期値: 30）: 更新N日前にアラート送信
- CSMフォロー期限日数（初期値: 3）: アラート発生からN日以内に対応必須
- 通知先メールアドレス（管理者）: 管理者通知先メールアドレス
- データ保持期間（初期値: 24ヶ月）: トレンド履歴等の保持期間

変更後は「設定を保存」ボタンを押してください。

画面下部にはGAS処理履歴が表示されます。
各自動処理・手動実行の処理名/処理件数/アラート発生件数/メール送信件数/処理結果/エラー内容が確認できます。

========================================
スプレッドシート構成（全17シート）
========================================

【マスタ系】（4シート）
- マスタ_顧客: 顧客情報（18列）
- マスタ_プラン: プラン情報（9列）
- マスタ_CSMスタッフ: CSM担当者情報（6列）
- マスタ_スコアリング設定: スコア計算の指標設定（6列）

【トランザクション系】（2シート）
- トランザクション_利用ログ: 顧客の利用ログ（10列）
- トランザクション_サポート問い合わせ: サポート問い合わせ記録（8列）

【集計系】（6シート）※GAS自動書き込み専用（編集不可）
- 集計_顧客別日次指標: 日次のスコア計算結果（16列）
- 集計_顧客別月次サマリー: 月次集計結果（13列）
- 集計_スコアトレンド履歴: スコアの推移記録（10列）
- 集計_CSM別パフォーマンス: CSMごとの月次実績（13列）
- 集計_アクション効果検証: フォローアクションの効果分析（8列）
- 集計_解約分析: 月別解約分析（13列）

【運用系】（3シート）
- アラート管理: 発生したアラートの管理（14列）
- フォロー対応記録: フォロー対応の記録（13列）
- 設定: システム設定値（3列）

【システム系】（2シート）※GAS自動書き込み専用（編集不可）
- ログ_GAS処理履歴: 自動処理の実行ログ（8列）
- ダッシュボード: ダッシュボード用集計値（2列）

========================================
自動処理（トリガー: 全12種）
========================================

【日次（毎日自動実行）】
- 利用ログ取込（AM1:00）: 利用ログを取り込み、重複を検知
- 日次指標計算（AM2:00）: 全顧客の予兆スコアを算出
- スコアトレンド記録（AM2:15）: スコアの推移を履歴に記録
- アラート検知（AM2:30）: 閾値超え/急落/未ログインのアラートを自動発生、CSMにメール通知
- 契約更新前アラート（AM3:00）: 更新60日前/30日前のアラートを発生、CSMと管理者にメール通知
- ダッシュボード更新（AM3:30）: ダッシュボードの集計値を更新
- 未対応アラートリマインダー（AM9:00）: 未対応アラートがあるCSMにリマインダーメール送信
- フォロー対応リマインダー（AM9:30）: フォロー予定日を過ぎた対応があるCSMにリマインダーメール送信

【月次（毎月1日自動実行）】
- 月次サマリー集計（AM4:00）: 前月の顧客別月次データを集計、解約分析も実行
- CSMパフォーマンス集計（AM4:30）: 前月のCSM別実績を集計
- アクション効果検証（AM5:00）: フォローアクションの効果を分析

【週次（毎週月曜自動実行）】
- 週次サマリーレポート（AM9:00）: 全CSMにHTML形式の週次レポートをメール送信

========================================
メール通知（全4種類）
========================================

1. 解約予兆アラート通知
   - 送信先: 担当CSM
   - タイミング: アラート検知時（日次AM2:30）
   - 内容: 顧客名/アラート種別/トリガー値/リスクランク/推奨対応

2. 契約更新アラート通知
   - 送信先: 担当CSM + 管理者
   - タイミング: 更新60日前/30日前（日次AM3:00）
   - 内容: 顧客名/更新日/残日数/現在のスコア

3. 未対応アラートリマインダー
   - 送信先: 担当CSM + 管理者
   - タイミング: 毎朝AM9:00
   - 内容: 未対応アラートの一覧/経過日数

4. 週次サマリーレポート
   - 送信先: 全CSM
   - タイミング: 毎週月曜AM9:00
   - 内容: ダッシュボード指標/高リスク顧客Top10

========================================
スプレッドシートのカスタムメニュー
========================================

スプレッドシートを開くと「解約予兆管理」メニューが表示されます。

- ログ取込: 利用ログを手動で一括取込
- スコア再計算: 日次指標計算を手動実行
- アラート検知実行: アラート検知を手動実行
- ダッシュボード更新: ダッシュボード集計値を手動更新

========================================
予兆スコアの算出ロジック
========================================

5つの指標をそれぞれ0-100点に正規化し、重みを掛けて合計します。

計算例:
- ログイン間隔が20日（閾値14日）の場合: (20/14)*100=143 → 上限100 × 重み30% = 30点
- 月次ログイン5回（閾値3回）の場合: (1-5/6)*100=17 × 重み25% = 4点
- 合計スコア = 各指標の加重合計（0-100点）

リスクランク判定:
- 70点以上: 高リスク（アラート対象）
- 42点以上: 中リスク（注意）
- 42点未満: 低リスク

========================================
条件付き書式（スプレッドシート側）
========================================

- 日次指標シート: リスク「高」の行は赤背景、「中」は黄背景。スコア変化が-15以下は赤太字
- アラート管理シート: 3日以上未対応の行はオレンジ背景。ステータス別色分け（未対応:赤/対応中:黄/完了:灰/継続確定:緑）
- CSMパフォーマンスシート: フォロー実施率 80%以上は緑、50-79%は黄、50%未満は赤

========================================
注意事項
========================================

- 集計系シートとログシートはGASが自動で書き込むため、手動編集しないでください
- スコアリング設定の重みの合計が100%になるように調整してください
- メール通知はCSMスタッフに登録されたメールアドレスに送信されます
- 管理者メールアドレスは設定画面から変更できます
- トリガーはシート初期化時に自動設定されます
- 利用ログは外部システムからスプレッドシートに直接書き込むか、手動で入力してください
