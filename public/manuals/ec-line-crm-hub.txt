================================================================================
  EC × LINE CRM ハブシステム　使い方説明書
================================================================================

■ このシステムについて
────────────────────────────────────────────────────────────────────────────────

EC × LINE CRM ハブシステムは、ECプラットフォーム（Shopify・楽天・BASE）の
購買データと LINE CRM（Lステップ）を自動連携するミドルウェアです。

管理画面URL: https://ec-line-crm-hub.netlify.app/

■ こんなことに使えます
────────────────────────────────────────────────────────────────────────────────

1. ECの購買データとLINEユーザーを自動で紐付ける
   → 「LINEの〇〇さんが楽天で何を買ったか」が分かるようになります

2. 購買行動に基づいた自動セグメント分類
   → 初回購入者・優良リピーター・休眠顧客などを自動判定します

3. セグメントに応じたLINE配信の自動実行
   → Lステップ経由で最適なタグ付け・シナリオ起動を自動化します

4. 複数ECプラットフォームの購買データを一元管理
   → Shopify・楽天・BASEの注文を1つの画面で確認できます

【活用例】
・初回購入者にLINEで使い方ガイドを自動配信
・90日以上購入がない高LTV顧客にVIP復帰クーポンを自動送信
・購入3回以上の優良顧客にだけ新作の先行案内を配信
・消耗品の補充タイミング（25日後）にリピート促進メッセージを配信
・楽天とShopifyの購買データを統合し、正確なLTVでセグメント判定


================================================================================
  ログイン
================================================================================

1. 管理画面URL にアクセスします
2. メールアドレスとパスワードを入力します
3. 「ログイン」をクリックします

※ ログインセッションは8時間有効です。8時間後は再ログインが必要です。
※ メールアドレス・パスワードは「設定」タブから変更できます。

【初期設定】
- メール: admin@ikemen.co.jp
- パスワード: admin1234

================================================================================
  画面構成（全5タブ）
================================================================================

管理画面は以下の5つのタブで構成されています。
左側メニューからタブを切り替えます。

  1. ダッシュボード …… KPI・グラフで全体状況を把握
  2. ユーザー管理   …… 突合済み顧客の一覧・検索

  4. 配信ログ       …… Lステップへの配信結果の確認・再実行
  5. 設定           …… ログイン情報・API連携・Webhook署名の管理

================================================================================
  セグメント自動判定について
================================================================================

このシステムは、購買データに基づいて顧客を5つのセグメントに
自動分類します。セグメントが変わるたびにLステップのタグが
自動で付け替えられます。

■ セグメント一覧
────────────────────────────────────────────────────────────────────────────────

  ┌──────────────────┬─────────────────────────────────┬──────────────────────┐
  │ セグメント名     │ 判定条件                        │ Lステップ操作        │
  ├──────────────────┼─────────────────────────────────┼──────────────────────┤
  │ 初回購入         │ 購入回数 = 1回                  │ タグ付与             │
  │                  │                                 │ + シナリオ起動       │
  │                  │                                 │  （onboarding）      │
  ├──────────────────┼─────────────────────────────────┼──────────────────────┤
  │ 優良リピーター   │ 購入回数 3回以上                │ タグ付与             │
  │                  │ かつ 最終購入から60日以内        │                      │
  ├──────────────────┼─────────────────────────────────┼──────────────────────┤
  │ 休眠（高LTV）    │ 最終購入から90日超              │ タグ付与             │
  │                  │ かつ 累計LTV 10,000円超         │                      │
  ├──────────────────┼─────────────────────────────────┼──────────────────────┤
  │ 休眠（低LTV）    │ 最終購入から90日超              │ タグ付与             │
  │                  │ かつ 累計LTV 10,000円以下       │                      │
  ├──────────────────┼─────────────────────────────────┼──────────────────────┤
  │ 補充タイミング   │ 最終購入から25日目              │ タグ付与             │
  │                  │ かつ 消耗品カテゴリの購入        │                      │
  ├──────────────────┼─────────────────────────────────┼──────────────────────┤
  │ 未分類           │ 上記のいずれにも該当しない      │ なし                 │
  └──────────────────┴─────────────────────────────────┴──────────────────────┘

  ※ 消耗品カテゴリ: supplement, skincare, food, beverage, consumable
  ※ セグメントが変更されると、旧タグが削除され新タグが自動付与されます。

■ 判定タイミング
────────────────────────────────────────────────────────────────────────────────

  ・突合完了直後    … EC顧客とLINEが紐付いた瞬間に即時判定
  ・1時間ごと       … 未処理の購買イベントをバッチ処理で判定
  ・毎朝9時         … 全ユーザーの補充タイミング（25日目）をチェック


================================================================================
  顧客突合（マッチング）の仕組み
================================================================================

ECの購買データとLINEユーザーを紐付ける方法は2種類あります。

■ 方法1: Lステップ スプシ同期による自動突合（推奨）
────────────────────────────────────────────────────────────────────────────────

  Lステップのプロプラン以上で利用できる「友だちデータの
  スプレッドシート同期」機能を使います。

  流れ:
  1. EC購買イベントを受信（Shopify/BASE/楽天）
  2. Lステップ同期スプシからメールアドレスが一致するLINE User IDを検索
  3. 一致 → 突合完了（顧客操作ゼロ）
  4. 不一致 → 方法2へフォールバック

  ※ メールアドレスの収集率が突合率を左右します。
     LINE登録直後にメールアドレスを聞くフローを設計してください。

■ 方法2: 顧客による手動入力での突合
────────────────────────────────────────────────────────────────────────────────

  注文確認メールにLINE誘導URLを挿入し、顧客にLINEで
  メールアドレスまたは注文番号を入力してもらいます。

  Shopify / BASE の場合:
  1. 注文確認メールにLINE誘導URLを記載
  2. 顧客がURLをタップ → LINEへ遷移
  3. ボットがメールアドレスの入力を促す
  4. 入力されたメールのハッシュ値で照合 → 突合完了

  楽天の場合:
  1. 注文確認メールにLINE誘導URLを記載
  2. 顧客がURLをタップ → LINEへ遷移
  3. ボットが注文番号の入力を促す
  4. 入力された注文番号で照合 → 突合完了

  スプレッドシートの構成
================================================================================

データは1つのGoogleスプレッドシート内の6つのシートで管理されています。

  ┌────────────────────┬────────────────────────────────────────────┐
  │ シート名           │ 内容                                       │

================================================================================
  よくある質問
================================================================================

Q. 購買イベントに注文が表示されるのに、ユーザー管理に出てこない
A. 購買イベントは受信済みですが、LINEユーザーとの突合が完了して
   いません。突合が完了するとユーザー管理に表示されます。

Q. 配信ログに「[MOCK] Lstep API未接続」と表示される
A. Lステップの APIトークンが設定されていません。
   設定タブ → Lステップ連携設定 からAPIトークンを登録してください。
   モックモードではLステップへの実際の配信は行われません。

Q. 配信ログに失敗が記録された
A. 配信ログタブで「失敗」でフィルターし、エラーメッセージを確認
   してください。「再実行」ボタンで再試行できます。
   繰り返し失敗する場合はLステップ側の設定を確認してください。

Q. セグメントが変わらない
A. セグメント判定は突合完了時と1時間ごとのバッチで実行されます。
   最大1時間の遅延が発生する場合があります。

Q. 楽天の注文がなかなか反映されない
A. 楽天は15分間隔のポーリングで取得しています。
   最大15分の遅延が発生します。
   設定タブで楽天APIキーが正しく設定されているか確認してください。

Q. 同じ注文が2件表示される
A. ShopifyのWebhookが同一イベントを複数回送信する場合があります。
   データの整合性には影響ありません。

Q. パスワードを忘れた
A. スプレッドシートのconfigシートまたはGASのPropertiesServiceから
   パスワードハッシュをリセットする必要があります。
   管理者にお問い合わせください。
