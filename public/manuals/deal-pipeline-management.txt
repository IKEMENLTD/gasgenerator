================================================================================
商談パイプライン管理 — 使い方説明書
================================================================================

■ このシステムでできること
────────────────────────────────────────
・案件をフェーズ別（リード〜受注/失注）に管理し、パイプライン全体を可視化
・停滞案件を自動検出し、担当者・管理者にメールでアラート通知
・フェーズ変更履歴を自動記録し、各フェーズの滞在日数を把握
・週次でパイプラインサマリーとスタッフ別実績を自動集計
・スタッフ別の目標達成率・受注率をダッシュボードで即座に確認
・カンバンビューで案件の進捗状況を直感的に把握

================================================================================
■ 画面構成（8タブ）
================================================================================

1. ダッシュボード
2. スタッフ管理
3. 顧客管理
4. 案件管理（パイプライン）
5. フェーズ変更履歴
6. パイプラインサマリー
7. スタッフ別パイプライン
8. 週次レポートログ

================================================================================
■ 各タブの使い方
================================================================================

────────────────────────────────────────
1. ダッシュボード
────────────────────────────────────────
パイプライン全体の状況を一目で把握するための画面です。

【統計カード】
・全加重期待値合計: パイプライン内の見込み売上合計（案件金額×受注確度）
・アクティブ案件数: 受注・失注を除く進行中の案件数
・停滞案件数: フェーズごとの閾値を超えて滞留している案件数
・期限超過案件数: 想定クローズ日を過ぎている案件数
・当週受注件数: 今週受注が確定した案件数

【チャート】
・フェーズ別案件件数ファネル: 各フェーズにある案件数を横棒グラフで表示
・フェーズ別加重期待値合計: 各フェーズの見込み売上を縦棒グラフで表示
・フェーズ別停滞率推移: 過去8週間のフェーズ別停滞率を折れ線グラフで表示
・スタッフ別目標達成率: スタッフごとの目標に対する達成率を横棒グラフで表示
・スタッフ別受注率: スタッフごとの受注率を横棒グラフで表示

【停滞・期限超過案件一覧】
・停滞中または期限超過の案件を一覧表示（停滞日数の多い順）
・案件名、顧客名、フェーズ、案件金額、停滞日数、状態、次アクション期日を確認可能

────────────────────────────────────────
2. スタッフ管理
────────────────────────────────────────
営業スタッフの情報を登録・管理する画面です。

【新規登録】
1.「+ 新規登録」ボタンをクリック
2. 以下の情報を入力:
   ・氏名（必須）
   ・メールアドレス（必須）— 停滞アラートや週次レポートの送信先
   ・部署: 営業部 / コンサル部 / マーケ部 / 経営企画 / その他
   ・役職: 担当 / リーダー / マネージャー / 部長 / その他
   ・月次売上目標（円）— 目標達成率の算出に使用
   ・有効フラグ: 有効 / 無効
3.「保存」ボタンで登録

【編集・削除】
・一覧の「編集」ボタンで情報を修正
・「削除」ボタンでスタッフを削除（確認ダイアログあり）

【フィルター・検索】
・部署で絞り込み
・有効/無効ステータスで絞り込み
・テキスト検索（氏名・メールアドレス等）

【補足】
・役職が「マネージャー」「部長」のスタッフには、全体の停滞アラートと
  週次レポートがメールで自動送信されます
・無効にしたスタッフは案件の担当者選択肢に表示されなくなります

────────────────────────────────────────
3. 顧客管理
────────────────────────────────────────
取引先（顧客企業）の情報を登録・管理する画面です。

【新規登録】
1.「+ 新規登録」ボタンをクリック
2. 以下の情報を入力:
   ・会社名（必須）
   ・業種: 製造業 / IT / 金融 / 不動産 / 医療 / 小売 / コンサル / その他
   ・企業規模: 大企業 / 中堅企業 / 中小企業 / スタートアップ
   ・担当者名 — 顧客側の窓口担当者
   ・担当者メール
   ・担当スタッフ — 自社側の担当営業
   ・地域: 北海道 / 東北 / 関東 / 中部 / 近畿 / 中国四国 / 九州沖縄
   ・備考（任意）
3.「保存」ボタンで登録

【編集・削除】
・一覧の「編集」ボタンで情報を修正
・「削除」ボタンで顧客を削除（確認ダイアログあり）

【フィルター・検索】
・業種で絞り込み
・地域で絞り込み
・テキスト検索（会社名・担当者名等）

────────────────────────────────────────
4. 案件管理（パイプライン）
────────────────────────────────────────
商談案件の登録・フェーズ管理を行うメイン画面です。
テーブルビューとカンバンビューを切り替えて使用できます。

【新規登録】
1.「+ 新規登録」ボタンをクリック
2. 以下の情報を入力:
   ・案件名（必須）
   ・顧客（必須）— プルダウンから選択
   ・担当スタッフ（必須）— プルダウンから選択
   ・現フェーズ: リード / 初回接触 / ヒアリング / 提案 / 見積 / クロージング / 受注 / 失注
   ・受注確度: 10% / 20% / 30% / 50% / 70% / 90% / 100%
   ・案件金額（必須・正の整数）
   ・初回接触日
   ・想定クローズ日 — 初回接触日より後の日付
   ・実クローズ日 — 受注/失注の場合は必須
   ・優先度: 高 / 中 / 低
   ・失注理由 — フェーズが「失注」の場合のみ表示
     （価格 / 競合他社 / 予算凍結 / 要件不一致 / タイミング / その他）
   ・次アクション（自由記述）
   ・次アクション期日
3.「保存」ボタンで登録

【自動算出される項目（編集不可）】
・加重期待値 = 案件金額 × 受注確度 / 100
・停滞日数 = 当日 − フェーズ更新日（受注/失注の場合は0）
・停滞フラグ: フェーズごとの閾値を超えると「停滞中」に自動変更
  - リード・初回接触・クロージング: 7日以上で停滞
  - ヒアリング・提案: 14日以上で停滞
  - 見積: 10日以上で停滞
・クローズ期限超過フラグ: 想定クローズ日を過ぎると「期限超過」に自動変更

【テーブルビュー】
・全案件を一覧表示
・行の背景色で状態を一目で判別:
  - オレンジ背景: 停滞中の案件
  - 赤背景: 期限超過の案件（最優先で対応が必要）
  - 緑背景: 受注済みの案件
  - グレー背景: 失注した案件
・受注確度の色分け:
  - 90〜100%: 緑太字
  - 10〜20%: 赤色
・次アクション期日の色分け:
  - 当日以前: 赤太字（対応遅れ）
  - 3日以内: オレンジ色（要注意）

【カンバンビュー】
・右上のボード表示アイコンで切り替え
・リード〜クロージングの6フェーズを列として表示（受注・失注は非表示）
・各カードに案件名、顧客名、案件金額、停滞日数、担当者を表示
・オレンジ枠: 停滞中 / 赤枠: 期限超過
・カードをクリックすると編集フォームを表示

【フィルター・検索】
・フェーズで絞り込み
・担当者で絞り込み
・優先度で絞り込み
・テキスト検索（案件名・顧客名・担当者名等）

────────────────────────────────────────
5. フェーズ変更履歴
────────────────────────────────────────
案件のフェーズが変更されるたびに自動記録される履歴の閲覧画面です。

【表示項目】
・案件名
・変更前フェーズ → 変更後フェーズ
・変更日時
・変更者（担当スタッフ名）
・フェーズ滞在日数（前のフェーズに何日間滞在したか）

【検索】
・テキスト検索で案件名やフェーズ名で絞り込み可能

【補足】
・この画面はGASが自動記録するため、手動での編集はできません
・フェーズの変更は案件管理タブの編集フォームまたはスプレッドシート上で行います

────────────────────────────────────────
6. パイプラインサマリー
────────────────────────────────────────
フェーズ別の案件状況を週単位で集計した画面です。毎日07:30に自動更新されます。

【表示項目】
・集計週（YYYY-WNN形式）
・集計基準日
・フェーズ名
・案件件数
・案件金額合計
・加重期待値合計
・停滞案件件数
・停滞率（%）— 停滞案件件数 / 全案件件数 × 100
・期限超過件数
・週次増減 — 前週からの案件件数の増減
・最終集計日時

【条件付き書式】
・停滞率 30%以上: 赤色（要注意）
・停滞率 15%以上: オレンジ色
・週次増減がマイナス: 赤色（案件数が減少）
・週次増減がプラス: 緑色（案件数が増加）

【フィルター】
・集計週で絞り込み

────────────────────────────────────────
7. スタッフ別パイプライン
────────────────────────────────────────
スタッフごとのパイプライン実績を週単位で集計した画面です。毎日08:00に自動更新。

【表示項目】
・集計週
・氏名
・部署
・担当案件数（受注・失注除く）
・担当案件金額合計
・加重期待値合計
・目標達成率（%）— 加重期待値合計 / 月次売上目標 × 100
・停滞案件件数
・受注件数
・失注件数
・受注率（%）— 受注件数 / (受注件数 + 失注件数) × 100
  ※受注+失注がゼロの場合は「-」表示
・最終集計日時

【条件付き書式】
・目標達成率 100%以上: 緑太字
・目標達成率 50%未満: 赤色
・受注率 50%以上: 緑色

【フィルター】
・集計週で絞り込み

────────────────────────────────────────
8. 週次レポートログ
────────────────────────────────────────
毎週月曜08:30に自動生成・送信される週次レポートの送信記録です。

【表示項目】
・集計週
・生成日時
・送信先メール
・送信ステータス（送信済み / 送信失敗）
・全案件数
・全加重期待値合計
・停滞案件総数
・期限超過案件総数
・備考（エラー詳細等）

【補足】
・レポートはマネージャー・部長以上の役職を持つスタッフに自動送信されます
・送信先がない場合は「送信先なし」として記録されます

================================================================================
■ 自動処理（トリガー）
================================================================================

以下の処理が自動で実行されます。手動操作は不要です。

1. フェーズ変更検知（即時）
   → 案件のフェーズを変更すると、変更履歴を自動記録

2. 加重期待値・停滞指標の日次更新（毎日 07:00）
   → 全案件の加重期待値・停滞日数・停滞フラグ・期限超過フラグを再計算

3. パイプラインサマリー集計（毎日 07:30）
   → フェーズ別の案件件数・金額・停滞率を週単位で集計

4. スタッフ別パイプライン集計（毎日 08:00）
   → スタッフごとの担当案件数・目標達成率・受注率を集計

5. 停滞案件アラートメール（毎日 09:00）
   → 停滞中・期限超過の案件を担当者にメール通知
   → マネージャー以上には全体の停滞一覧を送信

6. 週次レポート生成（毎週月曜 08:30）
   → パイプライン全体のサマリーレポートをマネージャー以上にメール送信
   → 送信結果を週次レポートログに記録

7. 新規案件初期値設定（即時）
   → スプレッドシートから直接案件を追加した際にID等を自動設定

================================================================================
■ メール通知
================================================================================

【停滞アラートメール（毎日 09:00）】
・対象: 停滞中または期限超過の案件がある担当スタッフ
・内容: 案件名、顧客名、フェーズ、案件金額、停滞日数、次アクション
・管理者向け: 全スタッフの停滞案件をまとめて送信

【週次レポートメール（毎週月曜 08:30）】
・対象: マネージャー・部長以上の役職スタッフ
・内容: アクティブ案件数、加重期待値合計、停滞案件数、
  期限超過案件数、フェーズ別内訳

================================================================================
■ 活用シーン
================================================================================

・営業チームの案件進捗を一元管理したい
・停滞している案件をいち早く発見し、対応漏れを防ぎたい
・マネージャーがチーム全体のパイプライン状況を即座に把握したい
・スタッフごとの売上目標に対する達成度を可視化したい
・フェーズの遷移パターンを分析し、営業プロセスを改善したい
・週次の営業会議で使えるレポートを自動生成したい
