========================================
生徒管理＋進捗記録システム 使い方説明書
========================================

--------------------
このシステムでできること
--------------------

学習塾・スクール向けの生徒管理と進捗記録を一元管理するWebアプリケーションです。

- 生徒の基本情報・保護者情報・コース・担当講師を統合管理
- 毎回の授業内容・理解度・演習結果を記録し、指導品質を可視化
- 出欠管理と連続欠席の自動検知・アラート通知
- 成績・テスト結果の記録と推移分析
- 退塾リスクスコアの自動算出（欠席率・成績変化・理解度・宿題提出率・保護者連絡頻度・進捗遅れの6指標）
- 保護者連絡・面談記録の一元管理とフォロー予定の追跡
- 月次集計（生徒別サマリー・講師パフォーマンス・コース分析・退塾分析・教材効果分析）
- Slack/メール通知による自動アラート

--------------------
画面構成
--------------------

[1] ダッシュボード
  - 在籍生徒数、退塾リスク（高/緊急）人数、連続欠席中の人数、未完了フォロー件数を一目で確認
  - 退塾リスク要注意生徒の一覧（スコア・担当講師表示）
  - 連続欠席中の生徒一覧
  - 授業記録未入力の講師一覧
  - 未完了フォロー予定一覧

[2] 生徒一覧
  - 全生徒をテーブル形式で表示
  - 在籍ステータスでフィルタリング（在籍/休会/退塾/卒業/全て）
  - 氏名・カナ・学校名・講師名でインクリメンタルサーチ
  - 各生徒の「詳細」ボタンから生徒サマリーを表示

[3] 生徒登録
  - 氏名・カナ・性別・生年月日・学年・学校名を入力
  - 保護者情報（氏名・続柄・電話番号・メール・連絡希望方法・時間帯）
  - コース・担当講師の選択
  - 入塾日・入塾経路・目標・目標期限・特記事項を入力
  - 登録と同時にカリキュラム進捗レコードが自動初期化される

[4] 出欠登録
  - 授業日・コース・時間帯を選択
  - 「生徒一覧を読み込む」ボタンで対象コースの在籍生徒を一覧表示
  - 生徒ごとに出欠区分（出席/欠席/遅刻/早退）を選択
  - 欠席の場合は理由を選択
  - 欠席時は連続欠席数が自動カウント・アラートチェック

[5] 授業記録入力
  - 授業日・生徒・コース・担当講師を選択
  - 実施単元・使用教材・ページ範囲を入力
  - 授業内容サマリーを記載
  - 理解度評価（1-5）を直感的なアイコンで入力
  - 演習正答率・理解度コメント・つまずき箇所/原因・良かった点を記録
  - 次回重点事項・宿題内容・授業態度を入力
  - 保護者共有フラグで共有内容を設定
  - カリキュラム進捗が自動更新される

[6] 成績・テスト結果登録
  - 生徒・テスト種別（定期テスト/模試/単元テスト等）を選択
  - テスト名・実施日・科目・満点・得点を入力
  - 偏差値・順位（クラス/全体）・目標得点を入力
  - 強かった分野・弱かった分野・講師コメントを記録
  - 前回差・得点率・目標達成フラグが自動計算
  - 成績低下時は自動アラート＋退塾リスク即時再計算

[7] 保護者連絡・面談記録
  - 生徒・対応者・連絡方法・種別を選択
  - 連絡内容・保護者からの要望・満足度を記録
  - クレームフラグで管理
  - 次回フォロー要の場合、予定日・内容を入力するとフォロー予定に自動追加

[8] フォロー予定
  - 未対応/対応中/完了/全てでフィルタリング
  - 優先度（高/中/低）と期限超過日数を表示
  - 「完了」ボタンでフォロー完了処理

[9] 退塾処理
  - 生徒・退塾日・退塾理由を選択
  - 自塾起因の理由の場合、改善アクション入力欄が表示
  - 二重確認後に退塾処理を実行

[10] 集計実行
  - 月次集計（全項目）: 生徒別サマリー/講師パフォーマンス/コース分析/退塾分析/教材効果分析を一括実行
  - 退塾リスク再計算: 全在籍生徒のリスクスコアを即時再計算

[11] 設定
  - システム設定一覧の確認（アラート閾値・通知先等）
  - テストデータ挿入: サンプルデータ（生徒8名、講師4名、コース5件等）を一括挿入
  - 全データクリア: 設定と退塾理由を除く全データを削除

--------------------
生徒サマリー（詳細表示）
--------------------

生徒一覧やダッシュボードから生徒名をクリックすると、以下を含むサマリーが表示されます:
- 基本情報（学年・コース・担当・入塾日・ステータス・リスクスコア・連続欠席数・目標）
- 特記事項がある場合はバナーで常時表示
- 出席統計（出席率・出席回数・総授業数）
- 目標・カリキュラム進捗（実績vs予定のバーチャート、遅延判定）
- 直近5件の授業記録タイムライン（理解度・正答率・つまずき・良い点）
- 成績推移（直近6回分のバーチャート、前回差表示）
- 未完了フォロー一覧
- 保護者連絡履歴（直近5件）

--------------------
退塾リスクスコアについて
--------------------

以下6指標の重み付けスコアで0-100の退塾リスクを自動算出します:
- 直近1ヶ月の欠席率（ウェイト30%）
- 直近2回のテスト成績変化（ウェイト20%）
- 直近1ヶ月の平均理解度（ウェイト15%）
- 直近1ヶ月の宿題提出率（ウェイト10%）
- 最終保護者連絡からの経過日数（ウェイト10%）
- 目標進捗率の遅れ幅（ウェイト15%）

判定基準:
- 0-39: 低（安定）
- 40-59: 中（経過観察）
- 60-79: 高（要注意）
- 80-100: 緊急（緊急対応）

--------------------
自動トリガー
--------------------

以下のトリガーが自動実行されます（GASエディタで setupTriggers を実行して設定）:
- 毎日 0:00 - 授業記録未入力チェック
- 毎日 1:00 - 退塾リスクスコア一括再計算
- 毎日 9:00 - フォロー期限チェック
- 毎日 10:00 - 連続欠席アラート
- 毎月 1日 2:00 - 月次集計（全項目）
- 毎月 25日 9:00 - 月次進捗レポート送付リマインド

--------------------
通知設定
--------------------

設定シートで以下を設定すると通知が有効になります:
- Slack Webhook URL: Slack通知用のIncoming Webhook URL
- 管理者メールアドレス: エスカレーション通知用

未設定の場合、通知はスキップされます（ログに記録のみ）。

--------------------
シート構成（21シート）
--------------------

[マスタ] 生徒 / コース / 講師 / 教材 / 単元・カリキュラム / 退塾理由 / 通知テンプレート
[トランザクション] 出欠記録 / 授業記録 / 成績・テスト結果 / 目標・カリキュラム進捗 / 面談・保護者連絡履歴 / フォロー予定 / 改善アクション
[集計] 生徒別月次サマリー / 講師別パフォーマンス / コース別分析 / 退塾分析 / 教材別効果分析
[その他] システム設定 / 操作履歴

--------------------
初回セットアップ
--------------------

1. GASエディタで initializeSheets 関数を実行（全シート＋ヘッダー＋デフォルト設定を自動作成）
2. GASエディタで setupTriggers 関数を実行（自動トリガーを設定）
3. ウェブアプリの「設定」ページで「テストデータ挿入」を実行（任意）
4. 必要に応じてSlack Webhook URLや管理者メールを設定シートに入力

--------------------
対応デバイス
--------------------

- PC（推奨）: フル機能利用可能
- タブレット: レスポンシブ対応
- スマートフォン: ハンバーガーメニューで全機能利用可能
