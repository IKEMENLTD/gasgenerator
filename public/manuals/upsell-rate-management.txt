==============================================================================
  アップセル実施率管理 ― 使い方説明書
==============================================================================

【このシステムでできること】
 - スタッフごとのアップセル提案率・成功率・客単価を数値で把握できる
 - 提案しないことで逃している売上（機会損失額）を自動で試算できる
 - 商品別・スタッフ別にどの提案が成功しやすいかパターン分析できる
 - 研修や指導の前後で数値がどう変わったか育成効果を追跡できる
 - 設定した閾値を下回るスタッフに自動でアラート通知を送れる
 - ダッシュボードで今月の全体状況をひと目で確認できる

【こんな方におすすめ】
 - 美容室・エステ・ネイルサロンのスタッフと店長
 - 飲食店のホールスタッフ・店長
 - サービス業の販売員・マネージャー
 - 複数店舗の実績を一元把握したいエリアマネージャー・経営者

==============================================================================
  1. ダッシュボード
==============================================================================

今月のアップセル実績を一画面で俯瞰できるページです。

【数値カード（6枚）】
 - 今月アップセル金額合計 … アップセル成功で得た金額の合計
 - 今月接客件数           … 今月の全接客数
 - 提案率                 … 接客のうちアップセルを提案した割合
 - 成功率                 … 提案のうち成功した割合
 - 平均客単価             … 基本料金＋アップセルの平均
 - 機会損失額合計         … 未提案による取りこぼし推定額

【グラフ】
 - 月別推移（12ヶ月）… 提案率・成功率・客単価の推移を折れ線グラフで表示
 - 週次提案率推移（8週）… 直近8週の提案率トレンド
 - 機会損失額スタッフ別 … スタッフごとの機会損失を棒グラフで比較

【テーブル】
 - スタッフ別パフォーマンス一覧
   → 提案率降順で全スタッフの実績・ランク（S/A/B/C/D）を表示
 - 要指導スタッフ Top3
   → 店舗平均との差が大きい下位3名を自動抽出
 - 商品別成功率ランキング Top10
   → 成功率が高い商品を上位10件表示

【その他】
 - スタッフ×商品 成功率マトリクス
   → 誰がどの商品の提案が得意かをヒートマップで可視化
   → 緑＝60%以上 / 黄＝30%以上 / 赤＝30%未満
 - 未解消アラート・育成効果サマリー
   → 未解消アラート件数、育成イベント数、改善率を表示
 - 集計対象期間 / データ最終更新日時


==============================================================================
  2. 接客記録
==============================================================================

アップセル提案の実績を登録・管理するメイン画面です。

【一覧画面】
 - 接客ID / 接客日時 / 店舗 / スタッフ / 客単価_基本 /
   提案有無 / アップセル合計 / 最終客単価
 - フィルタ: 店舗 / スタッフ / 日付範囲 / キーワード検索

【新規登録・編集】
 「+ 新規接客記録」ボタンで登録フォームを開きます。

 入力項目:
  - 接客日時（必須）
  - 店舗（ドロップダウン選択）
  - スタッフ（ドロップダウン選択）
  - 客単価_基本（アップセル前の料金）
  - 来店回数区分（初回 / 2〜5回目 / リピーター / VIP）
  - 客層（男性 / 女性 / 不明）
  - 年代（10代〜50代以上 / 不明）
  - 提案商品1〜3（ドロップダウンから選択、最大3商品まで）
  - 提案結果1〜3（成功 / 失敗 / なし）
  - 備考（自由入力）

 ※ アップセル合計金額・最終客単価・提案フラグは自動計算されます
 ※ 接客IDは自動採番されます

【削除】
 一覧の「削除」ボタンで個別に削除できます。
 ※ 確認ダイアログが表示されます

==============================================================================
  10. アラート管理
==============================================================================

設定した閾値を下回るスタッフを自動検知し、アラートを記録・通知します。

【アラート種別】
 - 提案率低下 … 提案率が閾値（デフォルト20%）を下回った
 - 成功率低下 … 成功率が閾値（デフォルト15%）を下回った
 - 提案ゼロ連続 … N日連続（デフォルト3日）提案がゼロ
 - 機会損失高額 … 機会損失額が閾値（デフォルト30,000円）以上

【表示項目】
 アラートID / 発生日時 / アラート種別 / 店舗 / スタッフ /
 トリガー値 / 通知ステータス

 フィルタ: 種別 / キーワード検索

 ※ アラートは毎日深夜に自動チェックされます
 ※ 該当するスタッフの店長・管理者にメール通知が送られます

  11. 設定
==============================================================================

■ 設定タブ
 システム全体の閾値やパラメータを管理します。

 【設定項目一覧】
  - 提案率アラート閾値 … %以下で店長へアラート（デフォルト: 20）
  - 成功率アラート閾値 … %以下でアラート（デフォルト: 15）
  - 提案ゼロ連続日数アラート … N日連続でアラート（デフォルト: 3）
  - 機会損失額アラート閾値 … 円以上でアラート（デフォルト: 30,000）
  - パフォーマンスランク_S閾値_提案率 … Sランク基準（デフォルト: 60%）
  - パフォーマンスランク_S閾値_成功率 … Sランク基準（デフォルト: 50%）
  - パフォーマンスランク_A閾値_提案率 … Aランク基準（デフォルト: 40%）
  - パフォーマンスランク_A閾値_成功率 … Aランク基準（デフォルト: 30%）
  - パフォーマンスランク_B閾値_提案率 … Bランク基準（デフォルト: 30%）
  - パフォーマンスランク_B閾値_成功率 … Bランク基準（デフォルト: 20%）
  - パフォーマンスランク_D閾値_提案率 … Dランク基準（デフォルト: 20%未満）
  - 月次レポート送信日 … 毎月N日に送信（デフォルト: 1）
  - 通知先メールアドレス（管理者）… 管理者宛通知先
  - 集計対象期間（月）… データ保持期間（デフォルト: 12ヶ月）

 設定値は「設定値」列を書き換えて「保存」ボタンで反映します。

 【ランク判定基準】
  S: 提案率≧S閾値 かつ 成功率≧S閾値
  A: 提案率≧A閾値 かつ 成功率≧A閾値
  B: 提案率≧B閾値 かつ 成功率≧B閾値
  D: 提案率＜D閾値
  C: 上記いずれにも該当しない場合

■ トリガー実行タブ
 通常は自動で実行されるトリガーを手動で実行できます。

 【実行ボタン一覧（11個）】

  - アラートチェック … 閾値チェック＆アラート生成

  - 月次レポート送信 … マネージャー向けHTML月次レポートを送信
  - 週次トレンド集計 … 前週の店舗別週次トレンドを集計
  - 育成効果トラッキング … 育成イベント前後の数値変化を計算
  - ランク再計算 … 全スタッフのランクを再計算

==============================================================================
  基本的な使い方の流れ
==============================================================================

【初期設定】
 1. 店舗管理で店舗を登録する
 2. スタッフ管理でスタッフを登録する（店舗・メールアドレスも入力）
 3. アップセル商品管理で提案対象の商品・メニューを登録する
 4. 設定タブでアラート閾値・ランク閾値を自社に合わせて調整する

【日常運用】
 1. 接客のたびに「接客記録」から提案内容と結果を登録する
 2. ダッシュボードで今月の全体傾向を確認する
 3. スタッフ別集計で個人の実績推移を確認する
 4. アラートが発生したら対象スタッフへのフォローを実施する

【月次レビュー】
 1. 月次集計で先月のスタッフ別・店舗別ランキングを確認する
 2. 商品分析で成功率の高い商品・低い商品を把握する
 3. 要指導スタッフへの面談・研修を計画する
 4. 育成イベントを登録して効果測定を開始する

【育成サイクル】
 1. 育成イベント管理で研修・指導を記録する
 2. 実施後14日以降に育成効果トラッキングが自動で数値を比較する
 3. ダッシュボードの育成効果サマリーで改善率を確認する
 4. 効果が出ていない場合は別のアプローチを検討する


==============================================================================
  パフォーマンスランクについて
==============================================================================

スタッフの提案率・成功率に基づき、S〜Dの5段階でランクが自動付与されます。

  ランク   条件（デフォルト）       意味
  ─────────────────────────────────────────────
    S      提案率≧60% かつ 成功率≧50%    模範スタッフ
    A      提案率≧40% かつ 成功率≧30%    優秀
    B      提案率≧30% かつ 成功率≧20%    標準
    C      上記以外                     要改善
    D      提案率＜20%                  要緊急フォロー

 ※ 閾値は設定タブで自由に変更できます
 ※ Dランクのスタッフには店長CCで強調通知メールが送信されます


==============================================================================
  機会損失額の計算ロジック
==============================================================================

「もしこのスタッフも提案していたら得られたであろう売上」を試算します。

  機会損失額 ＝ 未提案件数 × 店舗平均成功単価 × 店舗平均成功率

  例: 10件接客して3件しか提案しなかった場合
      未提案7件 × 平均成功単価3,000円 × 成功率40% ＝ 8,400円

この金額が「提案していれば得られた可能性のある追加売上」です。


==============================================================================
  よくある質問
==============================================================================

Q: 接客記録は1件ずつ入力するしかないですか？
A: Webアプリからの手動入力が基本です。スマートフォンからも入力可能です。

Q: 集計データがおかしいときはどうすればいいですか？
A: 設定 > トリガー実行タブから「日次集計実行」「月次集計実行」を手動実行して
   ください。最新の接客記録から再集計されます。

Q: ランクの基準を変えたいのですが？
A: 設定タブの「パフォーマンスランク_X閾値_提案率/成功率」を変更してください。
   変更後は「ランク再計算」トリガーを実行すると即反映されます。

Q: アラートメールが届かないのですが？
A: スタッフ管理の「メールアドレス」と店舗管理の「通知先メールアドレス」が
   正しく入力されているか確認してください。

Q: 古いデータは自動で削除されますか？
A: 毎月1日のデータクリーンアップで、設定した集計対象期間（デフォルト12ヶ月）
   を超えた古いデータがアーカイブされます。

Q: 複数店舗を同時に管理できますか？
A: はい。店舗管理で複数店舗を登録し、各スタッフに店舗を紐づけることで、
   店舗別の集計・比較が自動で行われます。
