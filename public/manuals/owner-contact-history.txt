====================================================
オーナー連絡履歴管理システム 使い方説明書
====================================================

----------------------------------------------------
1. このシステムでできること
----------------------------------------------------

不動産管理会社向けのオーナー連絡履歴管理システムです。
以下の業務を一元管理できます。

- オーナーとの連絡履歴の記録・管理（電話/メール/訪問/手紙/LINE/FAX/Zoom）
- 次回フォロー予定の登録・リマインド通知
- オーナープロファイル管理（性格/NGワード/こだわり等）
- 担当者交代時の引き継ぎサマリー自動生成（Googleドキュメント出力）
- 長期未連絡オーナーの自動検出・アラート通知
- オーナー別物件状況の紐付け表示
- 解約リスクスコア・関係値スコアの自動算出
- クレーム・要望の記録と対応状況管理
- 月次オーナー報告の管理
- 担当者別パフォーマンス集計・ランク判定
- リスクダッシュボードによる全体可視化

----------------------------------------------------
2. 画面構成（10タブ）
----------------------------------------------------

[ダッシュボード]
  システムの全体状況を一目で確認できます。
  - KPI表示: 取引中オーナー数/重要オーナー数/長期未連絡数/
             未完了フォロー数/期限超過数/リスク高オーナー数
  - 期限超過フォロー一覧（赤: 7日超過、オレンジ: 3日超過）
  - 本日のフォロー予定一覧
  - 長期未連絡オーナー一覧（経過日数順）
  - 解約リスク高オーナー一覧（スコア順）
  各オーナーをクリックするとオーナーサマリーに遷移します。

[連絡履歴記録]
  オーナーとの連絡内容を記録するフォームです。
  1. オーナーを検索（名前/カナ/ID/電話番号でインクリメンタルサーチ）
  2. オーナーを選択すると、NGワード/性格/こだわり等の
     プロファイル情報がバナーで表示されます
  3. 関連物件を選択（任意）
  4. 連絡日時/方法/方向/種別/重要度を入力
  5. 件名/内容詳細/オーナーの発言・反応を入力
  6. 合意事項があればフラグをONにして内容を記録
  7. 次回フォローが必要な場合はフラグをONにして予定日・内容を入力
     （フォロー予定シートに自動追加されます）
  8. 満足度(1-5)を任意入力
  記録すると、オーナーの最終連絡日が自動更新されます。

[連絡履歴一覧]
  過去の連絡履歴をタイムライン形式で閲覧できます。
  - オーナー名で絞り込み
  - 担当者で絞り込み
  - 連絡方法がアイコンで表示されます
  - 合意事項ありの連絡には「合意」バッジが表示されます
  - 重要度「高」には赤い「重要」バッジが表示されます
  - 満足度1-2の連絡は赤色で表示されます

[オーナーサマリー]
  特定のオーナーの全情報を一画面で確認できます。
  1. オーナーを検索して選択
  2. 以下の情報が表示されます:
     - 基本情報（連絡先/担当者/ステータス）
     - 関係値スコア/解約リスクスコア
     - プロファイル（性格/NGワード/こだわり/意思決定スタイル）
     - 保有物件一覧（空室日数/クレーム状況）
     - 直近10件の連絡履歴（タイムライン形式）
     - 未完了フォロー予定一覧
     - クレーム・要望履歴

[フォロー管理]
  フォロー予定の登録・管理・完了処理を行います。
  - 未対応: 未完了のフォロー予定を期限順で表示
  - 全件: 完了済み含む全フォロー予定を表示
  - 新規登録: フォロー予定を手動で追加
  - 「完了」ボタンで完了処理
  - 期限超過日数が赤く表示されます
  - 優先度(高/中/低)でバッジ表示

[クレーム・要望]
  オーナーからのクレーム・要望を記録・管理します。
  - 一覧: 全記録を受付日順で表示
  - 新規登録: クレーム/要望/提案/質問を記録
  - 種別/カテゴリ/重要度/オーナー納得度を入力
  - 対応方法/再発防止策を記入
  - ステータス管理（受付/対応中/完了）

[担当変更]
  オーナーの担当者を変更する処理です。
  - 変更登録: オーナーを選択し、新担当者と変更理由を入力
  - 実行すると以下が自動で行われます:
    * マスタのオーナー担当者情報が更新
    * 担当変更履歴に記録
    * 引き継ぎサマリー（Googleドキュメント）が自動生成
    * 関係者（前担当/新担当/管理者）にメール通知
  - 変更履歴: 過去の担当変更一覧を確認

[月次報告]
  オーナーへの月次報告を管理します。
  - 報告一覧: 全報告のステータスを確認
  - 新規登録: 報告年月/オーナー/内容を入力
  - 「送付済にする」ボタンでステータスを更新
  - 未送付の報告は黄色背景で強調表示

[マスタ管理]
  各種マスタデータの登録・管理を行います。
  - オーナー: 新規登録/一覧表示
  - 物件: 新規登録/一覧表示（オーナーと紐付け）
  - スタッフ: 新規登録/一覧表示（権限レベル設定）
  - 連絡種別: 種別一覧の確認

[設定]
  システム設定の確認・変更を行います。
  主な設定項目:
  - 長期未連絡アラート閾値（通常: 30日、重要: 14日、要注意: 7日）
  - フォローエスカレーション日数（3日超過で上長通知、7日超過で管理者通知）
  - 解約リスクスコアの重み設定（経過日数/クレーム/満足度/空室）
  - 月次報告リマインド日（毎月25日）
  - 引き継ぎサマリー自動生成フラグ
  - アラート通知先メールアドレス
  - Slack Webhook URL

  操作ボタン:
  - シート初期化: 全18シートをスプレッドシートに自動作成
  - トリガー設定: 自動処理（日次/週次/月次）のトリガーを設定
  - スコア再計算: 全オーナーの解約リスクスコアと関係値スコアを再計算

----------------------------------------------------
3. 自動処理（トリガー）
----------------------------------------------------

「設定」タブの「トリガー設定」ボタンで以下のトリガーが設定されます。

[毎日 9:00]
  - 長期未連絡オーナーアラート
    通常30日超/重要14日超/要注意7日超のオーナーを検出し、
    担当者にメール通知。50日超の場合は上長にエスカレーション。
  - フォロー期限リマインド
    本日フォロー予定のある担当者に通知。
    3日超過で上長に、7日超過で管理者にもエスカレーション。

[毎日 深夜1:00]
  - 解約リスクスコア一括再計算
    全取引中オーナーのリスクスコアを再計算。
    緊急レベル(80以上)のオーナーをSlack+メールで管理者に通知。

[毎週日曜 深夜1:00]
  - 関係値スコア更新
    全取引中オーナーの関係値スコアを再計算。
    先週比10ポイント以上下落したオーナーを担当者に通知。

[毎月1日 深夜2:00]
  - 連絡種別・傾向分析集計
    前月の連絡履歴を集計。
    オーナー別連絡サマリー/担当者別パフォーマンスを更新。
    ランクCのスタッフを管理者に通知。

[毎月25日 9:00]
  - 月次報告リマインド
    未送付の月次報告がある担当者にリマインド通知。

4. 色分け・アイコンの見方
----------------------------------------------------

[解約リスク]
  赤背景 = 緊急（スコア80以上）
  オレンジ背景 = 高（スコア60-79）
  黄色背景 = 中（スコア40-59）
  緑背景 = 低（スコア39以下）

[未連絡経過日数]
  赤背景 = 50日以上
  オレンジ背景 = 30日以上
  黄色背景 = 閾値超過

[フォロー期限]
  赤背景 = 7日以上超過
  オレンジ背景 = 3日以上超過
  黄色背景 = 当日

[バッジ]
  [重要] = 重要オーナー
  [要注意] = 要注意オーナー
  「合意」（赤バッジ）= 合意事項がある連絡
  「重要」（赤バッジ）= 重要度が高い連絡/クレーム

----------------------------------------------------
5. スコアの計算ロジック
----------------------------------------------------

[解約リスクスコア（0-100）]
  以下の4要素を重み付けして算出:
  - 最終連絡日からの経過日数（デフォルト重み: 40%）
    → 90日で100ポイント
  - 直近3ヶ月のクレーム件数（デフォルト重み: 20%）
    → 4件で100ポイント
  - 直近3ヶ月の平均満足度（デフォルト重み: 20%）
    → 満足度1で100ポイント、5で0ポイント
  - 空室状況（デフォルト重み: 20%）
    → 空室1件30pt + 長期空室(60日超)1件40pt

  リスク判定:
  80以上 = 緊急 → 推奨: 解約防止プラン提案
  60-79 = 高 → 推奨: 上長同席
  40-59 = 中 → 推奨: 訪問対応
  39以下 = 低

[関係値スコア（1-100）]
  - 連絡頻度（直近3ヶ月の件数x10、最大40pt）
  - 平均満足度（最大30pt）
  - フォロー完了率（最大20pt）
  - 基礎ポイント（10pt）
  - クレーム減点（1件あたり-5pt、最大-20pt）

----------------------------------------------------
6. 初回セットアップ手順
----------------------------------------------------

1. システムにアクセス
2. 「設定」タブを開く
3. 「シート初期化」ボタンをクリック
   → スプレッドシートに全18シートが自動作成されます
   → 初期設定値と連絡種別マスタが自動登録されます
4. 「トリガー設定」ボタンをクリック
   → 自動処理のトリガーが設定されます
5. 「マスタ管理」タブでスタッフを登録
6. 「マスタ管理」タブでオーナーを登録
7. 「マスタ管理」タブで物件を登録（オーナーと紐付け）
8. 設定タブで以下を必要に応じて変更:
   - アラート通知先メールアドレス
   - Slack Webhook URL
   - 各種閾値

----------------------------------------------------
7. シート構成（18シート）
----------------------------------------------------

[マスタ系]
  マスタ_オーナー: オーナー基本情報（41列）
  マスタ_オーナープロファイル: 性格/NGワード等の詳細情報（22列）
  マスタ_物件: 物件情報（13列）
  マスタ_スタッフ: スタッフ情報（9列）
  マスタ_連絡種別: 連絡種別マスタ（9列）
  マスタ_通知テンプレート: 通知テンプレート（10列）

[トランザクション系]
  トランザクション_連絡履歴: 連絡記録（29列）
  トランザクション_フォロー予定: フォロー予定（20列）
  トランザクション_担当変更履歴: 担当変更記録（17列）
  トランザクション_クレーム要望記録: クレーム/要望記録（19列）
  トランザクション_月次オーナー報告: 月次報告（18列）

[集計系]
  集計_オーナー別連絡サマリー: 月次オーナー別集計（18列）
  集計_担当者別パフォーマンス: 月次担当者別集計（16列）
  集計_オーナーリスク分析: リスク分析結果（16列）
  集計_連絡種別傾向分析: 連絡種別別集計（9列）

[分析系]
  分析_引き継ぎサマリー: 引き継ぎサマリー記録（21列）

[システム系]
  設定_システム設定: 各種設定値（8列）
  ログ_操作履歴: 操作ログ（10列）

----------------------------------------------------
8. 担当者パフォーマンスランク判定基準
----------------------------------------------------

  S: フォロー完了率90%以上 + 長期未連絡0件 + 平均満足度4.0以上
  A: フォロー完了率80%以上 + 長期未連絡1件以下 + 平均満足度3.5以上
  B: 上記以外（標準）
  C: フォロー完了率50%未満 または 長期未連絡5件以上
     → 管理者に自動通知されます
