============================================================
  来店客数カウント＋分析システム　使い方ガイド
============================================================

■ 画面の基本操作
  ・左のサイドバーからページを切り替えます
  ・スマホでは左上の「≡」メニューからサイドバーを開きます
  ・右上の店舗セレクタで「全店舗」または特定の店舗に絞り込めます
  ・データ登録時は確認ダイアログが表示されます
    OK → 実行 ／ キャンセル → 取り消し
  ・一覧が10件を超えるとテーブル下部に「前へ／次へ」が表示されます

============================================================
  各ページの使い方
============================================================

【ダッシュボード】

  開くと最初に表示される画面です。
  以下の情報をまとめて確認できます。

    - 本日の来店数（前日比つき）
    - 今月の累計来店数
    - 目標達成率
    - 平均客単価
    - 時間帯別来店グラフ（本日）
    - 直近7日の来店推移
    - 曜日別来店傾向
    - 客単価推移（7日）
    - 天気別来店傾向
    - 来店予測（今週）
    - シフト適正度（今週）
    - アラート一覧

  ※ 目標達成率を表示するにはKPI目標設定で当月の目標を
    登録しておく必要があります

──────────────────────────────────

【来店カウント入力】

  時間帯ごとの来店客数を記録します。

  1. 日付と店舗を選ぶ
  2. 時間帯のマスをタップして選択（青く光ります）
  3. ＋/−ボタンまたは数字直接入力で来店客数をセット
  4. 必要に応じて退店客数・滞在客数（ピーク）・備考を入力
  5.「来店カウントを登録」で保存

  ・日付や店舗を変更すると一覧が自動で切り替わります
  ・時間帯マスの数字は登録済みの合計です
  ・同じ日付・店舗・時間帯で再登録すると上書きされます

──────────────────────────────────

【売上データ入力】

  日ごとの売上データを記録します。

  入力 → 日付、店舗、総来店客数、購入/注文客数、
         売上金額、組数、目標売上
  →「売上データを登録」で保存

  一覧の購買率・客単価・目標達成率は自動計算です。

──────────────────────────────────

【時間帯別分析】

  対象月と店舗を選んで「表示」を押します。

  グラフの色分け：
    赤 = 来店が多い（ピークの80%以上）
    黄 = 中程度（ピークの50%以上）
    青 = 比較的少ない

  テーブルの内容：
    時間帯 / 来店数 / 売上 / 客単価 / スタッフ数 /
    1人あたり客数 / 回転率 / 構成比 / 前週比

──────────────────────────────────

【曜日別分析】

  年月と店舗を選んで「表示」を押します。

  グラフの色分け：土曜 = 青、日曜 = 赤、平日 = 青

  テーブルの内容：
    曜日 / 営業日数 / 平均来店 / 最大 / 最小 /
    標準偏差 / 平均売上 / 平均客単価 / 構成比 / 推奨スタッフ数

──────────────────────────────────

【天気・外部要因】

  天気やイベント情報と来店数の関係を記録します。
  来店予測の精度向上にも役立ちます。

  入力 → 日付、店舗、天気、気温、湿度、降水量、
         来店客数、イベント内容、近隣イベント、
         祝日、給料日、影響度コメント
  →「天気データを登録」で保存

  一覧では天気別の平均比が自動で表示されます。

──────────────────────────────────

【月次サマリー】

  年月と店舗を選んで「表示」を押します。

  上部カード：総来店客数 / 日平均 / 総売上 / 目標達成率
  グラフ：来店客数（棒）＋ 売上（折れ線）

  「月次レポート生成」→ 集計処理を手動で実行できます。

──────────────────────────────────

【シフト実績】

  日々のシフト実績を記録します。

  入力 → 日付、店舗、時間帯、予定スタッフ数、
         実績スタッフ数、来店客数、人件費、売上、備考
  →「シフト実績を登録」で保存

  一覧の適正判定は自動判定です：
    適正 … スタッフ数が適切
    過剰 … スタッフが多い
    不足 … スタッフが足りない

──────────────────────────────────

【シフト最適化提案】

  過去の実績をもとにしたシフト配置の提案を確認します。

  1. 店舗を選んで「表示」
  2. 各提案の「採用」ボタンでシフトに反映
  3.「提案を再生成」で最新データから提案を再作成

  表示内容：
    対象日 / 時間帯 / 予測来店数 /
    現在予定 → 推奨スタッフ数（差分）/
    調整提案 / 予測根拠

──────────────────────────────────

【来店予測】

  過去データに基づく来店予測を確認します。

  1. 店舗を選んで「表示」
  2.「予測を再生成」で最新データから予測を再計算

  表示内容：
    予測日 / 曜日 / 天気予報 / イベント有無 /
    過去平均（同曜日）/ 天気補正係数 /
    予測来店客数 / 予測売上 / 推奨スタッフ数 / 予測精度

  ※ データが蓄積されるほど予測精度が上がります

──────────────────────────────────

【KPI目標設定】

  月ごとの目標値を設定します。

  入力 → 店舗、年月、目標来店数（月間/日平均）、
         目標売上（月間）、目標客単価、
         目標購買率、目標人件費率
  →「KPI目標を保存」で保存

  一覧の実績値と達成率は自動計算です。
  ここで設定した目標がダッシュボードの達成率に反映されます。

──────────────────────────────────

【アラート設定】

  条件に応じた通知ルールを設定します。

  入力 → アラート名、
         条件（来店数急減 / 目標未達 / 人員過剰 / 人員不足）、
         条件値、対象店舗、通知先メールアドレス
  →「アラートを登録」で保存

  登録したアラートはダッシュボードにも表示されます。

──────────────────────────────────

【店舗管理】

  店舗情報を登録・編集します。

  入力 → 店舗名、業態、住所、営業開始/終了時間、
         定休日、座席数、店長名、
         目標来店数/日、目標売上/日、ステータス
  →「店舗を保存」で登録

  一覧の「編集」で既存店舗の情報を修正できます。
  ここで登録した店舗が各ページの店舗セレクタに表示されます。

──────────────────────────────────

【時間帯マスタ】

  営業時間帯の区分を登録・編集します。
  （例：朝、ランチ、アイドル、ディナー、深夜）

  入力 → 時間帯名、開始/終了時刻、
         種別（ピーク/通常/閑散）、標準必要人員数、備考
  →「時間帯を保存」で登録

  ここで登録した時間帯が来店カウント入力のマスや
  シフト実績の時間帯セレクタに反映されます。

──────────────────────────────────

【スタッフ管理】

  スタッフ情報を登録・編集します。

  入力 → 氏名、雇用形態（社員/パート/アルバイト）、
         担当店舗、時給/日給、勤務可能曜日、
         勤務可能時間帯、スキルレベル（1〜5）
  →「スタッフを保存」で登録

  一覧の「編集」で既存スタッフの情報を修正できます。

============================================================
  困ったときは
============================================================

  ● 店舗や時間帯のセレクタに選択肢がない
    → 先に「店舗管理」「時間帯マスタ」から登録してください

  ● データを登録したのに一覧に反映されない
    → 「更新」ボタンを押すか、サイドバーから
      同じページを再度タップしてください

  ● グラフが「データなし」と表示される
    → 対象期間にデータが未登録の状態です。
      データを入力するとグラフが表示されます

  ● 目標達成率が表示されない
    → KPI目標設定で当月の目標を登録してください
