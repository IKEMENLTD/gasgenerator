================================================================================
社内ナレッジ検索インデックスシステム 使い方説明書
================================================================================

■ このシステムでできること
================================================================================

社内に散在するマニュアル・FAQ・営業事例などのナレッジを、キーワード一発で
横断検索できるシステムです。GASが各シートを自動巡回してインデックスを構築し、
スコアリングで関連度の高い情報から順に表示します。

・複数シートに散在する情報をキーワードで一括検索
・カテゴリ・タグで分類して一覧化
・検索スコアリングで関連度順に表示
・閲覧数による人気コンテンツの可視化
・ゼロヒット分析で情報の不足領域を発見
・日次自動インデックス更新で常に最新状態を維持


■ 画面構成（6タブ）
================================================================================

1. ダッシュボード
2. ナレッジ検索
3. ソース管理
4. 検索ログ
5. カテゴリ管理
6. ユーザー管理


■ ダッシュボード
================================================================================

システム全体の状況を5つの指標で可視化します。

□ インデックス総件数
  有効なナレッジインデックスの総数を表示します。

□ ゼロヒット率（当週）
  今週の検索のうち、結果が0件だった割合を表示します。
  20%を超えると警告表示になります。
  ゼロヒット率が高い場合はナレッジの追加を検討してください。

□ カテゴリ別件数（円グラフ）
  ナレッジのカテゴリ別分布を表示します。
  偏りがある場合はカテゴリの見直しを検討してください。

□ 週間検索数推移（折れ線グラフ）
  直近4週間の検索回数の推移を表示します。

□ 閲覧数TOP10コンテンツ（横棒グラフ）
  最も閲覧されているナレッジのランキングを表示します。


■ ナレッジ検索
================================================================================

□ キーワード検索
  検索ボックスにキーワードを入力して「検索」ボタンをクリックします。
  スペース区切りで複数キーワードを指定できます。

  検索結果はスコア順に表示されます。
  スコアの計算方法:
    ・タイトルにキーワードが含まれる → +10点
    ・タグにキーワードが含まれる　　 → +5点
    ・本文にキーワードが含まれる　　 → +1点
    ・閲覧数 × 0.1　　　　　　　　　→ 人気補正
  最大50件まで表示されます。

□ カテゴリフィルタ
  右上のドロップダウンでカテゴリを選択すると、
  一覧・検索結果をカテゴリで絞り込めます。

□ タイトルクリック
  タイトルをクリックすると詳細モーダルが表示されます。
  クリック時に閲覧数が自動的に+1されます。

□ インデックス再構築ボタン
  手動でインデックスを即時再構築します。
  ソースシートにデータを追加・変更した直後に使用してください。
  （通常は毎日02:00に自動再構築されます）

□ CSV出力
  現在の表示内容（フィルタ適用後）をCSVファイルでダウンロードします。
  ファイル名: knowledge_YYYYMMDD.csv

□ 一覧表示（未検索時）
  検索していない状態では全インデックスが閲覧数の多い順に表示されます。


■ ソース管理
================================================================================

インデックス対象のシートを登録・管理する画面です。

□ ソースの追加
  「+ 新規追加」ボタンをクリックし、以下を入力します。
    ・ソースシート名: インデックス対象のシート名（スプシのシート名と完全一致）
    ・タイトル列: タイトルとして取得する列記号（例: B）
    ・本文列: 本文として取得する列記号（複数列はカンマ区切り。例: C,D,E）
    ・カテゴリ: マニュアル / FAQ / 営業事例 / 規程・ルール / テンプレート / その他
    ・タグ列: タグとして取得する列記号（任意）
    ・登録者列: 登録者として取得する列記号（任意）
    ・更新日列: 更新日として取得する列記号（任意）
    ・有効フラグ: 有効 / 無効
    ・備考: 自由記入

□ ソースの編集・削除
  一覧の「編集」「削除」ボタンから操作できます。

□ 自動更新情報
  「最終インデックス日時」「取得件数」はGASが自動更新します。
  1日以上更新されていないソースはオレンジ色で警告表示されます。

□ 新しいナレッジシートを追加する手順
  1. スプレッドシートに新しいシートを作成してデータを入力
     （1行目はヘッダー行にする）
  2. ソース管理で「+ 新規追加」からシート名・列を登録
  3. ナレッジ検索タブで「インデックス再構築」を実行


■ 検索ログ
================================================================================

すべての検索履歴が自動記録されます（手動操作不要）。

□ 表示項目
  ・ログID: 自動採番
  ・検索日時: 検索が実行された日時
  ・検索キーワード: 入力されたキーワード
  ・ヒット件数: 検索結果の件数
  ・検索者: Googleアカウント（自動取得）
  ・クリックされた結果: クリックされたナレッジのID
  ・ゼロヒットフラグ: ヒット0件の場合TRUE（赤バッジ表示）

□ フィルタ・検索
  テキスト検索、日付範囲（FROM/TO）で絞り込みできます。

□ CSV出力
  フィルタ適用後のログをCSVでダウンロードできます。
  ファイル名: searchlog_YYYYMMDD.csv


■ カテゴリ管理
================================================================================

ナレッジのカテゴリを管理します。

□ 登録項目
  ・カテゴリ名: マニュアル / FAQ / 営業事例 / 規程・ルール / テンプレート / その他
  ・説明: カテゴリの用途説明
  ・担当部署: 総務 / 営業 / 経理 / 全社 / その他
  ・有効フラグ: 有効 / 無効

□ 編集・削除
  一覧の「編集」「削除」ボタンから操作できます。


■ ユーザー管理
================================================================================

システムのユーザーを管理します。

□ 登録項目
  ・氏名: ユーザーの名前（必須）
  ・メールアドレス: Googleアカウントのメールアドレス（必須）
  ・部署: 総務 / 営業 / 経理 / 全社 / その他
  ・役割: 管理者 / 編集者 / 閲覧者
  ・有効フラグ: 有効 / 無効

□ 管理者の役割
  ・役割が「管理者」かつ有効フラグが「有効」のユーザーに、
    週次ゼロヒットレポートと日次ヘルスチェックレポートがメール送信されます。
  ・メール送信先を変更したい場合は、ユーザー管理で管理者を追加・変更してください。


■ 自動処理（トリガー）
================================================================================

以下のトリガーが自動設定されます。手動操作は不要です。

□ T1: インデックス自動再構築（毎日 02:00）
  有効なソースシートを全件走査し、検索インデックスを再構築します。

□ T5: ゼロヒット週次レポート（毎週月曜 08:00）
  直前7日間の検索ログから、ゼロヒット率と頻出キーワードTOP10を
  管理者にメール送信します。

□ T6: インデックスヘルスチェック（毎日 09:00）
  インデックスの有効/無効件数、エラーソース、未更新ソースを
  管理者にメール送信します。


■ 運用のポイント
================================================================================

□ ナレッジの追加
  スプレッドシートのソースシートにデータを追加するだけで、
  翌日02:00に自動的にインデックスに反映されます。
  すぐに反映したい場合は「インデックス再構築」ボタンを使用してください。

□ ゼロヒット率の活用
  ゼロヒット率が高い場合、ユーザーが求める情報が不足しています。
  週次レポートのゼロヒットキーワードTOP10を参考に、
  該当するナレッジを追加してください。

□ カテゴリの活用
  ナレッジが増えてきたらカテゴリを適切に設定することで、
  フィルタ検索がしやすくなります。

□ 閲覧数の活用
  閲覧数TOP10を参考に、よく参照されるナレッジを
  優先的にメンテナンスしてください。


■ スプレッドシート構成（5シート）
================================================================================

1. 検索インデックス（13列）
   GASが自動構築。手動編集禁止。

2. インデックス対象ソース（12列）
   インデックス対象シートの設定を管理。

3. 検索ログ（7列）
   GASが自動記録。手動編集禁止。

4. カテゴリマスタ（6列）
   カテゴリの定義を管理。

5. ユーザーマスタ（7列）
   ユーザー情報と役割を管理。

※ 上記5シートのほかに、ナレッジ元データのシート（マニュアル集、FAQ集など）が
   必要です。これらはソース管理で登録してください。
