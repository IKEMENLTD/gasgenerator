============================
報告テンプレート強制 使い方説明書
============================

■ 概要
報告のフォーマットを統一し、必要な情報の抜け漏れを防ぐシステムです。
テンプレート別の入力強制、品質スコアの自動計算、提出状況の可視化、
未提出者への自動リマインドまで一元管理できます。

============================
■ 画面構成（8ページ）
============================

1. ダッシュボード
   - 今日の提出数、今月の提出数、未承認件数、差戻し件数、平均品質スコア
   - 提出数推移グラフ（過去7日・折れ線）
   - 部署別提出率（今日・横棒グラフ）
   - 品質スコア分布（今月・ドーナツグラフ）
   - 提出数ランキング（今月・トップ5）
   - 今日の提出状況（テンプレート別）
   - 差戻し・再提出一覧
   - よくある不備項目TOP10（今月）
   - 未提出アラート一覧
   - 連続未提出者アラート

2. 報告入力・提出
   - テンプレートに沿った報告を入力・提出する画面
   - 必須項目のバリデーション、品質スコアの自動計算

3. 報告一覧・承認
   - 全報告の一覧表示
   - テンプレート・ステータス・キーワードでフィルタリング
   - 報告詳細の確認、承認・差戻し操作

4. テンプレート管理
   - テンプレート一覧：報告種別（日報/週報/商談報告等）の登録・編集・削除
   - 項目管理：テンプレートごとの入力項目（必須/任意、文字数制限等）の管理

5. 報告者管理
   - 報告者の登録・編集・削除
   - 必須テンプレートの割り当て、上長・通知方法の設定

6. アラート・リマインド
   - 未提出アラート：日次チェック結果の確認
   - アラート設定：通知条件の登録・編集・削除
   - リマインド履歴：送信済みリマインドの確認

7. レポート
   - 月次サマリーの生成
   - 報告者別サマリー（提出率/期限内率/品質/ランク）
   - 部署別サマリー（提出率/期限内率/品質/ランク）
   - テンプレート別サマリー（提出率/品質/よくある不備）

8. 設定
   - フィードバックテンプレート：承認・差戻し時に使うフィードバック文面の管理
   - トリガー情報：自動実行トリガーの確認

============================
■ 基本操作
============================

【報告を提出する】
1. サイドメニューの「報告入力・提出」をクリック
2. 報告者、テンプレート、対象日を選択
3. テンプレートの入力フォームが表示される
4. 各項目を入力（必須項目は入力しないと提出できません）
5. 「提出する」をクリック
6. 品質スコアが自動計算されて提出完了

【報告を承認・差戻しする】
1. 「報告一覧・承認」を開く
2. フィルターで絞り込み可能（テンプレート/ステータス/キーワード）
3. 「詳細」ボタンで報告内容を確認
4. 承認の場合：フィードバック（任意）を入力し「承認」をクリック
5. 差戻しの場合：差戻し理由（必須）を入力し「差戻し」をクリック
   → 報告者にメール通知が送信されます

【承認ステータスの流れ】
提出 → 未承認 → 承認済
                → 差戻し → 再提出
※ 承認不要のテンプレートは「不要」ステータスになります

【品質スコアの仕組み】
品質スコアは以下の3要素で自動計算されます（100点満点）：
- 必須項目充足率 … 50点（必須項目をすべて入力で満点）
- 文字数充足率 … 30点（最小文字数に対する実入力文字数の割合）
- 期限遵守 … 20点（期限内提出で20点、超過で0点）

ランク：S(90点以上) / A(80点以上) / B(70点以上) / C(60点以上) / D(60点未満)

============================
■ テンプレート管理
============================

【テンプレートを作成する】
1. 「テンプレート管理」→「テンプレート一覧」タブ
2. 「+ 新規テンプレート」をクリック
3. テンプレート名、カテゴリ（定期/都度/緊急）、対象者、提出頻度、
   提出期限、承認要否を設定
4. 「保存」をクリック

【テンプレート項目を設定する】
1. 「テンプレート管理」→「項目管理」タブ
2. テンプレートをフィルタで選択
3. 「+ 新規項目」をクリック
4. 項目名、入力形式（テキスト/数値/日付/選択/複数選択）、
   必須/任意、最小・最大文字数、選択肢を設定
5. 「保存」をクリック
※ 表示順は自動採番されます

【提出期限の書式】
テンプレートの提出期限は以下の形式で記入できます：
- 「当日18時」→ 対象日の18時が期限
- 「翌営業日10時」→ 対象日の翌営業日10時が期限（土日スキップ）
- 「発生後2時間以内」→ 対象日9時起点で2時間後が期限

============================
■ 報告者管理
============================

【報告者を登録する】
1. 「報告者管理」をクリック
2. 「+ 新規報告者」をクリック
3. 氏名、部署（必須）、役職、メールアドレス、Slack ID、
   必須テンプレート一覧（カンマ区切り：例 TPL001,TPL002）、
   報告先（上長ID）、報告先名、通知方法を入力
4. 「保存」をクリック

※ 必須テンプレート一覧に設定したテンプレートが、
  日次チェック時の未提出判定の対象になります

============================
■ アラート・リマインド
============================

【日次チェックを手動実行する】
1. 「アラート・リマインド」→「未提出アラート」タブ
2. 「日次チェック実行」をクリック
3. 前日の未提出者が検出され、アラートリストに追加
4. 未提出者にはメールでリマインドが送信されます

※ 通常は毎日9時にトリガーで自動実行されます

【アラート設定を管理する】
1. 「アラート設定」タブ
2. 「+ 新規アラート」で条件（未提出/期限超過/品質低下/連続未提出）、
   通知先（本人/上長/管理者）、有効/無効を設定

============================
■ レポート
============================

【月次サマリーを生成する】
1. 「レポート」をクリック
2. 月選択ドロップダウンで対象月を選択
3. 「月次サマリー生成」をクリック
4. 報告者別・部署別・テンプレート別のサマリーが生成

※ 同じ月を再生成すると、既存データが上書きされます（重複しません）
※ 毎月1日8時にトリガーで前月分が自動生成されます

【サマリーの見方】
- 報告者別：個人ごとの提出率、期限内率、品質スコア、差戻し回数、ランク
- 部署別：部署ごとの提出率、期限内率、品質スコア、ランク
- テンプレート別：テンプレートごとの提出率、品質スコア、よくある不備項目

============================
■ 設定
============================

【フィードバックテンプレートを管理する】
1. 「設定」→「フィードバックテンプレート」タブ
2. 「+ 新規フィードバック」で種別（称賛/改善指摘/差戻し理由）、
   対象項目、フィードバック文面を登録
※ 承認・差戻し時のコメントに活用できます

【自動トリガー】
以下のトリガーが自動設定されています：
- dailyCheck（毎日9時）：前日の提出状況チェック・未提出アラート・リマインド送信
- weeklyReport（毎週月曜10時）：管理者向け週次レポートメール送信
- monthlyReport（毎月1日8時）：月次サマリー自動生成

============================
■ サンプルデータ
============================

初回アクセス時に以下のサンプルデータが自動登録されます：

テンプレート（6種）：
- TPL001 日報（毎日/当日18時/承認なし）
- TPL002 週報（毎週/翌営業日10時/承認なし）
- TPL003 商談報告（都度/翌営業日10時/承認あり）
- TPL004 進捗報告（都度/翌営業日10時/承認あり）
- TPL005 障害報告（都度/発生後2時間以内/承認あり）
- TPL006 出張報告（都度/帰社後翌営業日/承認なし）

テンプレート項目（15項目）：日報5項目、商談報告5項目、障害報告5項目

報告者（8名）：
- 営業部：田中太郎、鈴木花子、佐藤部長
- 開発部：高橋一郎、山本二郎、山田部長
- 管理部：中村三郎、伊藤部長

アラート設定（4種）：未提出リマインド、期限超過エスカレーション、品質低下アラート、連続未提出警告

フィードバックテンプレート（4種）：称賛、文字数不足指摘、差戻し理由、具体性不足指摘

============================
■ シート構成（14シート）
============================

1. テンプレートマスタ（青）
2. テンプレート項目マスタ（青・やや明るい）
3. 報告者マスタ（緑）
4. 報告データ（赤）
5. 報告内容詳細（赤・やや明るい）
6. 報告提出状況（日次）（オレンジ）
7. 報告者別サマリー（月次）（紫）
8. 部署別サマリー（月次）（紫・やや明るい）
9. テンプレート別サマリー（月次）（紫・明るい）
10. 品質チェック履歴（ティール）
11. リマインド履歴（ディープオレンジ）
12. 未提出アラートリスト（ディープオレンジ・暗い）
13. アラート設定（アンバー）
14. フィードバックテンプレート（ライトブルー）

============================
■ 注意事項
============================

- スプレッドシートのフィルターを使用している場合、列を追加した後は
  フィルター範囲を再設定してください（データ→フィルタを削除→フィルタを作成）
- テンプレートの必須項目が未入力の場合、報告は提出できません
- 月次サマリーの年月列はテキスト形式で保存されます（日付型に変換しないでください）
- メール通知は報告者マスタに登録されたメールアドレス宛に送信されます
