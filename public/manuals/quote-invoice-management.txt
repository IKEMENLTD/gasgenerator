================================================================================
見積→請求 通貫管理 使い方説明書
================================================================================

■ このシステムでできること
================================================================================
見積書の作成から受注登録、請求書の発行・入金管理までを一つの画面で完結できる
管理システムです。

・見積書を作成し、承認フローを経て受注に自動変換
・受注ごとの原価・粗利を自動計算し、収支を見える化
・請求書をボタン1つでPDF生成し、Googleドライブに自動保存
・入金額を入力するだけで入金ステータスを自動判定
・ダッシュボードで受注金額推移・粗利率・入金状況をグラフ表示

対象: BtoB全般・制作会社・コンサルなど


■ 画面構成（5タブ）
================================================================================
1. ダッシュボード  … 全体の集計・グラフ（閲覧のみ）
2. 見積管理        … 見積の作成・明細追加・承認・受注変換
3. 受注管理        … 受注の編集・原価入力・請求書作成
4. 請求管理        … 請求の編集・入金処理・PDF発行
5. マスタ管理      … 顧客マスタ・ユーザーマスタの登録・編集


■ 初期設定（最初に行うこと）
================================================================================

1. マスタ管理 → ユーザーマスタ → 「+ 新規ユーザー」でユーザーを登録
   - 氏名、メールアドレス、役割（管理者/営業/経理/閲覧者）を入力
   - 見積の承認を行うユーザーは「承認権限」を「承認可」に設定

2. マスタ管理 → 顧客マスタ → 「+ 新規顧客」で取引先を登録
   - 顧客名、顧客区分、業種、担当者名、住所等を入力
   - 支払サイト（日）はデフォルト30日（請求書の支払期日に反映されます）


■ 見積管理の使い方
================================================================================

【見積を作成する】
1. 見積管理 → 「+ 新規見積」をクリック
2. 顧客をドロップダウンから選択（顧客マスタから取得）
3. 件名、消費税率（8%/10%）、作成者、承認者を入力
4. 有効期限は自動で作成日+30日がセットされます（変更可）
5. 「保存」をクリック → 見積ID（例: Q20260318-001）が自動採番されます

【見積明細を追加する】
1. 「見積明細」タブに切り替え
2. ドロップダウンで見積を選択
3. 「+ 明細追加」をクリック
4. 商品・サービス名、数量、単位、単価（税抜）、割引率を入力
5. 「保存」をクリック
   → 金額（税抜）= 数量 × 単価 が自動計算
   → 割引後金額 = 金額 × (1 - 割引率/100) が自動計算
   → 見積一覧の合計（税込）が自動更新されます

【承認フロー】
1. 見積一覧で「承認依頼」ボタンをクリック → ステータスが「承認待ち」に
2. 承認者が「承認」ボタンをクリック → ステータスが「受注済」に
   → 受注データが自動で作成されます
3. 「却下」ボタンで却下も可能 → 却下後は再編集できます

※ 承認すると自動的に受注に変換されます。見積の受注変換フラグも「変換済」に
  なります。


■ 受注管理の使い方
================================================================================

【受注データの確認・編集】
- 見積を承認すると受注が自動作成されます（手動作成は不要）
- 受注一覧で「編集」ボタンから以下を入力できます:
  - ステータス（進行中/完了/キャンセル）
  - 納品予定日、実納品日
  - 担当者（ユーザーマスタから選択）
  - 原価合計

【粗利の自動計算】
- 原価合計を入力すると以下が自動計算されます:
  - 粗利 = 受注金額（税込）- 原価合計
  - 粗利率 = 粗利 ÷ 受注金額 × 100（小数第2位まで）

【請求書の作成】
- 受注一覧で「請求書作成」ボタンをクリック
  → 請求データが自動作成されます
  → 受注の「請求済」フラグが「請求済」に変わります


■ 請求管理の使い方
================================================================================

【請求データの確認】
- 受注から請求書を作成すると、以下が自動セットされます:
  - 請求金額（税込）= 受注金額
  - 支払期日 = 請求日 + 顧客の支払サイト日数（デフォルト30日）
  - 支払方法 = 銀行振込（デフォルト）
  - 入金ステータス = 未入金

【入金処理】
1. 請求一覧で「編集」をクリック
2. 入金日と入金額を入力して「保存」
3. 入金ステータスが自動で判定されます:
   - 差額0円かつ入金あり → 「入金済」
   - 差額あり（正）かつ入金あり → 「部分入金」
   - 差額ありかつ入金なしかつ支払期日超過 → 「延滞」
   - それ以外 → 「未入金」

【PDF請求書の発行】
1. 請求一覧で「PDF発行」ボタンをクリック
2. Googleドライブの「請求書PDF」フォルダにPDFが自動保存されます
3. PDFには以下が含まれます:
   - 請求番号、請求日
   - 顧客情報（名前、住所、担当者）
   - 件名、明細テーブル（商品名、数量、単位、単価、金額）
   - 小計、消費税、合計（税込）
   - 支払方法、支払期日


■ ダッシュボードの見方
================================================================================

【統計カード（画面上部）】
- 見積件数: 登録されている見積の総数
- 受注件数: 登録されている受注の総数
- 未入金件数: 入金ステータスが「入金済」以外の請求数
- 今月受注金額: 今月の受注金額合計
- 平均粗利率: 全受注の粗利率の平均値

【グラフ（5種類）】
1. 月別受注金額推移（折れ線グラフ）
   - 月ごとの受注金額合計の推移
2. 見積ステータス別件数（ドーナツグラフ）
   - 下書き・承認待ち・承認済・却下・受注済の件数内訳
3. 入金ステータス別請求額（積上棒グラフ）
   - 未入金・入金済・部分入金・延滞ごとの請求金額
4. 案件別粗利率TOP10（横棒グラフ）
   - 粗利率が高い案件の上位10件
5. 顧客別累計受注金額（縦棒グラフ）
   - 顧客ごとの受注金額累計（全期間）


■ マスタ管理の使い方
================================================================================

【顧客マスタ】
- 「+ 新規顧客」で取引先を登録
- 顧客区分: 法人 / 個人事業主 / 個人
- 業種: 制作 / コンサル / IT / 製造 / 小売 / サービス / その他
- 支払サイト（日）: 請求書の支払期日計算に使用
- 有効フラグ: 無効にすると見積作成時の選択肢に表示されなくなります
- フィルタ: 顧客区分、有効/無効、キーワードで絞り込み可能

【ユーザーマスタ】
- 「+ 新規ユーザー」でシステム利用者を登録
- 役割: 管理者 / 営業 / 経理 / 閲覧者
- 承認権限: 「承認可」のユーザーのみ見積の承認者として選択可能
- 有効フラグ: 無効にするとドロップダウンに表示されなくなります
- フィルタ: 役割、キーワードで絞り込み可能


■ CSV出力
================================================================================
見積管理・受注管理・請求管理の各画面で「CSV出力」ボタンをクリックすると、
現在のフィルタ条件に一致するデータをCSVファイルとしてダウンロードできます。

- ファイル名: quotes_YYYYMMDD.csv / orders_YYYYMMDD.csv / invoices_YYYYMMDD.csv
- 文字コード: UTF-8（BOM付き、Excelで文字化けしません）
- 出力対象: フィルタ・検索で絞り込んだ結果のみ


■ 自動処理（バックグラウンド）
================================================================================
以下の処理が自動で実行されます:

- 日次集計（毎日7:00）: ダッシュボード用の集計データを更新
- 月次レポート（毎月1日8:00）: 前月の見積・受注・請求・入金の集計


■ ID採番ルール
================================================================================
- 見積ID: Q + 日付8桁 + 連番3桁（例: Q20260318-001）
- 見積明細ID: QI + 日付8桁 + 連番3桁（例: QI20260318-001）
- 受注ID: O + 日付8桁 + 連番3桁（例: O20260318-001）
- 請求ID: INV + 日付8桁 + 連番3桁（例: INV20260318-001）
- 顧客ID: C + 連番3桁（例: C001）
- ユーザーID: U + 連番3桁（例: U001）


■ よくある質問
================================================================================

Q. 見積を間違えて承認してしまった場合は？
A. 承認すると自動で受注に変換されます。受注管理で該当受注を削除し、
   スプレッドシートで見積のステータスを「下書き」に戻してください。

Q. 請求書のPDFはどこに保存されますか？
A. Googleドライブのルートフォルダ内の「請求書PDF」フォルダに保存されます。

Q. 消費税率を変更したい場合は？
A. 見積の編集画面で消費税率（8%/10%）を変更すると、合計が自動再計算されます。

Q. 入金を間違えて入力した場合は？
A. 請求の編集画面で入金額を修正して保存すれば、入金ステータスと差額が
   自動で再計算されます。

Q. データをエクスポートしたい場合は？
A. 各管理画面の「CSV出力」ボタンでCSVファイルをダウンロードできます。
   フィルタを設定してから出力すると、絞り込んだデータのみ出力されます。
