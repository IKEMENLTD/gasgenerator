=============================================
解約申し出後のウィンバック管理 - 使い方説明書
=============================================

■ このツールでできること
・解約申し出を受けた際の対応を標準化し、対応漏れを防止できます
・解約理由を分類・蓄積し、理由の傾向把握やプロダクト改善に活かせます
・ウィンバック率（引き止め成功率）をKPIとして計測・管理できます
・引き止め施策ごとの効果を自動集計し、最も効果的な施策を把握できます
・月次レポートで解約理由トレンド・MRR影響・施策効果を自動配信できます


=============================================
■ 画面一覧（全11画面）
=============================================

1. ダッシュボード
2. 解約申し出
3. 引き止め対応
4. 顧客マスタ
5. プランマスタ
6. 解約理由マスタ
7. 施策マスタ
8. 担当者マスタ
9. レポート（6タブ）
10. 設定
11. トリガー管理


=============================================
■ 各画面の使い方
=============================================

--------------------------------------------
1. ダッシュボード
--------------------------------------------
システムを開くと最初に表示される画面です。
現在の対応状況とKPIを一目で把握できます。

【表示項目】
・未対応 / 対応中 / 期限超過 / 今月申し出 / 今月WB率
・今月引き止め成功 / 今月解約確定 / MRR純損失
・解約理由Top5（今月）… ドーナツチャート
・月次トレンド … 申し出件数・解約確定件数（棒グラフ）+ WB率（折れ線）
・期限超過一覧 … 対応期限を過ぎた申し出の一覧
・高単価・VIP顧客 … VIP/上位ランクで対応中の申し出一覧
・担当者別ウィンバック率（今月）… 順位・対応件数・成功数・WB率・防止MRR
・施策効果ランキング（最新月）… 施策名・実施件数・成功率・防止MRR・ROI
・プラン別解約比較 … プランごとの申し出件数・WB率
・解約理由Top5（累計）… 全期間の理由別件数

※ MRR純損失は「解約確定MRR + ダウングレード差額」で計算されます。


--------------------------------------------
2. 解約申し出
--------------------------------------------
解約の申し出があった際に登録・管理する画面です。

【新規登録の手順】
1. 「+ 新規登録」ボタンをクリック
2. 以下の項目を入力します：
   - 顧客（必須）… ドロップダウンから選択。MRR・ランク・担当CSMが自動入力されます
   - 申し出日（必須）… 解約の申し出を受けた日付
   - 解約希望日 … 顧客が希望する解約日（任意）
   - 申し出チャネル … 電話/メール/フォーム/窓口来店/その他
   - 主理由（必須）… ドロップダウンから選択。選択すると推奨施策が表示されます
   - 副理由 … 副次的な理由がある場合に選択
   - 担当CSM … 対応する担当者を選択
   - 理由詳細 … 自由記述で詳細を入力
   - 備考
3. 「保存」をクリック

【自動処理】
・対応期限日が自動計算されます（申し出日 + 設定の対応期限日数）
・顧客マスタのステータスが「解約申し出中」に自動更新されます
・担当CSMと管理者に通知メールが送信されます（推奨施策情報付き）
・VIP顧客や高単価顧客の場合はエスカレーションアラートが追加されます

【編集時の結果入力】
・一覧から「編集」をクリックすると結果入力欄が表示されます
・結果を選択して保存すると以下が自動処理されます：
  - 継続/ダウングレード継続/一時停止 → ステータス「引き止め成功」
  - 解約/無応答解約 → ステータス「解約確定」
  - 顧客マスタのステータスも連動更新（ウィンバック済/解約済）
  - 引き止め成功時は管理者に成功通知メール送信

【一覧の機能】
・ステータスフィルタ … 未対応/対応中/引き止め成功/解約確定/期限切れで絞り込み
・検索 … フリーワードで検索
・新しい順に表示されます


--------------------------------------------
3. 引き止め対応
--------------------------------------------
解約申し出に対する対応記録を管理する画面です。

【対応記録の登録手順】
1. 「+ 対応記録追加」ボタンをクリック
2. 以下の項目を入力します：
   - 申し出ID（必須）… 対応する解約申し出を選択
   - 担当者（必須）… 対応した担当者を選択
   - 対応日時 … 対応した日時
   - 対応ステップ … 初回連絡/理由深掘りヒアリング/提案/クロージング/結果確認/その他
   - 対応手段 … 電話/メール/オンライン商談/対面/その他
   - 実施施策（複数可）… チェックボックスで複数の施策を選択可能
   - 提案内容詳細 … 具体的な提案内容を記述
   - 顧客反応 … 前向き/中立/否定的/応答なし
   - 次回アクション予定日 … 次に対応する予定日
   - 次回アクション内容 … 次回のアクション内容
   - 備考
3. 「保存」をクリック

【自動処理】
・対応記録を初めて登録すると、申し出のステータスが「未対応」→「対応中」に自動更新されます
・最終対応日が自動更新されます

【一覧の機能】
・申し出IDフィルタ … 特定の申し出に絞り込み
・検索 … フリーワード検索
・新しい順に表示されます


--------------------------------------------
4. 顧客マスタ
--------------------------------------------
顧客情報を管理するマスタです。

【登録項目】
・会社名 / 氏名（必須）
・プラン … プランマスタから選択
・契約開始日
・契約形態 … 月次/年次/複数年
・MRR（月次経常収益・円）
・累計契約月数
・担当CSM … 担当者マスタから選択
・担当営業 … 担当者マスタから選択
・業種 … SaaS/小売/製造/医療/保険/携帯/教育/その他
・顧客ランク … 標準/VIP/上位/低単価
・ステータス … 契約中/解約申し出中/ウィンバック済/解約済
・備考

【一覧の機能】
・ステータスフィルタで絞り込み
・検索機能

※ ステータスは解約申し出登録・結果入力時に自動更新されます。


--------------------------------------------
5. プランマスタ
--------------------------------------------
契約プラン情報を管理するマスタです。

【登録項目】
・プラン名（必須）
・月額料金（円）
・年額料金（円）
・プラン種別 … 基本/上位/プレミアム
・ダウングレード先プラン … ドロップダウンから選択（引き止め提案用）
・備考


--------------------------------------------
6. 解約理由マスタ
--------------------------------------------
解約理由を分類・管理するマスタです。

【登録項目】
・理由カテゴリ（必須）… 価格/機能不足/競合乗り換え/利用頻度低下/サービス不満/事業縮小/担当変更/その他
・理由詳細 … 具体的な内容
・推奨引き止め施策 … ドロップダウンから複数選択可能（チェックボックス）
  → 解約申し出登録時にこの理由を選ぶと、推奨施策が自動表示されます
・対応難易度 … 低/中/高
・備考


--------------------------------------------
7. 施策マスタ
--------------------------------------------
引き止め施策を管理するマスタです。

【登録項目】
・施策名（必須）
・施策種別 … 価格割引/プラン変更/サポート強化/機能追加約束/特典提供/オンボーディング再実施/担当変更/その他
・費用・原資（円または%引き）
・利益率への影響 … 低/中/高
・推奨対象理由 … ドロップダウンから複数選択可能（チェックボックス）
・備考


--------------------------------------------
8. 担当者マスタ
--------------------------------------------
対応担当者を管理するマスタです。

【登録項目】
・氏名（必須）
・メールアドレス … 通知メールの送信先になります
・部署 … CS/営業/カスタマーサポート
・ステータス … 在籍中/休職/退職

※ ステータスが「在籍中」の担当者のみ、各フォームの選択肢に表示されます。


--------------------------------------------
9. レポート
--------------------------------------------
月次集計データを確認する画面です。6つのタブがあります。

【サマリー】
・月ごとの申し出件数・MRR合計・引き止め成功件数・WB率・解約確定件数・MRR純損失など

【理由別】
・月ごとの解約理由別の件数・構成比・WB率・対象MRR・前月比変化

【施策効果】
・月ごとの施策別の実施件数・成功件数・成功率・費用合計・防止MRR・ROI・平均対応回数

【担当者別】
・月ごとの担当者別の対応件数・成功件数・WB率・平均対応日数・期限切れ件数・防止MRR

【プラン別】
・月ごとのプラン別の申し出件数・WB率・解約確定件数・ダウングレード件数・主要解約理由

【トレンド】
・月ごとの申し出件数・WB率・MRR・前月比変化・期限切れ率

※ データは月次集計トリガーの実行後に表示されます。


--------------------------------------------
10. 設定
--------------------------------------------
システムの動作を制御する設定値を管理します。

【設定項目】
・対応期限日数 … 申し出からN日以内に初回対応必須（初期値: 5）
・引き止め完了期限日数 … 申し出からN日以内にクロージング必須（初期値: 14）
・高単価アラート閾値MRR … 円以上の顧客申し出は即時エスカレーション（初期値: 50000）
・期限前リマインダー日数 … 期限のN日前にリマインダー送信（初期値: 3）
・通知先メールアドレス（管理者）… 管理者通知先（初期値: admin@company.com）
・ウィンバック率目標 … %（初期値: 40）
・データ保持期間（月）… 月（初期値: 36）

値を変更して「設定を保存」をクリックしてください。


--------------------------------------------
11. トリガー管理
--------------------------------------------
自動処理の一覧表示と手動実行ができます。

【トリガー一覧】
1. シート初期化 … 全シートのヘッダーと設定初期値を作成（手動）
2. 対応期限アラート … 毎日9:00に期限超過・期限間近の通知を送信
3. 次回アクションリマインダー … 毎日9:30に次回アクション予定のリマインダーを送信
4. 月次サマリー集計 … 毎月1日3:00に前月の申し出データを集計
5. 月次解約理由・プラン分析 … 毎月1日3:30に理由別・プラン別を集計
6. 月次施策効果集計 … 毎月1日4:00に施策別の効果を集計
7. 月次担当者パフォーマンス … 毎月1日4:30に担当者別の成果を集計
8. 月次トレンド集計 … 毎月1日5:00に月次トレンドを更新
9. 月次レポートメール送信 … 毎月1日9:00にHTML形式の月次レポートを全員に配信

【操作】
・各トリガーの「手動実行」ボタンで即時実行できます
・「全トリガー設定」で自動スケジュールを一括登録
・「全トリガー削除」で全トリガーを一括削除（重複時のリセット用）


=============================================
■ 運用の流れ
=============================================

【日常業務】
1. 解約申し出を受けたら「解約申し出」画面で新規登録
   → 担当者と管理者に通知メールが届きます
2. 対応のたびに「引き止め対応」画面で対応記録を追加
   → ステータスが自動で「対応中」に変わります
3. 結果が確定したら「解約申し出」の編集から結果を入力
   → ステータスと顧客マスタが自動更新されます

【定期確認】
・ダッシュボードで未対応・期限超過を毎日チェック
・月初にレポート画面で前月の集計結果を確認
・施策効果ランキングから効果的な施策を把握し、対応方針を改善

【メール通知（自動）】
・新規申し出通知（登録時即時）
・VIP/高単価顧客エスカレーション（登録時即時）
・対応期限リマインダー（毎日9:00）
・次回アクションリマインダー（毎日9:30）
・引き止め成功通知（結果入力時即時）
・月次レポート（毎月1日9:00）


=============================================
■ 注意事項
=============================================

・顧客マスタのステータスは解約申し出登録・結果入力時に自動更新されます。
  手動で変更する場合は整合性にご注意ください。
・月次集計データは毎月1日に前月分が自動集計されます。
  すぐに確認したい場合はトリガー管理から手動実行してください。
・設定の「通知先メールアドレス（管理者）」は運用開始前に
  実際の管理者のメールアドレスに変更してください。
・担当者マスタのメールアドレスも実際のアドレスを設定してください。
  メール通知はこのアドレスに送信されます。
