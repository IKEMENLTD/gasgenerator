===========================================================
  納期アラートシステム 使い方説明書
===========================================================

【このシステムでできること】
------------------------------------------------------------
案件の納期を一元管理し、期限が迫った案件を自動でメール
通知するシステムです。以下のような用途に活用できます。

- 複数の案件の納期を一覧で把握する
- 納期超過や期限間近の案件をひと目で確認する
- 担当者ごとに納期アラートメールを自動送信する
- 案件のステータス（進行中・確認待ち・完了・納品済）を管理する
- 優先度（高・中・低）で案件の重要度を区別する
- クライアント名・担当者・キーワードで案件を絞り込む

制作会社、デザイン事務所、フリーランスなど、
複数案件を同時進行で管理する方に最適です。


===========================================================
  Webアプリの使い方（全画面の説明）
===========================================================

------------------------------------------------------------
■ ダッシュボード（トップページ）
------------------------------------------------------------
アプリを開くと最初に表示される画面です。
案件全体の状況をひと目で把握できます。

【統計カード（画面上部の4つの数字）】
  - 進行中案件 ... 完了・納品済以外の案件数
  - 超過案件   ... 納期を過ぎた案件数
  - 今週納期   ... 今日から7日以内に納期がある案件数
  - 今月完了   ... 今月中に完了・納品済になった案件数

【超過・緊急案件リスト（左側）】
  納期を過ぎた案件、または残り3日以内の案件が
  一覧表示されます。赤は超過、黄色は期限間近です。

【今週の納期リスト（右側）】
  今日から7日以内に納期がある案件の一覧です。

【直近の案件テーブル（下部）】
  最近の案件10件が納期順に表示されます。
  「すべて見る」ボタンで案件一覧ページに移動できます。

------------------------------------------------------------
■ 案件登録
------------------------------------------------------------
新しい案件を登録する画面です。

【入力項目】
  - 案件名（必須）     ... 案件の名前
  - クライアント名（必須）... 取引先の名前
  - 担当者（必須）     ... ドロップダウンから選択
                        （担当者は設定画面で事前登録）
  - 受注日           ... 受注した日付
  - 納期（必須）      ... 納品期限の日付
  - ステータス        ... 進行中 / 確認待ち / 完了 / 納品済
  - 優先度           ... 高 / 中 / 低
  - 備考            ... メモや注意事項など

【操作方法】
  1. 各項目を入力する
  2. 「登録する」ボタンを押す
  3. 「登録完了」画面が表示される
  4. 「続けて登録」で別の案件を追加、
     「案件一覧へ」で一覧画面に移動

【案件の編集】
  案件一覧の「編集」ボタンから、この画面が
  「案件編集」モードで開きます。
  内容を変更して「更新する」ボタンで保存します。

------------------------------------------------------------
■ 案件一覧
------------------------------------------------------------
登録済みの全案件を一覧表示する画面です。

【フィルター機能（画面上部）】
  - ステータス   ... 進行中 / 確認待ち / 完了 / 納品済 で絞り込み
  - 担当者      ... 担当者名で絞り込み
  - 優先度      ... 高 / 中 / 低 で絞り込み
  - キーワード検索 ... 案件名やクライアント名で検索

【案件テーブル】
  各案件の以下の情報が表示されます：
  - 案件名 / クライアント / 担当者 / 納期 / 残り日数
  - ステータス / 優先度 / 操作ボタン

  残り日数の色：
    赤字 ... 納期超過
    黄字 ... 残り3日以内
    黒字 ... 4日以上余裕あり

【ステータス変更】
  ステータスのバッジをクリックすると
  変更ダイアログが開きます。
  新しいステータスを選んで「変更する」を押します。

【案件の編集】
  操作列の編集ボタン（ペンのアイコン）を押すと
  案件登録画面が編集モードで開きます。

【案件の削除】
  操作列の削除ボタン（ゴミ箱のアイコン）を押すと
  確認ダイアログが表示されます。
  「削除する」を押すと案件が削除されます。
  ※削除した案件は元に戻せません。

------------------------------------------------------------
■ 設定
------------------------------------------------------------
担当者の管理とアラートの設定を行う画面です。

【担当者管理（左側）】
  メールアラートの送信先となる担当者を管理します。

  ・追加：
    1. 「+ 追加」ボタンを押す
    2. 氏名とメールアドレスを入力
    3. 「追加する」ボタンで登録
    ※ここで登録した担当者が、案件登録の担当者
      ドロップダウンに表示されます。

  ・編集：
    担当者リストの編集ボタン（ペンのアイコン）を押して
    内容を変更し「更新する」を押します。

  ・削除：
    削除ボタン（ゴミ箱のアイコン）を押して
    確認ダイアログで「削除する」を押します。

【アラート設定（右側）】
  メールアラートを送信するタイミングを設定します。
  チェックボックスのON/OFFで切り替えます。

  - 7日前にアラート   ... 納期7日前にリマインダーメール
  - 3日前にアラート   ... 納期3日前に注意喚起メール
  - 1日前にアラート   ... 納期前日に警告メール
  - 当日アラート      ... 納期当日に緊急メール
  - 超過時アラート    ... 納期超過後、毎日緊急メール

  設定を変更したら「設定を保存」ボタンを押してください。


===========================================================
  スプレッドシートの各タブ説明
===========================================================

データはすべてGoogleスプレッドシートに保存されます。
Webアプリを通さず、直接スプレッドシートで確認・編集も可能です。

------------------------------------------------------------
■ 案件マスタ タブ
------------------------------------------------------------
全案件のデータが保存されるメインのタブです。

【列の説明】
  A列: 案件ID       ... 自動生成される固有ID
  B列: 案件名       ... 案件の名称
  C列: クライアント名 ... 取引先名
  D列: 担当者       ... 担当者名
  E列: 受注日       ... 受注した日付
  F列: 納期         ... 納品期限
  G列: ステータス    ... 進行中 / 確認待ち / 完了 / 納品済
  H列: 優先度       ... 高 / 中 / 低
  I列: 備考         ... メモ・注意事項
  J列: アラートフラグ ... 送信済みアラートの記録（自動管理）

※アラートフラグ列はシステムが自動で管理します。
  手動で編集しないでください。

------------------------------------------------------------
■ 担当者マスタ タブ
------------------------------------------------------------
案件の担当者情報を管理するタブです。

【列の説明】
  A列: 担当者ID      ... 自動生成される固有ID
  B列: 氏名          ... 担当者の名前
  C列: メールアドレス  ... アラートメールの送信先

※メールアドレスが未登録の担当者にはアラートが送信されません。

------------------------------------------------------------
■ アラート設定 タブ
------------------------------------------------------------
アラートメールの送信タイミングを管理するタブです。

【列の説明】
  A列: アラートID        ... 設定の番号
  B列: 通知タイミング（日前）... 何日前に通知するか
                           （-1 = 超過後毎日）
  C列: 有効/無効         ... 「有効」または「無効」

【初期設定】
  ID 1: 7日前  - 有効
  ID 2: 3日前  - 有効
  ID 3: 1日前  - 有効
  ID 4: 当日   - 有効
  ID 5: 超過時 - 有効

------------------------------------------------------------
■ アラートログ タブ
------------------------------------------------------------
アラートメールの送信履歴が自動記録されるタブです。

【列の説明】
  A列: 送信日時   ... メールを送信した日時
  B列: 案件ID    ... 対象案件のID
  C列: 案件名    ... 対象案件の名前
  D列: 納期      ... 案件の納期
  E列: 残り日数   ... 送信時点の残り日数（マイナス=超過）
  F列: 担当者    ... メール送信先の担当者名
  G列: 送信結果   ... 送信成功 / 送信失敗 / メールアドレス未設定

※このタブはシステムが自動で記録します。
  アラートが正しく送信されているか確認するときに使います。


===========================================================
  アラートメールについて
===========================================================

【メールの件名パターン】
  - 7日前〜4日前:  【納期リマインド】案件名 (残りX日)
  - 3日前〜2日前:  【納期間近】案件名 (残りX日)
  - 1日前:        【明日納期】案件名
  - 当日:         【本日納期】案件名
  - 超過後:       【緊急】納期超過: 案件名

【メールの内容】
  案件名、クライアント名、担当者名、納期、
  優先度、ステータス、備考が記載されます。

【送信条件】
  - ステータスが「完了」「納品済」以外の案件が対象
  - 担当者マスタにメールアドレスが登録されていること
  - アラート設定で該当のタイミングが「有効」であること
  - 同じタイミングのアラートは1案件につき1回のみ送信
    （超過時アラートは毎日送信）

【自動送信の仕組み】
  毎日朝9時に自動でチェックが実行され、
  条件に合致する案件のアラートメールが送信されます。


===========================================================
  よくある質問
===========================================================

Q: 案件を直接スプレッドシートで編集しても大丈夫？
A: はい。案件マスタ・担当者マスタのB列以降は
   直接編集しても問題ありません。
   ただしA列（ID）とJ列（アラートフラグ）は
   変更しないでください。

Q: 担当者を追加したのに案件登録で表示されない
A: 設定画面で担当者を登録した後、案件登録画面を
   再読み込み（リロード）してください。

Q: アラートメールが届かない
A: 以下を確認してください：
   1. 担当者マスタにメールアドレスが登録されているか
   2. アラート設定タブで該当タイミングが「有効」か
   3. 案件のステータスが「完了」「納品済」以外か
   4. アラートログタブで送信結果を確認

Q: 削除した案件を復元したい
A: Webアプリからの削除は元に戻せません。
   スプレッドシートの変更履歴（ファイル→変更履歴）
   から復元できる場合があります。

Q: スマートフォンから使える？
A: はい。レスポンシブデザインに対応しているので、
   スマートフォンのブラウザからも利用できます。
   画面下部にナビゲーションバーが表示されます。
