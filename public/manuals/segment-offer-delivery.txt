================================================================================
  顧客セグメント別オファー配信システム — 使い方説明書
================================================================================

■ このシステムでできること
────────────────────────────────────────────────────────────────

全顧客に同じキャンペーンを配信していませんか？
本システムは、顧客の購買履歴・来店頻度・購入金額をRFM分析で自動スコアリングし、
セグメントごとに最適なオファーをメール配信します。

・VIP顧客には限定割引、休眠顧客には復帰キャンペーンなど、
  顧客の状態に合わせた訴求でCVR・ROIを向上させます。
・配信後の開封率・クリック率・コンバージョン率を自動追跡し、
  ダッシュボードで効果を可視化します。

【活用例】
  - EC事業: 購入頻度が下がった顧客にクーポンを自動配信
  - 店舗運営: 来店回数に応じたランク別の限定案内を配信
  - サービス業: 休眠顧客にイベント招待メールで再来店を促進
  - マーケ施策の効果測定: セグメント別にCVR・売上貢献額を比較分析

■ 画面構成
================================================================================

本システムは6つのタブで構成されています。

┌──────────────┬────────────────────────────────────┐
│ タブ名        │ 概要                               │
├──────────────┼────────────────────────────────────┤
│ ダッシュボード │ KPI・6種グラフで配信効果を可視化    │
│ 顧客管理      │ 顧客情報の登録・編集・削除          │
│ 購買履歴      │ 購入データの登録・編集・削除        │
│ セグメント設定 │ 顧客セグメントの条件設定            │
│ オファー設定   │ 配信メールのテンプレート管理        │
│ 配信ログ      │ メール配信結果の閲覧（自動記録）    │
└──────────────┴────────────────────────────────────┘

■ ダッシュボード
================================================================================

画面上部にKPIカード、下部に6つのグラフが表示されます。

【KPIカード】
  - 累計売上貢献額: コンバージョン顧客の購入金額合計
  - 有効顧客数: 有効フラグ=有効の顧客数
  - 総配信数: メール配信の総件数
  - 平均開封率: 成功配信のうち開封された割合
  - 平均CV率: 成功配信のうちコンバージョンに至った割合

【グラフ一覧】
  1. セグメント別配信件数（横棒グラフ）
     → どのセグメントに多く配信しているかを確認
  2. セグメント別開封率推移（折れ線グラフ）
     → 各セグメントの開封率の日次推移を追跡
  3. セグメント別コンバージョン率（縦棒グラフ）
     → セグメントごとのCVRを比較
  4. 月次売上貢献額推移（面グラフ）
     → 月ごとの売上貢献額の推移を確認
  5. RFMスコア分布（ヒストグラム）
     → 顧客のスコア分布を10刻みで表示
  6. 業種別セグメント構成比（積み上げ棒グラフ）
     → 業種ごとのセグメント割合を可視化

■ 顧客管理
================================================================================

顧客情報を登録・管理するタブです。

【新規登録】
  1. 「+ 新規登録」ボタンをクリック
  2. 以下の項目を入力:
     - 顧客名（必須）
     - メールアドレス（必須）: 配信先のメールアドレス
     - 電話番号（任意）
     - 業種区分: EC / 店舗 / サービス業
     - 会員ランク: ブロンズ / シルバー / ゴールド / プラチナ
     - 有効フラグ: 有効 / 無効
     - 配信停止フラグ: 配信可 / 配信停止
     - 備考（任意）
  3. 「保存」をクリック

  ※ 顧客IDはシステムが自動採番します（C-xxxxxxxx形式）
  ※ 最終来店日・来店回数・累計購入金額は購買データから自動集計されます

【編集・削除】
  - 各行の「編集」ボタンで内容を修正できます
  - 各行の「削除」ボタンで顧客を削除できます

【検索】
  - 右上の検索ボックスにキーワードを入力すると、
    顧客名・メール・業種・会員ランク・セグメント名で絞り込めます

【セグメント列】
  - RFMスコア算出後、自動で割り当てられたセグメント名が表示されます
  - 未算出の場合は「未算出」と表示されます

【表示ルール】
  - 有効フラグ=無効の顧客はグレーアウト表示になります
  - 配信停止フラグ=配信停止は赤字で表示されます

================================================================================
■ 購買履歴
================================================================================

顧客の購入データを登録・管理するタブです。

【新規登録】
  1. 「+ 新規登録」ボタンをクリック
  2. 以下の項目を入力:
     - 顧客（必須）: ドロップダウンから選択
     - 購入日（必須）: 日付を選択
     - 商品カテゴリ（必須）: 食品 / 衣類 / 家電 / 美容 / サービス / その他
     - 商品名（必須）: テキスト入力
     - 購入金額（必須）: 金額を入力（円）
     - 購入チャネル: EC / 店舗 / アプリ
  3. 「保存」をクリック

  ※ 取引IDはシステムが自動採番します（T-xxxxxxxx形式）
  ※ キャンペーン経由フラグはシステムが自動判定します（編集不可）
  ※ 登録日時は自動記録されます

【フィルタ】
  - 顧客: 全顧客またはドロップダウンで特定の顧客に絞り込み
  - 開始日・終了日: 日付範囲で絞り込み
  - カテゴリ: 商品カテゴリで絞り込み

【表示順】
  - 購入日の新しい順に表示されます

■ セグメント設定
================================================================================

顧客をグループ分けする条件を設定するタブです。

【セグメントとは】
  RFMスコア（購買行動を数値化したもの）に基づいて顧客を分類するグループです。
  例: VIP顧客（スコア70〜100）、一般顧客（スコア30〜69）、休眠顧客（スコア0〜20）

【新規登録】
  1. 「+ 新規登録」ボタンをクリック
  2. 以下の項目を入力:
     - セグメント名（必須）: グループの名前
     - 正規化スコア下限（必須）: 0〜100の整数
     - 正規化スコア上限（必須）: 下限より大きい値
     - 来店回数条件（下限）（任意）: 空欄=条件なし
     - 会員ランク条件: 指定なし / ブロンズ / シルバー / ゴールド / プラチナ
     - 業種区分条件: 全対象 / EC / 店舗 / サービス業
     - 対応オファーID: このセグメントに配信するオファーを選択
     - 優先順位: 小さい数字ほど優先（複数条件に合致する場合の優先度）
     - 有効フラグ: 有効 / 無効
  3. 「保存」をクリック

  ※ セグメントIDはシステムが自動採番します（S-xxxxxxxx形式）

【重要: スコア範囲の設定】
  - スコア範囲にギャップがあると、その範囲の顧客は「未分類」になります
  - 例: VIP=70〜100、一般=30〜69、休眠=0〜29 → ギャップなし（推奨）
  - 例: VIP=70〜100、一般=30〜69、休眠=0〜20 → 21〜29が未分類になる

【対応オファーIDの設定】
  - 各セグメントにオファーIDを紐づけると、配信バッチで自動配信されます
  - オファーIDはオファー設定タブで確認できます

■ オファー設定
================================================================================

配信するメールのテンプレートを管理するタブです。

【新規登録】
  1. 「+ 新規登録」ボタンをクリック
  2. 以下の項目を入力:
     - オファー名（必須）: 管理用の名前
     - 件名テンプレート（必須）: メールの件名
     - 本文テンプレート（必須）: メールの本文
     - オファー種別（必須）: 割引クーポン / ポイント付与 / 限定商品案内 /
                             イベント招待 / その他
     - 割引率（%）: 0〜100（割引なしの場合は0）
     - 有効期限（日数）: 配信日からの有効日数（デフォルト30日）
     - 対象業種: 全対象 / EC / 店舗 / サービス業
     - 有効フラグ: 有効 / 無効
  3. 「保存」をクリック

  ※ オファーIDはシステムが自動採番します（O-xxxxxxxx形式）

【プレースホルダ】
  件名・本文に以下のプレースホルダを使用できます:
  - {{顧客名}}: 配信時に顧客名に自動置換されます
  - 例: 件名「{{顧客名}}様限定！20%OFFクーポン」
        → 「田中太郎様限定！20%OFFクーポン」として配信

■ 配信ログ
================================================================================

メール配信の結果を確認するタブです（閲覧専用）。

【表示項目】
  - 配信ID: 各配信の一意キー
  - 顧客名: 配信先の顧客名
  - オファー: 配信したオファー名
  - セグメント: 顧客が属するセグメント名
  - 配信日時: メールが送信された日時
  - ステータス: 成功 / 失敗
  - 開封: 開封済 / 未開封（自動更新）
  - クリック: クリック済 / 未クリック（自動更新）
  - CV: 転換済 / 未転換（自動更新）
  - バッチID: 同一配信バッチのグループID

【フィルタ】
  - 顧客: 全顧客またはドロップダウンで特定の顧客に絞り込み
  - 開始日・終了日: 日付範囲で絞り込み
  - ステータス: 成功 / 失敗 / バウンス

【表示順】
  - 配信日時の新しい順に表示されます

【ステータスバッジ】
  - 成功: 青色バッジ
  - 開封済: 緑色バッジ
  - クリック済: 緑色バッジ

■ RFMスコアリングの仕組み
================================================================================

RFMスコアは3つの指標で顧客の購買行動を数値化します。

【3つの指標】
  R (Recency)  : 最後に購入してからの経過日数
  F (Frequency): 過去365日間の購入回数
  M (Monetary) : 過去365日間の購入金額合計

【スコア算出基準】（各1〜5点）

  Rスコア:
    30日以内=5, 31〜60日=4, 61〜90日=3, 91〜180日=2, 181日以上=1

  Fスコア:
    10回以上=5, 7〜9回=4, 4〜6回=3, 2〜3回=2, 1回以下=1

  Mスコア:
    100,001円以上=5, 50,001〜100,000円=4, 20,001〜50,000円=3,
    5,001〜20,000円=2, 5,000円以下=1

【合計スコアと正規化】
  RFM合計スコア = Rスコア + Fスコア + Mスコア（3〜15点）
  正規化スコア = (合計スコア - 3) ÷ 12 × 100（0〜100点）

  この正規化スコアをセグメント設定の範囲条件と突合して、
  顧客を自動的にセグメントに割り当てます。

■ 自動処理（トリガー）
================================================================================

以下の7つの処理が自動で実行されます。手動操作は不要です。

┌─────┬───────────────────────┬──────────────────────────────────┐
│ 時刻 │ 処理名                │ 内容                             │
├─────┼───────────────────────┼──────────────────────────────────┤
│02:00│顧客集計更新           │購買データから来店数・金額を集計   │
│02:30│RFMスコア算出          │全顧客のRFMスコアとセグメントを更新│
│09:00│セグメント別配信バッチ  │セグメントに応じたオファーメール配信│
│18:00│開封・クリック追跡      │配信メールの開封・クリック状況を更新│
│19:00│コンバージョン判定      │配信後に購入があった顧客をCV判定   │
│20:00│ダッシュボード集計      │当日の配信結果をダッシュボードに集計│
│毎月1日│月次レポート生成      │前月分のサマリーをPDF付きメール送信│
│10:00│                       │                                  │
└─────┴───────────────────────┴──────────────────────────────────┘

【配信条件】
  メールが配信されるには以下の全条件を満たす必要があります:
  1. 顧客の有効フラグ = 有効
  2. 顧客の配信停止フラグ = 配信可
  3. セグメントの有効フラグ = 有効
  4. セグメントに対応オファーIDが設定されている
  5. オファーの有効フラグ = 有効
  6. 同日に同じ顧客×同じオファーの配信履歴がない（重複防止）

【コンバージョン判定の条件】
  配信日からオファーの有効期限日数以内に購入があった場合、
  その配信を「転換済」、購買履歴を「キャンペーン経由あり」に更新します。

================================================================================
■ 運用のポイント
================================================================================

1. まず顧客と購買履歴を登録してください
   → 翌日02:00以降にRFMスコアが自動算出されます

2. セグメントを設定し、対応オファーIDを紐づけてください
   → スコア範囲にギャップがないように設定することを推奨します

3. オファーテンプレートを作成してください
   → {{顧客名}}プレースホルダで個別化できます

4. 配信結果はダッシュボードで確認してください
   → セグメント別の開封率・CV率を比較し、効果の低いセグメントの
     オファー内容を見直すことで改善サイクルを回せます

5. 月次レポートが毎月1日に管理者メールに届きます
   → 前月の配信実績サマリーがPDF添付で送信されます
