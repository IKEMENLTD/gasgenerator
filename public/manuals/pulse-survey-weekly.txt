=====================================
パルスサーベイ（週次コンディション確認）
使い方説明書
=====================================

■ このシステムでできること
────────────────────────────
・メンバーのコンディション（体調・モチベーション・業務量・職場環境）を
  週次で数値化し、継続的に把握できます
・スコアが低下したメンバーを自動検知し、3段階のアラートレベルで
  上長・人事にメール通知します
・チーム別の平均スコアやトレンドをダッシュボードで可視化し、
  組織全体のコンディションを一目で把握できます
・離職やバーンアウトの兆候を早期に発見し、適切な対応につなげられます


■ 画面構成（5タブ）
────────────────────────────

1. ダッシュボード
2. メンバー管理
3. 回答入力
4. 週次コンディション
5. アラートログ


■ 各画面の使い方
────────────────────────────

【1. ダッシュボード】

システム全体の状況を一目で確認できる画面です。

● 統計カード（4つ）
  - 有効メンバー数: 配布対象のメンバー数
  - 回答数: 今週の回答数
  - 全体平均スコア: 今週の全メンバーの平均（0〜100点）
    ※ 50点未満の場合は赤色で警告表示されます
  - アラート発生数: 今週のアラート件数
    ※ 1件以上の場合は黄色で警告表示されます

● グラフ（6つ）
  - チーム別平均スコア（今週）: チームごとの平均スコアを棒グラフで表示
  - 週次スコアトレンド（全体平均）: 直近12週の全体平均スコアの推移
  - アラートレベル別メンバー数: なし/Lv1/Lv2/Lv3の分布をドーナツグラフで表示
  - 週次アラート発生数（レベル別）: 直近12週のアラート発生数を積み上げ棒グラフで表示
  - 未回答率（週次）: 直近12週の未回答率の推移
  - 連続低スコア者一覧（今週）: 2週以上連続で低スコアのメンバー一覧


【2. メンバー管理】

サーベイ対象のメンバーを管理する画面です。

● メンバー追加
  1. 「+ メンバー追加」ボタンをクリック
  2. モーダルで以下を入力
     - 氏名（必須）
     - メールアドレス（必須）
     - 所属チーム: 開発 / 営業 / マーケ / 人事 / その他
     - 上長メールアドレス（必須）: アラート通知先
     - 人事担当メールアドレス（必須）: アラート通知先
     - 備考（任意）
  3. 「保存」をクリック
  ※ MemberID（M001, M002...）と登録日は自動で付与されます

● メンバー編集
  1. 対象メンバーの「編集」ボタンをクリック
  2. モーダルで情報を修正
  3. 「保存」をクリック
  ※ 有効/無効の切替もこの画面で行えます

● メンバー削除
  1. 対象メンバーの「削除」ボタン（赤）をクリック
  2. 確認ダイアログで「OK」をクリック
  ※ スプレッドシートからデータが完全に削除されます


【3. 回答入力】

メンバーのコンディション回答を登録する画面です。
Google Formsの代わりに管理画面から直接入力できます。

● 回答の登録
  1. 「+ 回答を入力」ボタンをクリック
  2. モーダルで以下を入力
     - 対象メンバー: ドロップダウンから選択（有効メンバーのみ表示）
     - Q1: 今週の体調は？（1〜5）
     - Q2: 仕事へのモチベーションは？（1〜5）
     - Q3: 業務量は適切か？（1〜5）
     - Q4: 職場環境・人間関係は良好か？（1〜5）
     - Q5: 誰かに相談したいことがあるか？（1=はい / 2=いいえ）
  3. 「回答を登録」をクリック

  ※ 登録と同時に以下が自動計算されます
    - 合計スコア = Q1 + Q2 + Q3 + Q4（4〜20点）
    - 正規化スコア = (合計 - 4) / 16 × 100（0〜100点）
    - 週番号（ISO 8601形式: YYYY-Www）
    - MemberID（メールアドレスから自動照合）

● 登録済み回答データ
  下部のテーブルに登録済みの回答が新しい順で表示されます。


【4. 週次コンディション】

週ごと・メンバーごとの集計データを閲覧する画面です。
データは「集計・アラート処理を実行」後に反映されます。

● 表示項目
  - 週番号 / MemberID / 氏名 / 所属チーム
  - 正規化スコア: バッジで色分け表示
    ・青: 60点以上（良好）
    ・水色: 50〜59点（注意）
    ・黄: 40〜49点（警告）
    ・赤: 40点未満（危険）
  - 前週比スコア差: +/-表示
  - 連続低スコア週数: 50点未満が続いた週数
  - アラートレベル: なし / Lv1 / Lv2 / Lv3

● フィルタ
  - 週番号: 特定の週を絞り込み
  - チーム: 所属チームで絞り込み
  - アラート: Lv1 / Lv2 / Lv3 で絞り込み

● 並び順
  週番号が新しい順、同一週内はスコアが低い順（要注意者が上）


【5. アラートログ】

送信されたアラートの履歴を確認する画面です。

● 表示項目
  AlertID / 送信日時 / MemberID / 氏名 / 週番号 /
  アラートレベル / 発動条件 / 正規化スコア / 前週比スコア差 / 送信結果

● フィルタ
  - 開始日〜終了日: 期間で絞り込み
  - アラートレベル: Lv1 / Lv2 / Lv3
  - 発動条件: C1 / C2 / C3 / C4 など
  - 送信結果: 成功 / 失敗

● CSV出力
  「CSV出力」ボタンで、フィルタ適用後のデータをCSVダウンロードできます。
  ファイル名: AlertLog_YYYYMMDD.csv

● 集計・アラート処理を実行
  「集計・アラート処理を実行」ボタンで、手動でアラート判定と
  週次集計を実行できます。通常は毎週月曜に自動実行されます。


■ アラート判定のしくみ
────────────────────────────

回答データから以下の4条件を自動判定し、3段階のアラートを送信します。

● 判定条件
  C1（単発低スコア）: 正規化スコア < 40
  C2（連続低スコア）: 今週 < 50 かつ 前週 < 50
  C3（急激悪化）    : 前週比スコア差 < -25
  C4（SOSサイン）   : Q5 = 1（相談したいことがある）

● アラートレベル（優先順）
  Lv3【緊急】: C4（SOS）、または C1かつC3が同時成立
    → 上長＋人事にメール送信
    → 当日中に面談実施推奨

  Lv2【警告】: C1、C2、C3 のいずれか（Lv3非該当時）
    → 上長＋人事にメール送信
    → 今週中に声がけ推奨

  Lv1【注意】: スコア50〜59 かつ 前週比 < -10
    → 上長のみにメール送信
    → 来週も継続してウォッチ

● 未回答者への対応
  2週連続で未回答のメンバーがいる場合、上長・人事に確認依頼メールが
  自動送信されます。


■ 自動処理スケジュール
────────────────────────────

毎週月曜日に以下の処理が自動実行されます。

  7:30  フォーム配布メール送信（有効メンバー全員）
  8:30  スコア算出（回答データの集計列を更新）
  9:00  アラート判定・メール送信
 10:00  週次集計の更新

※ 手動で即時実行したい場合は、アラートログページの
  「集計・アラート処理を実行」ボタンを使用してください。


■ スコアの計算方法
────────────────────────────

  合計スコア = Q1 + Q2 + Q3 + Q4（範囲: 4〜20点）
  ※ Q5（SOS）は合計に含みません

  正規化スコア = (合計スコア - 4) ÷ 16 × 100（範囲: 0〜100点）

  例: Q1=3, Q2=2, Q3=3, Q4=2 の場合
      合計 = 10、正規化 = (10-4)/16×100 = 37.5点


■ スプレッドシートの構成
────────────────────────────

  メンバーマスタ  : メンバー情報の管理
  回答生データ    : 回答の生データ（自動集計列付き）
  週次集計        : 週×メンバー単位の集計データ
  アラートログ    : 送信済みアラートの履歴


■ 注意事項
────────────────────────────

・コンディションスコアは個人情報として扱ってください
・回答内容はメンバー本人には表示されません（上長・人事のみ閲覧可）
・スプレッドシートのアクセス権限は管理職・人事のみに限定してください
・フォーム配布メールを使用するにはGoogle FormsのURLを
設定する必要があります
