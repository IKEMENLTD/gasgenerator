================================================================================
定例MTGアジェンダ自動収集システム — 使い方説明書
================================================================================

【このシステムでできること】

  定例ミーティングの運営に必要な作業を自動化するシステムです。

  ・メンバーがWebアプリから議題を投稿できる
  ・議題収集の開始をメールで自動通知
  ・未提出者へのリマインドを自動送信
  ・投稿された議題からアジェンダを自動生成・メール配信
  ・議事録テンプレートを自動作成
  ・過去の議題を検索できる
  ・議題に優先度の投票ができる

  これにより「議題をチャットで集める」「手動でアジェンダを作る」
  「リマインドを個別に送る」といった手作業が不要になります。


================================================================================
1. スプレッドシートの使い方（管理者向け）
================================================================================

  スプレッドシートを開くと、上部メニューに「MTGアジェンダ」が
  表示されます。ここから各種操作ができます。

  ┌─────────────────────────────────────────────┐
  │ MTGアジェンダ メニュー                      │
  ├─────────────────────────────────────────────┤
  │ トリガーを設定      自動通知を開始する      │
  │ トリガーを削除      自動通知を停止する      │
  │ ─────────────────── ────────────────────── │
  │ 手動: 議題収集開始通知  今すぐ収集開始メール │
  │ 手動: リマインド送信    今すぐリマインド送信 │
  │ 手動: アジェンダ生成    今すぐアジェンダ作成 │
  │ ─────────────────── ────────────────────── │
  │ WebアプリURLを設定  WebアプリのURLを登録     │
  └─────────────────────────────────────────────┘

  ※メニューが表示されない場合は、スプレッドシートを
    一度閉じて開き直してください。


── 1-1. 自動通知を設定する ──

  メニュー「MTGアジェンダ」→「トリガーを設定」を選択

  以下のスケジュールで自動実行されます:
    ・毎日  8時 → 議題収集開始通知 ＆ アジェンダ配信
    ・毎日 18時 → 未提出者へリマインド

  ※一度設定すれば、以降は自動で動き続けます。
  ※停止したい場合は「トリガーを削除」を選択してください。


── 1-2. 手動で通知・アジェンダ生成する ──

  自動通知の設定とは別に、必要なときに手動で実行できます。

  ・「手動: 議題収集開始通知」
    → 対象MTGのメンバーに議題収集のメールを送信

  ・「手動: リマインド送信」
    → 議題未提出のメンバーにリマインドメールを送信

  ・「手動: アジェンダ生成」
    → 投稿済みの議題からアジェンダを作成し、メールで配信


── 1-3. WebアプリURLを設定する ──

  メニュー「MTGアジェンダ」→「WebアプリURLを設定」

  通知メールに含まれるWebアプリへのリンクURLを設定します。
  GASをデプロイした際に発行されたURLを入力してください。


================================================================================
2. シートの説明
================================================================================

  スプレッドシートには7つのシートがあります。

  ── メンバーマスタ ──
    メンバー情報を管理するシートです。
    列: メンバーID / 氏名 / メールアドレス / 所属チーム /
        役職 / Slack ID / ステータス

    ・新しいメンバーを追加するには、行を追加して情報を入力
    ・ステータスを「休止」にすると通知対象から外れます

  ── 定例MTG設定 ──
    MTGの情報を管理するシートです。
    列: MTG ID / MTG名 / 開催曜日 / 開催時間 / 開催頻度 /
        参加メンバー / 議題収集開始（日前）/ 議題締切（日前）/
        アジェンダ配信（日前）/ ファシリテーター / 議事録担当 /
        ステータス

    ・「参加メンバー」にはメンバーIDをカンマ区切りで入力
      例: MEM001,MEM002,MEM003
    ・開催頻度は「毎週」「隔週」「毎月」から選択
    ・ステータスを「停止」にすると通知対象から外れます

  ── 議題収集 ──
    Webアプリから投稿された議題が記録されるシートです。
    自動で追加されるため、通常は編集不要です。

  ── アジェンダ ──
    自動生成されたアジェンダが記録されるシートです。
    議題の優先度順に並べられ、配分時間が設定されます。

  ── 議事録 ──
    アジェンダ生成時に自動作成される議事録テンプレートです。
    MTG後に「議論内容」「決定事項」「アクションアイテム」等を
    記入してください。

  ── 通知ログ ──
    送信されたメールの履歴が記録されるシートです。
    送信日時・通知種別・宛先・送信結果が確認できます。

  ── 提出状況 ──
    各メンバーの議題提出状況が記録されるシートです。
    「提出済」「なし報告」「未提出」のステータスが確認できます。

3. Webアプリの使い方（メンバー向け）
================================================================================

  メール通知に記載されたURLを開くと、Webアプリが表示されます。
  3つのタブがあります。


── 3-1. ホームタブ — 議題を投稿する ──

  (1) MTGを選択
      表示されたMTG一覧から、参加するMTGをタップします。

  (2) 名前を選択
      自分の名前をタップします。

  (3) 「議題を投稿する」または「議題なしを報告」を選択

      ■ 議題を投稿する場合:
        以下を入力して「送信する」をタップします。

        ・議題タイトル（必須）
          例: 新機能の進捗共有
        ・議題詳細
          議題の背景や話し合いたいポイントを記入
        ・議題種別（必須）
          「共有」「相談」「決定」「ブレスト」から選択
        ・希望時間
          この議題に使いたい時間（5〜30分）
        ・優先度
          「高」「中」「低」から選択
        ・関連資料URL
          参考資料があればURLを入力

        ※タイトル入力中に過去の類似議題がサジェスト表示されます。
        ※複数の議題がある場合は「続けて投稿」で追加できます。

      ■ 議題がない場合:
        「議題なしを報告」をタップするだけで完了です。
        これにより管理者がリマインド不要と判断できます。


── 3-2. 検索タブ — 過去の議題を検索する ──

  過去に投稿された議題を検索できます。

  ・キーワード: 議題タイトルや詳細に含まれる文字で検索
  ・議題種別: 「共有」「相談」等で絞り込み
  ・MTG: 特定のMTGの議題だけに絞り込み

  「検索」ボタンをタップすると結果が表示されます。

  活用例:
    ・「前回あの件どうなったっけ？」→ キーワードで検索
    ・「過去の決定事項を確認したい」→ 種別「決定」で検索


── 3-3. 投票タブ — 議題の優先度を投票する ──

  投稿された議題に対して優先度の投票ができます。

  (1) MTGを選択
  (2) 議題一覧が表示される
  (3) 各議題の星（☆☆☆☆☆）をタップして1〜5で評価

  ※投票するには、先にホームタブでメンバーを選択してください。
  ※投票結果（平均評価・投票数）はリアルタイムで表示されます。
  ※投票結果はアジェンダの並び順の参考にできます。


================================================================================
4. 自動通知のタイミング
================================================================================

  以下のタイミングでメンバーにメールが届きます。

  ┌──────────┬──────────────────────────────────────┐
  │ タイミング │ 内容                                 │
  ├──────────┼──────────────────────────────────────┤
  │ MTG数日前 │ 議題収集開始の通知メール               │
  │          │ （Webアプリへのリンク付き）             │
  ├──────────┼──────────────────────────────────────┤
  │ 締切当日  │ 未提出者へのリマインドメール           │
  │ (18時)   │                                      │
  ├──────────┼──────────────────────────────────────┤
  │ MTG前日  │ アジェンダ配信メール                   │
  │          │ （全議題・時間配分・資料リンク付き）    │
  ├──────────┼──────────────────────────────────────┤
  │ MTG翌日  │ 議事録担当者へ議事録入力リマインド     │
  └──────────┴──────────────────────────────────────┘

  ※「数日前」はシートの「議題収集開始（日前）」列の設定値です。
  ※通知ログシートで送信履歴を確認できます。

5. よくある質問
================================================================================

  Q. メンバーを追加・変更したい
  A. 「メンバーマスタ」シートに行を追加・編集してください。
     追加後、「定例MTG設定」シートの「参加メンバー」列に
     新しいメンバーIDを追加してください。

  Q. 新しい定例MTGを追加したい
  A. 「定例MTG設定」シートに行を追加してください。
     MTG IDは既存と重複しないユニークな値を設定してください。

  Q. 通知メールが届かない
  A. 以下を確認してください。
     ・メンバーマスタのメールアドレスが正しいか
     ・メンバーのステータスが「参加」になっているか
     ・MTGのステータスが「有効」になっているか
     ・トリガーが設定されているか（メニュー→トリガーを設定）
     ・通知ログシートでエラーが出ていないか

  Q. 議題の締切日を変更したい
  A. 「定例MTG設定」シートの「議題締切（日前）」列を変更します。
     例: 2 → MTGの2日前が締切

  Q. 投稿された議題を削除・編集したい
  A. 「議題収集」シートで直接行を削除・編集できます。

  Q. 一時的にMTGを休止したい
  A. 「定例MTG設定」シートのステータスを「停止」にしてください。
     通知・アジェンダ生成の対象から外れます。
     再開時は「有効」に戻してください。
