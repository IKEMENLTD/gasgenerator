=====================================
粗利管理システム 使い方説明書
=====================================

1. システム概要
-------------------------------------
粗利管理システムは、売上・原価データの登録から粗利分析、アラート通知、
シミュレーションまでを一元管理するウェブアプリケーションです。
Google Apps Script (GAS) で動作し、データはGoogleスプレッドシートに保存されます。

アクセスURL:
https://script.google.com/macros/s/AKfycbzmD9RMr-xOQzI0fjGWJJ1iiMbB1Hswv67qTZtIbfh_cf1ck8dgIgID3QnN6KPPIfW-2A/exec


2. 画面構成
-------------------------------------
画面左側のサイドバーから各機能にアクセスできます。
スマートフォンでは左上のメニューボタンでサイドバーを開閉します。

[概要]
  - ダッシュボード

[データ管理]
  - 売上・原価データ
  - 商品マスタ
  - 顧客マスタ
  - 担当者マスタ
  - 粗利目標

[分析]
  - 商品別分析
  - 顧客別分析
  - 担当者別分析
  - 月次サマリー

[アラート・管理]
  - アラート
  - 原価変動
  - シミュレーション
  - アラート設定


3. 各機能の使い方
=====================================

3-1. ダッシュボード
-------------------------------------
システムの全体状況を確認する画面です。

表示内容:
  - KPIカード: 売上合計、粗利合計、粗利率、取引件数などの主要指標
  - 月別粗利推移グラフ: 売上額・粗利額(棒グラフ)と粗利率(折れ線)
  - カテゴリ別粗利構成グラフ: 商品カテゴリごとの粗利構成比
  - 担当者別粗利率グラフ: 担当者ごとの粗利率比較
  - 粗利ステータス内訳グラフ: 高収益/標準/低粗利/赤字の分布
  - 顧客別粗利ランキング: 粗利額上位の顧客一覧
  - 未確認アラート一覧: 対応が必要なアラートの表示

操作:
  - 年月セレクタで表示対象月を切り替え
  - 「更新」ボタンでデータを再読み込み
  - 「サマリー再計算」ボタンで全サマリーシートを再集計


3-2. 売上・原価データ
-------------------------------------
売上取引の登録・編集・削除を行う画面です。

登録手順:
  1. 「+ 新規登録」ボタンをクリック
  2. 以下の項目を入力
     - 取引日: 取引が発生した日付
     - 顧客: ドロップダウンから顧客を選択
     - 担当者: ドロップダウンから担当者を選択
     - 商品: ドロップダウンから商品を選択(選択すると標準原価が自動反映)
     - 売上: 売上金額を入力
     - 実績原価: 実際の原価を入力(商品選択時に自動セットされますが変更可)
  3. 「保存」ボタンをクリック

  ※ 粗利額・粗利率・粗利ステータスはシステムが自動計算します。
  ※ 粗利ステータスの判定基準:
     - 高収益: 粗利率 >= 商品マスタの目標粗利率
     - 標準: 粗利率 >= 商品マスタの最低許容粗利率
     - 低粗利: 粗利率 > 0% かつ最低許容粗利率未満
     - 赤字: 粗利率 <= 0%

フィルタ:
  - 取引年月: 月単位でデータを絞り込み
  - ステータス: 全ステータス / 高収益 / 標準 / 低粗利 / 赤字

編集: 一覧の「編集」ボタンから既存データを修正できます。
削除: 一覧の「削除」ボタンからデータを削除できます(確認ダイアログあり)。


3-3. 商品マスタ
-------------------------------------
商品情報の管理画面です。

登録項目:
  - 商品名 (必須)
  - カテゴリ
  - 標準販売価格 (必須)
  - 標準仕入原価 (必須)
  - 目標粗利率(%): デフォルト30%
  - 最低許容粗利率(%): デフォルト15%
  - 原価変動フラグ: 固定 / 変動
  - 仕入先
  - ステータス: 販売中 / 販売終了

※ 目標粗利率と最低許容粗利率は、売上データの粗利ステータス判定に使用されます。


3-4. 顧客マスタ
-------------------------------------
顧客情報の管理画面です。

登録項目:
  - 顧客名 (必須)
  - 業種
  - 顧客ランク: A / B / C
  - 担当者: ドロップダウンから選択
  - 取引開始日
  - 特別価格適用: あり / なし

一覧では売上合計・粗利合計・平均粗利率・粗利ランクも表示されます。


3-5. 担当者マスタ
-------------------------------------
担当者(営業スタッフ)の管理画面です。

登録項目:
  - 氏名 (必須)
  - 部署
  - 役職
  - メールアドレス (アラート通知先として使用)
  - 売上目標
  - 粗利目標
  - 目標粗利率(%)
  - ステータス


3-6. 粗利目標
-------------------------------------
年度・月別・対象別の粗利目標を設定する画面です。

登録項目:
  - 年度 (必須): 対象年度(例: 2026)
  - 月: 1-12を指定(空欄の場合は年間目標)
  - 対象: 全社 / 部署 / 担当者 / 商品カテゴリ
  - 対象ID
  - 対象名
  - 売上目標
  - 粗利目標額
  - 目標粗利率(%)


3-7. 商品別分析
-------------------------------------
商品ごとの粗利実績を分析する画面です。

  - 年月フィルタで表示期間を選択
  - 横棒グラフで商品別の粗利を比較
  - 一覧表で売上額、粗利額、粗利率、高収益/低粗利/赤字件数を確認


3-8. 顧客別分析
-------------------------------------
顧客ごとの粗利実績を分析する画面です。

  - 年月フィルタで表示期間を選択
  - グラフで顧客別の粗利を比較
  - 一覧表で売上額、粗利額、粗利率、達成率、前月比を確認


3-9. 担当者別分析
-------------------------------------
担当者ごとの粗利実績を分析する画面です。

  - 年月フィルタで表示期間を選択
  - 棒グラフ(売上・粗利)と折れ線(粗利率)の複合グラフ
  - 一覧表で売上額、粗利額、粗利率、目標達成状況、低粗利/赤字件数を確認


3-10. 月次サマリー
-------------------------------------
月ごとの粗利推移を確認する画面です。

  - 棒グラフ(売上合計・粗利合計)と折れ線(粗利率)の複合グラフ
  - 一覧表で売上合計、原価合計、粗利合計、粗利率、目標、達成率、前月比を確認


3-11. アラート
-------------------------------------
粗利低下や赤字取引のアラートを管理する画面です。

フィルタ:
  - ステータス: 未確認 / 確認済 / 対応済
  - アラート種別: 赤字 / 低粗利

アラート対応手順:
  1. 一覧から対象のアラートの「対応」ボタンをクリック
  2. 対応モーダルで以下を入力
     - 対応ステータス: 未確認 / 確認済 / 対応済
     - 原因: 値引き / 原価上昇 / 特別対応 / その他
     - 対応内容: 自由記述
  3. 「保存」をクリック


3-12. 原価変動
-------------------------------------
商品の原価変動を記録・追跡する画面です。

登録手順:
  1. 「+ 原価変動を登録」ボタンをクリック
  2. 以下の項目を入力
     - 商品 (必須): ドロップダウンから選択
     - 変動日: 原価が変動した日付
     - 新しい仕入原価 (必須)
     - 変動理由: 仕入先変更 / 為替 / 原材料高騰 / その他
     - 登録者
     - 対応: 未定 / 価格転嫁 / 吸収 / 仕入先交渉
  3. 「保存」をクリック

※ 変動率は自動計算されます。


3-13. シミュレーション
-------------------------------------
粗利改善策のシミュレーションを行う画面です。

登録手順:
  1. 「+ シミュレーション作成」ボタンをクリック
  2. 以下の項目を入力
     - 対象種別: 商品 / 顧客 / 全体
     - 対象名
     - 現状売上 (必須)
     - 現状粗利率(%) (必須)
     - 施策: 価格改定 / 原価削減 / 商品構成変更
     - 変更値(%): 改善率を入力(例: 5)
     - 作成者
     - 備考
  3. 「保存」をクリック

※ 予測粗利率と改善額はシステムが自動計算して表示します。


3-14. アラート設定
-------------------------------------
アラートの発生条件と通知先を設定する画面です。

登録項目:
  - アラート名 (必須)
  - 条件: 粗利率以下 / 赤字 / 目標未達
  - 条件値: しきい値(例: 粗利率10%以下でアラートなら「10」)
  - 対象: 全取引 / 特定商品 / 特定顧客
  - 通知先: メールアドレス
  - 通知方法: メール / Slack
  - 有効/無効: 有効 / 無効


4. 共通操作
=====================================

4-1. データの登録・更新
-------------------------------------
  - 「+ 新規登録」ボタンで入力フォームを開きます
  - 必須項目(*)を入力して「保存」をクリックします
  - 登録中はスピナーが表示されます。処理完了まで操作をお待ちください
  - 二重登録防止のため、処理中はボタンが無効化されます

4-2. データの編集
-------------------------------------
  - 一覧表の「編集」ボタンをクリックすると、入力フォームに既存データが読み込まれます
  - 修正後「保存」をクリックすると更新されます

4-3. データの削除
-------------------------------------
  - 一覧表の「削除」ボタンをクリックします
  - 確認ダイアログが表示されるので「OK」をクリックすると削除されます

4-4. 金額の表示形式
-------------------------------------
  - 金額は円記号付き3桁区切り(例: ¥1,234,567)で表示されます
  - パーセンテージは%記号付き(例: 25.3%)で表示されます

4-5. スマートフォンでの利用
-------------------------------------
  - 画面幅768px以下ではサイドバーが自動的に隠れます
  - 左上のメニューボタンでサイドバーを開閉できます
  - サイドバー外をタップすると閉じます


5. サマリー再計算について
=====================================
ダッシュボードの「サマリー再計算」ボタンをクリックすると、
以下のサマリーシートが全て再計算されます。

  - 商品別粗利サマリー
  - 顧客別粗利サマリー
  - 担当者別粗利サマリー
  - 月次粗利サマリー

大量のデータがある場合は処理に時間がかかることがあります。
処理中はスピナーが表示されますのでお待ちください。


6. アラート通知について
=====================================
売上データの登録・更新時に、アラート設定の条件に合致するデータが
検出された場合、自動的にアラートリストに追加されます。

通知先メールアドレスが設定されている場合は、メールで通知が送信されます。
通知内容には取引情報、粗利率、アラート種別が含まれます。


7. 注意事項
=====================================
  - データはGoogleスプレッドシートに保存されます。
    スプレッドシートを直接編集した場合、データの整合性が崩れる可能性があります。
    データの追加・編集・削除は本システムから行ってください。
  - 処理中(スピナー表示中)にブラウザを閉じたり、戻るボタンを押さないでください。
  - 商品マスタの目標粗利率・最低許容粗利率を変更した場合、
    既存の売上データの粗利ステータスには自動反映されません。
    サマリー再計算を実行してください。
